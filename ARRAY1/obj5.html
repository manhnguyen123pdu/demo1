PHẦN 1: TÍNH ĐA HÌNH LÀ GÌ?

Định nghĩa

Đa hình (Polymorphism) là việc triệu gọi đến các phương thức được ghi đè từ đối tượng cha trong các đối tượng con.

Cùng là một phương thức được định nghĩa trong đối tượng cha, nhưng các lớp con có thể ghi đè lại hành vi đó theo cách riêng của mình.

1.1. Ví dụ minh họa

Đối tượng: Tứ Giác

Hành vi chung: Tính chu vi (phương thức trừu tượng)

// Định nghĩa hành vi trừu tượng: Tính chu vi
protected abstract void tinhChuVi();

Hình Vuông

@Override
public void tinhChuVi() {
    chuVi = doDaiCanh * 4;
}

Hình Chữ Nhật

@Override
public void tinhChuVi() {
    chuVi = (chieuDai + chieuRong) * 2;
}

Nhận xét

Cùng là một hành vi tinhChuVi() nhưng:

  • Hình Vuông tính chu vi theo công thức riêng: cạnh × 4
  • Hình Chữ Nhật tính chu vi theo công thức riêng: (dài + rộng) × 2

→ Đây chính là ĐA HÌNH

TuGiac.java - Ví dụ đầy đủ
// Lớp cha trừu tượng
public abstract class TuGiac {
    protected double chuVi;
    
    // Phương thức trừu tượng - để lớp con ghi đè
    public abstract void tinhChuVi();
    
    public void inThongTin() {
        System.out.println("Chu vi: " + chuVi);
    }
}

// Lớp con HinhVuong
public class HinhVuong extends TuGiac {
    private double doDaiCanh;
    
    public HinhVuong(double canh) {
        this.doDaiCanh = canh;
    }
    
    @Override
    public void tinhChuVi() {
        chuVi = doDaiCanh * 4;
        System.out.println("Chu vi hình vuông: " + chuVi);
    }
}

// Lớp con HinhChuNhat
public class HinhChuNhat extends TuGiac {
    private double chieuDai, chieuRong;
    
    public HinhChuNhat(double dai, double rong) {
        this.chieuDai = dai;
        this.chieuRong = rong;
    }
    
    @Override
    public void tinhChuVi() {
        chuVi = (chieuDai + chieuRong) * 2;
        System.out.println("Chu vi hình chữ nhật: " + chuVi);
    }
}

PHẦN 2: Ý NGHĨA CỦA ĐA HÌNH

2.1. Quản lý danh sách đối tượng có mối quan hệ kế thừa

Định nghĩa

Khi phải quản lý hai hoặc nhiều loại đối tượng có mối quan hệ kế thừa, ta quy về quản lý danh sách duy nhất một loại đối tượng - đó chính là lớp cha của nhóm đối tượng đó.

Ví dụ: Quản lý hình học

❌ Tách riêng từng loại đối tượng ✔️ Quy về 1 đối tượng cha
HinhChuNhat hcn = new HinhChuNhat();
HinhVuong hv = new HinhVuong();
// Quản lý riêng biệt
TuGiac tg1 = new HinhChuNhat();
TuGiac tg2 = new HinhVuong();
// Quản lý chung qua lớp cha
QuanLyHinhHoc.java
import java.util.ArrayList;
import java.util.List;

public class QuanLyHinhHoc {
    public static void main(String[] args) {
        // Tạo danh sách chứa các tứ giác
        List<TuGiac> danhSach = new ArrayList<>();
        
        // Thêm các hình khác nhau vào danh sách
        danhSach.add(new HinhVuong(5));
        danhSach.add(new HinhChuNhat(6, 4));
        danhSach.add(new HinhVuong(3));
        danhSach.add(new HinhChuNhat(8, 2));
        
        // Tính chu vi cho tất cả hình trong danh sách
        for (TuGiac hinh : danhSach) {
            hinh.tinhChuVi(); // ĐA HÌNH: mỗi hình tự biết cách tính
            hinh.inThongTin();
        }
    }
}

Kết luận

Thay vì quản lý từng loại đối tượng riêng lẻ, chúng ta quy về quản lý một loại đối tượng duy nhất là Tứ Giác.

  • Không cần quan tâm phần tử cụ thể trong danh sách là đối tượng nào
  • Chỉ cần gọi các hành vi chung của lớp Tứ Giác
  • Khi đó, từng phần tử con sẽ tự biết thể hiện đúng bản chất của mình

2.2. Quản lý bộ hành vi của các đối tượng không cùng bản chất

Bài toán

Quản lý hành vi: Tính chu vi và diện tích cho các đối tượng:

  • Hình Tròn
  • Hình Vuông
  • Hình Chữ Nhật

→ 3 đối tượng không cùng bản chất, không thể áp dụng kế thừa thông thường.

Bộ hành vi chung - Interface
// Định nghĩa bộ hành vi chung
interface BoHanhViCVDT {
    void tinhChuVi(); // Tính chu vi
    void tinhDienTich(); // Tính diện tích

PHẦN 3: TỪ KHÓA INSTANCEOF VÀ ÉP KIỂU

3.1. Định nghĩa

instanceof là gì?

instanceof dùng để kiểm tra kiểu dữ liệu thực tế của một đối tượng:

  • Nếu đúng → trả về true
  • Nếu sai → trả về false
Ví dụ instanceof
// Khai báo đối tượng
TuGiac tg1 = new HinhChuNhat(5, 3);

// Kiểm tra kiểu thực tế
boolean kt1 = tg1 instanceof HinhChuNhat; // true
boolean kt2 = tg1 instanceof TuGiac; // true
boolean kt3 = tg1 instanceof HinhVuong; // false

System.out.println("tg1 có phải HinhChuNhat? " + kt1);
System.out.println("tg1 có phải TuGiac? " + kt2);
System.out.println("tg1 có phải HinhVuong? " + kt3);

3.2. Vấn đề phát sinh và cách giải quyết

Vấn đề

Nếu khai báo: TuGiac tg1 = new HinhChuNhat();

Ta chỉ có thể gọi các hành vi được khai báo trong lớp Tứ Giác, không thể gọi các hành vi riêng của Hình Chữ Nhật.

Bài toán: In thông tin chiều dài – chiều rộng của tất cả Hình Chữ Nhật trong danh sách

GiaiQuyetVanDe.java
// Giả sử HinhChuNhat có phương thức riêng
public class HinhChuNhat extends TuGiac {
    private double chieuDai, chieuRong;
    
    // ... constructor và các phương thức khác ...
    
    // Phương thức riêng của HinhChuNhat
    public void inTTCDCR() {
        System.out.println("Chiều dài: " + chieuDai + ", Chiều rộng: " + chieuRong);
    }
}

// Trong chương trình chính
List<TuGiac> danhSach = new ArrayList<>();
danhSach.add(new HinhChuNhat(5, 3));
danhSach.add(new HinhVuong(4));
danhSach.add(new HinhChuNhat(6, 2));

// Duyệt danh sách và in thông tin riêng của HinhChuNhat
for (TuGiac hinh : danhSach) {
    if (hinh instanceof HinhChuNhat) {
        // Ép kiểu về HinhChuNhat
        HinhChuNhat hcn = (HinhChuNhat) hinh;
        hcn.inTTCDCR(); // Gọi phương thức riêng
    }
}

Các bước giải quyết

  1. Bước 1: Kiểm tra kiểu của phần tử có phải HinhChuNhat hay không
  2. Bước 2: Nếu kết quả là true thì ép kiểu phần tử đó về HinhChuNhat
  3. Bước 3: Gọi các hành vi mong muốn

PHẦN 4: INTERFACE - ĐỊNH NGHĨA BỘ HÀNH VI

4.1. Từ khóa interface

Định nghĩa

interface được sử dụng để định nghĩa bộ hành vi cho đối tượng. Một interface chỉ chứa các phương thức trừu tượng (trước Java 8) và các hằng số.

Cú pháp interface
interface TenInterface {
    // Khai báo phương thức trừu tượng
    void phuongThuc1();
    int phuongThuc2(int thamSo);
    
    // Khai báo hằng số (mặc định public static final)
    int HANG_SO = 100;
}

4.2. Từ khóa implements

implements là gì?

Dùng khi một lớp muốn thực thi (implement) bộ hành vi đã được định nghĩa trong interface.

Lưu ý: Một lớp chỉ kế thừa được 1 lớp cha, nhưng có thể implements nhiều interface.

Cú pháp implements
class TenLop implements Interface1, Interface2 {
    // Phải ghi đè toàn bộ phương thức của interface
    @Override
    public void phuongThuc1() {
        // Triển khai cụ thể
    }
}

4.3. Ví dụ: Hình Tròn implements interface

HinhTron.java
// Định nghĩa interface
interface BoHanhViCVDT {
    void tinhChuVi();
    void tinhDienTich();
}

// Lớp HinhTron implement interface
public class HinhTron implements BoHanhViCVDT {
    private double r;
    private double chuVi, dienTich;
    
    public HinhTron(double r) {
        this.r = r;
    }
    
    @Override
    public void tinhChuVi() {
        chuVi = Math.PI * 2 * r;
        System.out.println("Chu vi hình tròn: " + chuVi);
    }
    
    @Override
    public void tinhDienTich() {
        dienTich = Math.PI * r * r;
        System.out.println("Diện tích hình tròn: " + dienTich);
    }
}

Giải thích:

Khi HinhTron implements BoHanhViCVDT, lớp HinhTron bắt buộc phải ghi đè toàn bộ các phương thức trong interface.

PHẦN 5: KẾT HỢP ABSTRACT CLASS VÀ INTERFACE

5.1. Cách 1: Lớp con implement interface

Cach1.java - Lớp con implement
// Lớp cha trừu tượng (không định nghĩa chu vi, diện tích)
public abstract class TuGiac {
    protected double chuVi, dienTich;
    
    // Có thể có các phương thức chung khác
    public void inThongTin() {
        System.out.println("Chu vi: " + chuVi + ", Diện tích: " + dienTich);
    }
}

// Lớp con implement interface
public class HinhVuong extends TuGiac implements BoHanhViCVDT {
    private double doDaiCanh;
    
    public HinhVuong(double canh) {
        this.doDaiCanh = canh;
    }
    
    @Override
    public void tinhChuVi() {
        chuVi = doDaiCanh * 4;
    }
    
    @Override
    public void tinhDienTich() {
        dienTich = doDaiCanh * doDaiCanh;
    }
}

5.2. Cách 2: Lớp cha implement interface

Cach2.java - Lớp cha implement
// Lớp cha trừu tượng implement interface
public abstract class TuGiac implements BoHanhViCVDT {
    protected double chuVi, dienTich;
    
    // Lớp trừu tượng không cần ghi đè ngay
    // Cho phép lớp con ghi đè sau
}

// Lớp con kế thừa và ghi đè
public class HinhChuNhat extends TuGiac {
    private double chieuDai, chieuRong;
    
    public HinhChuNhat(double dai, double rong) {
        this.chieuDai = dai;
        this.chieuRong = rong;
    }
    
    @Override
    public void tinhChuVi() {
        chuVi = (chieuDai + chieuRong) * 2;
    }
    
    @Override
    public void tinhDienTich() {
        dienTich = chieuDai * chieuRong;
    }
}

Giải thích:

TuGiac là lớp trừu tượng implement interface, nên không bắt buộc ghi đè ngay các phương thức. Các lớp con sẽ ghi đè sau.

5.3. Kế thừa giữa các interface

InterfaceKeThua.java
// Interface 1: Bộ hành vi tính chu vi
interface BoHanhViCV {
    void tinhChuVi();
}

// Interface 2: Bộ hành vi tính diện tích
interface BoHanhViDT {
    void tinhDienTich();
}

// Interface kế thừa nhiều interface
interface BoHanhViCVDT extends BoHanhViCV, BoHanhViDT {
    // Kế thừa cả tinhChuVi() và tinhDienTich()
}

Quan trọng:

Một interface có thể kế thừa nhiều interface khác, không giống như class chỉ kế thừa được 1 class cha.

PHẦN 6: ĐA HÌNH VỚI INTERFACE

6.1. Quản lý đa hình qua Interface

DaHinhVoiInterface.java
import java.util.ArrayList;
import java.util.List;

// Interface chung
interface HinhHoc {
    void tinhChuVi();
    void tinhDienTich();
    void inThongTin();
}

// Các lớp implement
class HinhVuong implements HinhHoc {
    private double canh;
    private double chuVi, dienTich;
    
    public HinhVuong(double canh) { this.canh = canh; }
    
    @Override public void tinhChuVi() { chuVi = canh * 4; }
    @Override public void tinhDienTich() { dienTich = canh * canh; }
    @Override public void inThongTin() {
        System.out.println("Hình vuông cạnh " + canh + ": CV=" + chuVi + ", DT=" + dienTich);
    }
}

class HinhTron implements HinhHoc {
    private double banKinh;
    private double chuVi, dienTich;
    
    public HinhTron(double r) { this.banKinh = r; }
    
    @Override public void tinhChuVi() { chuVi = 2 * Math.PI * banKinh; }
    @Override public void tinhDienTich() { dienTich = Math.PI * banKinh * banKinh; }
    @Override public void inThongTin() {
        System.out.println("Hình tròn bán kính " + banKinh + ": CV=" + chuVi + ", DT=" + dienTich);
    }
}

public class Main {
    public static void main(String[] args) {
        List<HinhHoc> danhSach = new ArrayList<>();
        
        // Thêm các hình khác nhau vào danh sách
        danhSach.add(new HinhVuong(5));
        danhSach.add(new HinhTron(3));
        danhSach.add(new HinhVuong(7));
        danhSach.add(new HinhTron(4));
        
        // Tính toán và in thông tin tất cả hình
        for (HinhHoc hinh : danhSach) {
            hinh.tinhChuVi(); // ĐA HÌNH
            hinh.tinhDienTich(); // ĐA HÌNH
   &�nbsp;       hinh.inThongTin(); // ĐA HÌNH
        }
    }
}

Ưu điểm của đa hình với Interface

  • Linh hoạt: Các lớp không cần có quan hệ kế thừa vẫn có thể có hành vi chung
  • Tách biệt: Tách giao diện (interface) khỏi cài đặt (implementation)
  • Dễ mở rộng: Thêm lớp mới implement interface mà không ảnh hưởng code cũ
  • Đa kế thừa: Một lớp có thể implement nhiều interface

6.2. So sánh Abstract Class và Interface

Tiêu chí Abstract Class Interface
Kế thừa Chỉ kế thừa 1 lớp Implement nhiều interface
Phương thức Có thể có phương thức cụ thể Chỉ phương thức trừu tượng (trước Java 8)
Biến Có thể có biến instance Chỉ hằng số (public static final)
Constructor Có constructor Không có constructor
Access modifier Có thể private, protected Mặc định public
Khi nào dùng Các lớp có quan hệ "IS-A" Các lớp có hành vi chung "CAN-DO"

Hướng dẫn làm bài tập

Bài tập 1: Quản lý động vật

BaiTap1.java - Động vật
// 1. Tạo interface DongVat với phương thức tiengKeu()
interface DongVat {
    void tiengKeu();
}

// 2. Tạo các lớp ConCho, ConMeo, ConGa implement DongVat
class ConCho implements DongVat {
    @Override
    public void tiengKeu() {

    }
}

class ConMeo implements DongVat {
    @Override
    public void tiengKeu() {
        System.out.println("Meo meo!");
    }
}

class ConGa implements DongVat {
    @Override
    public void tiengKeu() {
        System.out.println("Ò ó o!");
    }
}

// 3. Viết chương trình chính tạo danh sách động vật
public class QuanLyDongVat {

Bài tập 2: Tính lương nhân viên

BaiTap2.java - Tính lương
// Abstract class NhanVien
abstract class NhanVien {
    String ten;
    int heSoLuong;
    
    // Phương thức trừu tượng tính lương
    abstract double tinhLuong();

// Lớp NVVanPhong

    @Override
    double tinhLuong() {
        return heSoLuong * 1490000 + soNgayLam * 200000;
    }
}

// Lớp NVSanXuat
class NVSanXuat extends NhanVien {
    int soSanPham;
    
    @Override

        return heSoLuong * 1490000 + soSanPham * 50000;

TỔNG KẾT - 5 điều cần nhớ về Đa hình:

  1. Đa hình = Nhiều hình thái: Cùng tên phương thức, nhiều cách triển khai
  2. Ghi đè (Override) là cốt lõi: Lớp con ghi đè phương thức lớp cha
  3. Quản lý tập trung: Dùng lớp cha/interface để quản lý nhiều đối tượng
  4. instanceof và ép kiểu: Kiểm tra và xử lý đặc thù từng loại đối tượng
  5. Interface linh hoạt hơn kế thừa: Cho phép đa hình giữa các lớp không cùng hệ thống kế thừa
