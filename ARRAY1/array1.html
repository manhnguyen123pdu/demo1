11.1.1 Định nghĩa OOP

Lập trình hướng đối tượng (Object-Oriented Programming - OOP) là phương pháp lập trình trong đó chương trình được xây dựng dựa trên các đối tượng. Mỗi đối tượng đại diện cho một thực thể trong thế giới thực, có trạng thái (dữ liệu) và hành vi (xử lý).

Ví dụ minh họa: Đối tượng Sinh viên

Student.java - Ví dụ về Class 
class Student {
    // Fields (Thuộc tính)
    String name;
    int age;
    double grade;
    
    // Methods (Phương thức)
    void study() {
        System.out.println(name + " đang học");
    }
    
    void takeExam() {
        System.out.println(name + " đang thi");
    }
}

Phân tích:

  • Lớp Student mô tả sinh viên
  • Dữ liệu (name, age, grade) và xử lý (study, takeExam) được đặt chung trong một lớp
  • Đúng tư duy OOP: đóng gói dữ liệu và hành vi liên quan

11.1.2 So sánh OOP vs Lập trình thủ tục

Tiêu chí Lập trình thủ tục Lập trình hướng đối tượng
Tổ chức Theo hàm Theo đối tượng
Dữ liệu Tách rời với hàm xử lý Đóng gói trong đối tượng
Bảo trì Khó khi code lớn Dễ dàng hơn
Dự án lớn Không phù hợp Rất phù hợp
Tái sử dụng Hạn chế Cao (qua kế thừa)
Bảo mật Dữ liệu dễ bị truy cập trực tiếp Dữ liệu được bảo vệ

Vấn đề với lập trình thủ tục:

Code lớn dùng thủ tục → rất dễ loạn do dữ liệu và xử lý tách rời. OOP chia nhỏ thành object → mỗi object tự quản lý → dễ quản lý, bảo trì.

11.1.3 Vì sao Java dùng OOP?

Java là ngôn ngữ hướng đối tượng gần như hoàn toàn (trừ các kiểu dữ liệu nguyên thủy).

Lý do Java chọn OOP:

  • Java được thiết kế cho ứng dụng lớn, phức tạp
  • Hỗ trợ làm việc nhóm (nhiều lập trình viên)
  • Dễ bảo trì, nâng cấp trong thời gian dài
  • Phù hợp với các hệ thống doanh nghiệp

11.2.1 Khái niệm Class

Class (lớp) là bản thiết kế (khuôn mẫu, template) dùng để tạo ra các đối tượng. Nó định nghĩa các thuộc tính và phương thức mà các đối tượng thuộc lớp đó sẽ có.

Cú pháp cơ bản của Class 
class ClassName {
    // fields - thuộc tính
    // methods - phương thức
    // constructors - hàm khởi tạo
}

Ví dụ về Class:

class Student {
    String name;
    int age;
    double gpa;
}

Phân tích: Class chỉ là định nghĩa, chưa có dữ liệu thật. Giống như bản vẽ nhà - chưa phải ngôi nhà thật.

11.2.2 Khái niệm Object

Object (đối tượng) là một thể hiện cụ thể của class, có dữ liệu thật và chiếm bộ nhớ.

Cú pháp tạo Object 
ClassName objectName = new ClassName();

Giải thích:

  • new: toán tử cấp phát bộ nhớ cho đối tượng
  • ClassName(): gọi constructor (hàm khởi tạo)
  • objectName: biến tham chiếu đến đối tượng

11.2.3 Mối quan hệ Class - Object

Car.java - Ví dụ về một Class tạo nhiều Object 
class Car {
    String brand;
    String color;
    int speed;
    
    void accelerate() {
        speed += 10;
    }
}

// Tạo nhiều object từ class Car
Car car1 = new Car();
car1.brand = "Toyota";
car1.color = "Red";

Car car2 = new Car();
car2.brand = "Honda";
car2.color = "Blue";

Phân tích:

  • Class Car là bản thiết kế chung
  • car1 và car2 là các xe cụ thể
  • Chung cấu trúc (đều có brand, color, speed, accelerate())
  • Dữ liệu độc lập (brand, color khác nhau)

CẤU TRÚC CLASS TRONG JAVA

11.3.1 Thuộc tính (Fields/Attributes)

Field là biến dùng để lưu trữ trạng thái của đối tượng.

Cú pháp và ví dụ về Fields 
class Person {
    // Fields - Thuộc tính
    String name;  // Không có giá trị mặc định
    int age = 18;  // Giá trị mặc định
    String address;
    final String COUNTRY = "Vietnam";  // Hằng số
    
    // Static field - thuộc về Class
    static int count = 0;
}

Đặc điểm của Fields:

  • Mỗi object có bản sao riêng của các fields (trừ static fields)
  • Có thể khởi tạo giá trị mặc định
  • final cho hằng số - không thay đổi được
  • static thuộc về Class, không thuộc về Object

11.3.2 Phương thức (Methods)

Method mô tả hành vi của đối tượng, là các thao tác mà đối tượng có thể thực hiện.

Calculator.java - Ví dụ về Methods 
class Calculator {
    // Method không trả về giá trị
    void displayHello() {
        System.out.println("Hello World!");
    }
    
    // Method trả về giá trị
    int add(int a, int b) {
        return a + b;
    }
    
    // Method với tham số
    void displayInfo(String name, int age) {
        System.out.println("Name: " + name + ", Age: " + age);
    }
    
    // Static method
    static int multiply(int a, int b) {
        return a * b;
    }
}

Phân tích:

  • Method định nghĩa "hành vi" của object
  • Có thể có hoặc không có tham số
  • Có thể trả về giá trị hoặc không (void)
  • Có thể là static (thuộc về Class) hoặc non-static (thuộc về Object)

11.3.3 Constructor (Hàm khởi tạo)

Constructor là phương thức đặc biệt, được gọi tự động khi tạo đối tượng, dùng để khởi tạo giá trị cho các thuộc tính.

Đặc điểm của Constructor 
class ClassName {
    // Constructor
    ClassName(parameters) {
        // khởi tạo giá trị
    }
}

Đặc điểm của Constructor:

  • Tên trùng với tên class
  • Không có kiểu trả về (không cần void)
  • Có thể có nhiều constructor (overloading)
  • Được gọi tự động khi tạo object với từ khóa new

11.4.1 Constructor mặc định (Default Constructor)

Constructor mặc định 
class Student {
    String name;
    int age;
    
    // Constructor mặc định
    Student() {
        name = "Unknown";
        age = 0;
    }
}

LƯU Ý QUAN TRỌNG:

  • Nếu không viết constructor nào, Java tự động tạo constructor mặc định (không tham số, rỗng)
  • Nếu đã viết constructor có tham số, Java KHÔNG tự tạo constructor mặc định nữa

11.4.2 Constructor có tham số

Constructor có tham số 
class Student {
    String name;
    int age;
    
    // Constructor có tham số
    Student(String studentName, int studentAge) {
        name = studentName;
        age = studentAge;
    }
}

// Sử dụng
Student s1 = new Student("Nguyen Van A", 20);

11.4.3 Overloading Constructor

Overloading Constructor 
class Student {
    String name;
    int age;
    String major;
    
    // Constructor 1: Không tham số
    Student() {
        name = "Unknown";
        age = 0;
        major = "Undeclared";
    }
    
    // Constructor 2: 2 tham số
    Student(String name, int age) {
        this.name = name;
        this.age = age;
        major = "Undeclared";
    }
    
    // Constructor 3: 3 tham số
    Student(String name, int age, String major) {
        this.name = name;
        this.age = age;
        this.major = major;
    }
}

// Sử dụng
Student s1 = new Student();  // Dùng constructor 1
Student s2 = new Student("Nam", 20);  // Dùng constructor 2
Student s3 = new Student("Lan", 21, "IT");  // Dùng constructor 3

11.4.4 Từ khóa this

Công dụng của từ khóa this 
// 1. Phân biệt thuộc tính và tham số khi trùng tên
class Student {
    String name;
    int age;
    
    Student(String name, int age) {
        this.name = name;  // this.name là thuộc tính, name là tham số
        this.age = age;    // this.age là thuộc tính, age là tham số
    }
}

// 2. Gọi constructor khác trong cùng lớp
class Student {
    String name;
    int age;
    String major;
    
    Student() {
        this("Unknown", 0, "Undeclared");  // Gọi constructor 3 tham số
    }
    
    Student(String name, int age, String major) {
        this.name = name;
        this.age = age;
        this.major = major;
    }
}

// 3. Trả về đối tượng hiện tại (method chaining)
class Student {
    String name;
    
    Student setName(String name) {
        this.name = name;
        return this;  // Trả về chính đối tượng này
    }
}

11.5.2 Truy cập fields và methods

Ví dụ đầy đủ về tạo và sử dụng Object 
class Student {
    String name;
    int age;
    
    Student(String name, int age) {
        this.name = name;
        this.age = age;
    }
    
    void displayInfo() {
        System.out.println("Name: " + name);
        System.out.println("Age: " + age);
    }
    
    void study(String subject) {
        System.out.println(name + " is studying " + subject);
    }
    
    int getAgeAfterYears(int years) {
        return age + years;
    }
}

public class Main {
    public static void main(String[] args) {
        // Tạo object
        Student student = new Student("Nguyen Van A", 20);
        
        // Truy cập field
        System.out.println("Student name: " + student.name);
        
        // Thay đổi field
        student.age = 21;
        
        // Gọi methods
        student.displayInfo();
        student.study("Java Programming");
        
        int futureAge = student.getAgeAfterYears(5);
        System.out.println("Age after 5 years: " + futureAge);
    }
}

11.5.3 Vòng đời của object

Object Lifecycle - Vòng đời của đối tượng 
public class ObjectLifecycle {
    public static void main(String[] args) {
        // Giai đoạn 1: Tạo object
        Student s1 = new Student("A", 20);
        
        // Giai đoạn 2: Sử dụng
        s1.displayInfo();
        
        // Tạo thêm tham chiếu
        Student s2 = s1;
        
        // Giai đoạn 3: Không còn tham chiếu
        s1 = null;  // Object vẫn được tham chiếu bởi s2
        s2 = null;  // Giờ object không được tham chiếu nữa
        
        // Giai đoạn 4: Garbage Collector sẽ thu hồi bộ nhớ
        // (tự động, không cần lập trình viên can thiệp)
    }
}

Garbage Collector (GC):

  • Tự động thu hồi bộ nhớ của các object không còn được sử dụng
  • Lập trình viên không cần giải phóng bộ nhớ thủ công
  • GC chạy không đồng bộ, không thể dự đoán thời điểm chính xác

Những điểm chính cần nhớ

Tóm tắt kiến thức:

  1. OOP là lập trình dựa trên đối tượng, gom dữ liệu và xử lý
  2. Class là bản thiết kế, Object là thể hiện cụ thể
  3. Constructor dùng để khởi tạo object
  4. this tham chiếu đến object hiện tại
  5. Object được tạo bằng new, sử dụng qua dấu chấm .

Lưu ý quan trọng:

  • Một class tạo nhiều object, mỗi object có dữ liệu riêng
  • Nếu không viết constructor, Java tự tạo constructor mặc định
  • Nếu đã viết constructor có tham số, phải tự viết constructor không tham số nếu cần
  • Object trở thành rác khi không còn tham chiếu, GC sẽ tự động thu hồi
