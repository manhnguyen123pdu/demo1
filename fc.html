PHẦN 1: PHƯƠNG THỨC TRONG JAVA LÀ GÌ ?

1.1. phương thức (Method) là gì? - Ví dụ đầu bếp

Không có phương thức : Mỗi lần nấu mì, bạn phải làm từ A-Z: đun nước, cho mì, nêm gia vị...

Có phương thức : Bạn tạo ra "công thức nấu mì" (phương thức ). Khi cần, chỉ cần gọi "nấu mì" là xong!

Trong lập trình:

  • phương thức là một khối lệnh thực hiện một công việc cụ thể
  • Được đặt tên để gọi lại nhiều lần

1.2. Tại sao cần phương thức ?

Ví dụ: Tính tổng 2 số
// KHÔNG dùng phương thức :
int a = 5, b = 3;
int tong1 = a + b; // Lần 1
System.out.println("Tổng 1: " + tong1);

int c = 10, d = 7;
int tong2 = c + d; // Lần 2 - phải viết lại phép cộng
System.out.println("Tổng 2: " + tong2);

// DÙNG phương thức :
public static int tinhTong(int x, int y) {
    return x + y;
}

// Sử dụng:
int tong1 = tinhTong(5, 3); // Gọi phương thức
int tong2 = tinhTong(10, 7); // Gọi lại phương thức

Lợi ích của phương thức :

  • Tái sử dụng: Viết 1 lần, dùng nhiều lần
  • Dễ đọc: tinhTong(5, 3) dễ hiểu hơn 5 + 3
  • Dễ sửa: Sửa 1 chỗ, áp dụng mọi nơi
  • Chia nhỏ công việc: Chương trình lớn thành các phần nhỏ

PHẦN 5: THAM SỐ ĐẶC BIỆT

public class Varargs {

5.2. Truyền tham số: Giá trị vs Tham chiếu

TruyenThamSo.java
public class TruyenThamSo {
    // Truyền giá trị (primitive types)
    public static void tangGiaTri(int x) {
        x = x + 10;
    }
    
    // Truyền tham chiếu (reference types)
    public static void tangPhanTu(int[] mang) {
        if (mang.length > 0) {
            mang[0] = mang[0] + 10;
        }
    }
}

PHẦN 7: ĐỆ QUY (RECURSION)

6.1. Khái niệm

phương thức gọi lại chính nó

DeQuy.java - Tính giai thừa
public class DeQuy {
    // Tính giai thừa: n! = n * (n-1)!
    public static long giaiThua(int n) {
        // Điều kiện dừng
        if (n <= 1) {
            return 1;
        }
        // Bước đệ quy
        return n * giaiThua(n - 1);
    }
}

6.2. Cấu trúc phương thức đệ quy

Mọi phương thức đệ quy đều có:

  • Phần cơ sở: Điều kiện dừng
  • Phần đệ quy: Gọi lại chính phương thức

Lưu ý quan trọng

  • Phải có điều kiện dừng - nếu không sẽ gây StackOverflow
  • Điều kiện dừng phải đạt được - không được vô hạn
  • Tránh đệ quy sâu - có thể gây chậm hoặc lỗi bộ nhớ

Bài tập 2: phương thức với mảng

BaiTap2.java
public class BaiTap2 {
    // 1. Viết phương thức tìm giá trị lớn nhất trong mảng
    public static int timMax(int[] mang) {
        if (mang == null || mang.length == 0) {
            return -1;
        }
        int max = mang[0];
        for (int i = 1; i < mang.length; i++) {
            if (mang[i] > max) {
                max = mang[i];
            }
        }
        return max;
    }
}

PHẦN 10: THỰC HÀNH TỔNG HỢP

Project nhỏ: Quản lý sinh viên

QuanLySinhVien.java - Project tổng hợp
import java.util.ArrayList;
import java.util.List;

public class QuanLySinhVien {
    private List<SinhVien> danhSach = new ArrayList<>();
    
    // Lớp SinhVien nhúng bên trong
    class SinhVien {
        String maSV;
        String ten;
        double diem;
    }
    
    // 1. Thêm sinh viên
    public void themSinhVien(String ma, String ten, double diem) {
        // Kiểm tra mã không trùng
        for (SinhVien sv : danhSach) {
            if (sv.maSV.equals(ma)) {
                System.out.println("Mã sinh viên đã tồn tại!");
                return;
            }
        }
        
        SinhVien sv = new SinhVien();
        sv.maSV = ma;
        sv.ten = ten;
        sv.diem = diem;
        danhSach.add(sv);
        System.out.println("Thêm thành công!");
    }
}

TỔNG KẾT - 10 điều cần nhớ về phương thức trong Java:

  1. Mỗi phương thức làm một việc: phương thức càng nhỏ, càng dễ hiểu, dễ sửa
  2. Đặt tên có ý nghĩa: tinhDiemTrungBinh() tốt hơn tinh()
  3. Hạn chế tham số: Tối ưu 0-3 tham số, dùng đối tượng nếu nhiều
  4. Tránh side effect: phương thức không nên thay đổi bất ngờ
  5. Kiểm tra đầu vào: Luôn kiểm tra null và giá trị hợp lệ
  6. Xử lý lỗi: Dùng return đặc biệt hoặc ném ngoại lệ
  7. Comment rõ ràng: Giải thích phương thức phức tạp, tham số đặc biệt
  8. Tránh đệ quy sâu: Có thể gây StackOverflow
  9. Tái sử dụng: Nếu code lặp 3 lần, hãy tạo phương thức
  10. Test kỹ: Test với các trường hợp biên
