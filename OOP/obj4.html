TỔNG QUAN VỀ TÍNH KẾ THỪA

A. TÍNH KẾ THỪA TRONG LẬP TRÌNH HƯỚNG ĐỐI TƯỢNG

Kế thừa (Inheritance) là một trong bốn tính chất cơ bản của lập trình hướng đối tượng (OOP), cho phép một lớp (class) có thể thừa hưởng các thuộc tính và phương thức từ một lớp khác.

Bốn tính chất cơ bản của OOP:

  • Tính đóng gói (Encapsulation) - Ẩn dữ liệu và chỉ hiển thị cần thiết
  • Tính kế thừa (Inheritance) - Tái sử dụng code từ lớp cha
  • Tính đa hình (Polymorphism) - Nhiều hình thái khác nhau
  • Tính trừu tượng (Abstraction) - Ẩn chi tiết phức tạp
Sơ đồ kế thừa cơ bản:
Lớp Cha (Parent Class)
ConNguoi
↓ kế thừa (extends)
Lớp Con 1 (Child Class)
HocSinh
Lớp Con 2 (Child Class)
NhanVien

Đặc điểm quan trọng của kế thừa trong Java:

  • Java chỉ hỗ trợ đơn kế thừa (mỗi lớp chỉ kế thừa từ một lớp cha)
  • Sử dụng từ khóa extends để thiết lập quan hệ kế thừa
  • Giúp tái sử dụng code, giảm trùng lặp
  • Tất cả lớp trong Java đều kế thừa từ lớp Object

1. Định nghĩa kế thừa

Kế thừa là cơ chế cho phép một lớp tiếp nhận (thừa hưởng) các thuộc tính và phương thức của một lớp khác.

Cú pháp cơ bản
class LopCon extends LopCha {
    // Các thuộc tính và phương thức riêng của lớp con
}

1.1. Ví dụ cơ bản:

InheritanceBasic.java
// Lớp cha
class ConNguoi {
    String hoTen;
    int tuoi;

    void anUong() {
        System.out.println("Đang ăn uống...");
    }
}

// Lớp con kế thừa từ lớp cha
class HocSinh extends ConNguoi {
    String maHS;
    String lop;

    void hocBai() {
        System.out.println("Đang học bài...");
    }
}

public class InheritanceBasic {
    public static void main(String[] args) {
        HocSinh hs = new HocSinh();
        hs.hoTen = "Nguyễn Văn A"; // Thuộc tính kế thừa từ ConNguoi
        hs.tuoi = 10;
        hs.maHS = "HS001";

        hs.anUong(); // Phương thức kế thừa từ ConNguoi
        hs.hocBai(); // Phương thức riêng của HocSinh
    }
}

1.2. Thuật ngữ quan trọng

Thuật ngữ Định nghĩa Ví dụ
Lớp cha (Superclass/Parent class) Lớp được kế thừa, chứa các thành phần chung ConNguoi
Lớp con (Subclass/Child class) Lớp kế thừa từ lớp cha, có thể thêm thành phần riêng HocSinh, NhanVien
IS-A relationship Mối quan hệ "là một" giữa lớp con và lớp cha HocSinh LÀ MỘT ConNguoi
Từ khóa extends Từ khóa để thiết lập quan hệ kế thừa class HocSinh extends ConNguoi
Từ khóa super Tham chiếu đến đối tượng lớp cha super.nhapTT(...)

Quan hệ IS-A trong thực tế:

  • Học sinh LÀ MỘT con người
  • Nhân viên LÀ MỘT con người
  • Giáo viên LÀ MỘT nhân viên
  • Ô tô LÀ MỘT phương tiện giao thông
  • Chó LÀ MỘT động vật

B. NGUYÊN TẮC XÁC ĐỊNH KẾ THỪA

Khi hai hay nhiều đối tượng có chung đặc điểm và hành vi, ta tạo ra một lớp cha chứa các thành phần chung này.

Quy trình phân tích để xác định kế thừa:

  1. Xác định các đối tượng trong hệ thống
  2. Tìm điểm chung về thuộc tính và phương thức giữa các đối tượng
  3. Trừu tượng hóa thành lớp cha chứa các thành phần chung
  4. Thiết lập quan hệ kế thừa giữa lớp cha và các lớp con
  5. Bổ sung thành phần riêng cho từng lớp con

Ví dụ phân tích đối tượng:

Trong hệ thống quản lý trường học:

  • Điểm chung (đưa vào lớp ConNguoi):
    • Họ tên, tuổi, giới tính, địa chỉ
    • Phương thức: ăn uống, di chuyển
  • Điểm riêng của Học sinh:
    • Mã HS, lớp, điểm toán, điểm văn, điểm anh
    • Phương thức: học bài, tính điểm trung bình
  • Điểm riêng của Nhân viên:
    • Mã NV, chức vụ, lương
    • Phương thức: làm việc, viết báo cáo

3.2. Xác định lớp cha ConNguoi

ConNguoi.java
// Lớp cha ConNguoi chứa các thuộc tính và phương thức chung
class ConNguoi {
    // Thuộc tính
    String hoTen;
    int tuoi;
    String gioiTinh;
    String diaChi;
    
    // Phương thức
    void nhapTT(String hoTen, String gioiTinh,
                    String diaChi, int tuoi) {
        this.hoTen = hoTen;
        this.gioiTinh = gioiTinh;
        this.diaChi = diaChi;
        this.tuoi = tuoi;
    }
    
    void anUong() {
        System.out.println("Đang ăn uống...");
    }
    
    void diChuyen() {
        System.out.println("Đang di chuyển...");
    }
    
    void inTT() {
        System.out.println("Họ tên: " + hoTen);
        System.out.println("Tuổi: " + tuoi);
        System.out.println("Giới tính: " + gioiTinh);
        System.out.println("Địa chỉ: " + diaChi);
    }
}

3.3. Triển khai lớp con HocSinh

HocSinh.java
class HocSinh extends ConNguoi {
    // Thuộc tính riêng
    String maHS;
    String lop;
    int diemToan;
    int diemVan;
    int diemAnh;
    
    // Nạp chồng phương thức nhapTT
    void nhapTT(String hoTen, String gioiTinh, String diaChi, int tuoi,
                    String maHS, String lop, int diemToan, int diemVan, int diemAnh) {
        // Gọi phương thức của lớp cha
        super.nhapTT(hoTen, gioiTinh, diaChi, tuoi);
        
        // Nhập thông tin riêng
        this.maHS = maHS;
        this.lop = lop;
        this.diemToan = diemToan;
        this.diemVan = diemVan;
        this.diemAnh = diemAnh;
    }
    
    // Ghi đè phương thức inTT
    @Override
    void inTT() {
        // Gọi phương thức inTT của lớp cha
        super.inTT();
        
        // In thông tin riêng
        System.out.println("Mã HS: " + maHS);
        System.out.println("Lớp: " + lop);
        System.out.println("Điểm Toán: " + diemToan);
        System.out.println("Điểm Văn: " + diemVan);
        System.out.println("Điểm Anh: " + diemAnh);
        System.out.println("Điểm TB: " + tinhDiemTB());
    }
    
    // Phương thức riêng
    void hocBai() {
        System.out.println("Đang học bài...");
    }
    
    double tinhDiemTB() {
        return (diemToan + diemVan + diemAnh) / 3.0;
    }
}

D. CÁC VẤN ĐỀ KỸ THUẬT TRONG KẾ THỪA

4. Nạp chồng phương thức (Overloading)

Khái niệm: Là việc tạo nhiều phương thức cùng tên nhưng khác tham số. Xảy ra trong cùng một lớp hoặc giữa lớp cha và lớp con.

Ví dụ nạp chồng
// Trong lớp ConNguoi
void nhapTT(String hoTen, String gioiTinh, String diaChi, int tuoi)

// Trong lớp HocSinh (nạp chồng)
void nhapTT(String hoTen, String gioiTinh, String diaChi, int tuoi,
                    String maHS, String lop, int diemToan, int diemVan, int diemAnh)

5. Ghi đè phương thức (Overriding)

Khái niệm: Là việc lớp con cung cấp triển khai mới cho phương thức đã có ở lớp cha.

5.2. Hai hình thức ghi đè:
a) Ghi đè hoàn toàn:
@Override
void phuongThuc() {
    // Triển khai hoàn toàn mới
}
b) Ghi đè không hoàn toàn:
@Override
void inTT() {
    super.inTT(); // Gọi phương thức của lớp cha
    // Bổ sung thêm xử lý
    System.out.println("Thông tin bổ sung...");
}

5.3. So sánh Overloading vs Overriding:

Tiêu chí Overloading Overriding
Quan hệ Cùng lớp hoặc cha-con Chỉ giữa lớp cha và con
Tham số Phải khác nhau Phải giống nhau
Kiểu trả về Có thể khác nhau Phải giống nhau
Từ khóa Không cần @Override (khuyến khích)
Mục đích Xử lý các tình huống với đầu vào khác nhau Cung cấp triển khai mới cho phương thức kế thừa

E. LỚP QUẢN LÝ QUANLY

QuanLy.java
class QuanLy {
    // Sử dụng mảng thay cho ArrayList
    private NhanVien[] dsNhanVien = new NhanVien[100];
    private HocSinh[] dsHocSinh = new HocSinh[100];
    
    // Biến đếm số lượng thực tế
    private int soNV = 0;
    private int soHS = 0;
    
    // Thêm nhân viên
    void themNhanVien(NhanVien nv) {
        if (soNV < 100) {
            dsNhanVien[soNV] = nv;
            soNV++;
            System.out.println("Đã thêm nhân viên: " + nv.hoTen);
        } else {
            System.out.println("Danh sách nhân viên đã đầy!");
        }
    }
    
    // Thêm học sinh
    void themHocSinh(HocSinh hs) {
        if (soHS < 100) {
            dsHocSinh[soHS] = hs;
            soHS++;
            System.out.println("Đã thêm học sinh: " + hs.hoTen);
        } else {
            System.out.println("Danh sách học sinh đã đầy!");
        }
    }
    
    // Tìm kiếm nhân viên
    NhanVien timKiemNhanVien(String maNV) {
        for (int i = 0; i < soNV; i++) {
            if (dsNhanVien[i].maNV.equals(maNV)) {
                return dsNhanVien[i];
            }
        }
        return null;
    }
    
    // Tìm kiếm học sinh
    HocSinh timKiemHocSinh(String maHS) {
        for (int i = 0; i < soHS; i++) {
            if (dsHocSinh[i].maHS.equals(maHS)) {
                return dsHocSinh[i];
            }
        }
        return null;
    }
    
    // Hiển thị danh sách nhân viên
    void hienThiDSNV() {
        System.out.println("=== DANH SÁCH NHÂN VIÊN ===");
        for (int i = 0; i < soNV; i++) {
            dsNhanVien[i].inTT();
            System.out.println("-------------------");
        }
    }
    
    // Hiển thị danh sách học sinh
    void hienThiDSHS() {
        System.out.println("=== DANH SÁCH HỌC SINH ===");
        for (int i = 0; i < soHS; i++) {
            dsHocSinh[i].inTT();
            System.out.println("-------------------");
        }
    }
}

F. LỢI ÍCH CỦA KẾ THỪA

1. Tái sử dụng code

3. Mở rộng dễ dàng

9. Lưu ý quan trọng

  • Java không hỗ trợ đa kế thừa với lớp (một lớp chỉ extends một lớp cha)
  • Tất cả lớp trong Java đều kế thừa từ lớp Object
  • Sử dụng super để gọi phương thức/constructor của lớp cha
  • Có thể sử dụng final để ngăn kế thừa hoặc ghi đè
  • Tránh kế thừa quá sâu (deep inheritance hierarchy) - khó bảo trì

  • Sử dụng @Override khi ghi đè phương thức để dễ kiểm tra lỗi
  • Ưu tiên composition over inheritance khi phù hợp
  • Thiết kế lớp cha với phương thức trừu tượng nếu cần
  • Tuân thủ nguyên tắc Liskov Substitution Principle
  • Đặt tên lớp, phương thức rõ ràng, thể hiện mối quan hệ IS-A

TỪ KHÓA FINAL TRONG JAVA

14.4 TỔNG QUAN VỀ TỪ KHÓA FINAL

Từ khóa final trong Java là một công cụ mạnh mẽ để kiểm soát tính linh hoạt của mã nguồn, đảm bảo tính ổn định và an toàn của chương trình.

1. FINAL CLASS (LỚP KHÔNG THỂ KẾ THỪA)

Khái niệm: Khi một lớp được khai báo với từ khóa final, lớp đó không thể được kế thừa bởi bất kỳ lớp nào khác.

Ví dụ final class
// Lớp final - không thể kế thừa
final class HinhTron {
    double banKinh;
    
    HinhTron(double r) {
        this.banKinh = r;
    }
    
    double tinhDienTich() {
        return Math.PI * banKinh * banKinh;
    }
}

// // ...
// }

2. FINAL METHOD (PHƯƠNG THỨC KHÔNG THỂ GHI ĐÈ)

Khái niệm: Khi một phương thức được khai báo với từ khóa final, phương thức đó không thể bị ghi đè (override) bởi các lớp con.

Ví dụ final method
class ConNguoi {
    String hoTen;
    
    // Constructor
    ConNguoi(String hoTen) {
        this.hoTen = hoTen;
    }
    
    // Phương thức final - không thể ghi đè
    final void gioiThieu() {
        System.out.println("Tôi là: " + hoTen);
        System.out.println("Tôi là con người!");
    }
    
    // Phương thức thông thường - có thể ghi đè
    void anUong() {
        System.out.println("Đang ăn uống...");
    }
}
