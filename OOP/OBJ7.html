PHẦN 1: TỔNG QUAN VỀ TÍNH TRỪU TƯỢNG

1.1. LÝ THUYẾT VỀ TÍNH TRỪU TƯỢNG

Khái niệm

Tính trừu tượng (Abstraction) là một trong 4 tính chất cốt lõi của Lập trình hướng đối tượng (OOP), cùng với:

  1. Đóng gói (Encapsulation)
  2. Kế thừa (Inheritance)
  3. Đa hình (Polymorphism)

Định nghĩa

Tính trừu tượng là quá trình ẩn đi những chi tiết phức tạp của hệ thống và chỉ hiển thị những tính năng thiết yếu cho người dùng.

1.2. MỤC ĐÍCH CỦA TÍNH TRỪU TƯỢNG

Tại sao cần tính trừu tượng?

Mục đích Giải thích
Giảm độ phức tạp Che giấu chi tiết triển khai phức tạp
Tăng tính bảo mật Ngăn truy cập trực tiếp vào dữ liệu
Tái sử dụng code Dễ dàng sử dụng lại ở nhiều nơi
Dễ bảo trì Thay đổi bên trong không ảnh hưởng bên ngoài
Tăng tính linh hoạt Có thể thay đổi triển khai mà không ảnh hưởng đến code sử dụng

PHẦN 2: LỚP TRỪU TƯỢNG (ABSTRACT CLASS)

2.1. LÝ THUYẾT

Khái niệm

Lớp trừu tượng là lớp không thể tạo đối tượng trực tiếp, chỉ có thể được kế thừa bởi các lớp con.

Đặc điểm:

  • Chứa ít nhất một phương thức trừu tượng
  • Có thể chứa phương thức thông thường
  • Có thể chứa thuộc tính và constructor
  • Được sử dụng làm khuôn mẫu cho các lớp con

2.2. MỤC ĐÍCH

Mục đích Giải thích
Tạo template/khuôn mẫu Định nghĩa cấu trúc chung
Bắt buộc triển khai Ép lớp con phải triển khai phương thức trừu tượng
Chia sẻ code Phương thức thông thường có thể dùng chung
Thiết kế có tổ chức Phân cấp rõ ràng trong hệ thống

2.3. CÚ PHÁP

// Khai báo lớp trừu tượng
[access-modifier] abstract class TenLop {
    // Thuộc tính
    [access-modifier] kieuDuLieu tenThuocTinh;
    
    // Constructor
    [access-modifier] TenLop(thamSo) {
        // Khởi tạo
    }
    
    // Phương thức thông thường
    [access-modifier] kieuTraVe tenPhuongThuc() {
        // Thân phương thức
    }
    
    // Phương thức trừu tượng (KHÔNG có thân)
    [access-modifier] abstract kieuTraVe tenPhuongThucTruuTuong();
}

2.4. GIẢI THÍCH CÚ PHÁP

  • abstract class: Từ khóa bắt buộc để khai báo lớp trừu tượng
  • abstract method: Phương thức không có thân, kết thúc bằng dấu ;
  • access-modifier: public, protected, hoặc mặc định (package-private)
  • Lớp con phải triển khai tất cả phương thức trừu tượng (trừ khi nó cũng là lớp trừu tượng)

2.5. VÍ DỤ CHI TIẾT

LÝ THUYẾT ÁP DỤNG

Hình học là khái niệm trừu tượng. Chúng ta biết hình có diện tích, chu vi nhưng không biết cách tính cụ thể cho từng hình.

CÚ PHÁP THỰC HÀNH - Hinh.java
// Lớp trừu tượng Hinh
abstract class Hinh {
    // --- THUỘC TÍNH ---
    protected String mauSac;
    
    // --- CONSTRUCTOR ---
    public Hinh(String mauSac) {
        this.mauSac = mauSac;
    }
    
    // --- PHƯƠNG THỨC THÔNG THƯỜNG ---
    public void hienThiMauSac() {
        System.out.println("Màu sắc: " + mauSac);
    }
    
    // --- PHƯƠNG THỨC TRỪU TƯỢNG ---
    public abstract double tinhDienTich();
    
    public abstract double tinhChuVi();
    
    // --- PHƯƠNG THỨC THÔNG THƯỜNG KHÁC ---
    public void inThongTin() {
        System.out.println("Đây là một hình học");
        hienThiMauSac();
    }
}
LỚP CON KẾ THỪA - HinhVuong.java
// Lớp HinhVuong phải triển khai TẤT CẢ phương thức trừu tượng
class HinhVuong extends Hinh {
    private double canh;
    
    public HinhVuong(String mauSac, double canh) {
        super(mauSac);
        this.canh = canh;
    }
    
    @Override
    public double tinhDienTich() {
        return canh * canh;
    }
    
    @Override
    public double tinhChuVi() {
        return 4 * canh;
    }
    
    public void inThongTinChiTiet() {
        System.out.println("Hình vuông:");
        System.out.println("- Cạnh: " + canh);
        super.inThongTin();
        System.out.println("- Diện tích: " + tinhDienTich());
        System.out.println("- Chu vi: " + tinhChuVi());
    }
}
Lớp HinhTron cũng kế thừa từ Hinh
class HinhTron extends Hinh {
    private double banKinh;
    
    public HinhTron(String mauSac, double banKinh) {
        super(mauSac);
        this.banKinh = banKinh;
    }
    
    @Override
    public double tinhDienTich() {
        return Math.PI * banKinh * banKinh;
    }
    
    @Override
    public double tinhChuVi() {
        return 2 * Math.PI * banKinh;
    }
    
    public void inThongTinChiTiet() {
        System.out.println("Hình tròn:");
        System.out.println("- Bán kính: " + banKinh);
        super.inThongTin();
        System.out.printf("- Diện tích: %.2f\n", tinhDienTich());
        System.out.printf("- Chu vi: %.2f\n", tinhChuVi());
    }
}
CHƯƠNG TRÌNH CHÍNH - DemoAbstractClass.java
public class DemoAbstractClass {
    public static void main(String[] args) {
        System.out.println("=== DEMO LỚP TRỪU TƯỢNG TRONG JAVA ===\n");
        
        // KHÔNG THỂ TẠO ĐỐI TƯỢNG TỪ LỚP TRỪU TƯỢNG
        // Hinh h = new Hinh("Đỏ"); // LỖI! Không thể tạo đối tượng
        
        // TẠO ĐỐI TƯỢNG TỪ LỚP CON
        HinhVuong hv = new HinhVuong("Xanh", 5.0);
        HinhTron ht = new HinhTron("Đỏ", 3.0);
        
        System.out.println("=== THÔNG TIN HÌNH VUÔNG ===");
        hv.inThongTinChiTiet();
        
        System.out.println("\n=== THÔNG TIN HÌNH TRÒN ===");
        ht.inThongTinChiTiet();
        
        // DEMO ĐA HÌNH VỚI LỚP TRỪU TƯỢNG
        System.out.println("\n=== ĐA HÌNH VỚI LỚP TRỪU TƯỢNG ===");
        
        // Mảng chứa các đối tượng Hinh (lớp trừu tượng)
        Hinh[] danhSachHinh = new Hinh[2];
        danhSachHinh[0] = hv;
        danhSachHinh[1] = ht;
        
        double tongDienTich = 0;
        double tongChuVi = 0;
        
        for (Hinh h : danhSachHinh) {
            System.out.println("\n--- Xử lý hình ---");
            h.hienThiMauSac();
            
            // Gọi phương thức trừu tượng (đa hình tại runtime)
            double dienTich = h.tinhDienTich();
            double chuVi = h.tinhChuVi();
            
            System.out.println("Diện tích: " + dienTich);
            System.out.println("Chu vi: " + chuVi);
            
            tongDienTich += dienTich;
            tongChuVi += chuVi;
        }
        
        System.out.println("\n=== TỔNG KẾT ===");
        System.out.println("Tổng diện tích: " + tongDienTich);
        System.out.println("Tổng chu vi: " + tongChuVi);
    }
}

2.6. PHÂN TÍCH VÍ DỤ

1. Tại sao cần lớp trừu tượng Hinh?

  • Hình học là khái niệm chung, không cụ thể
  • Các hình cụ thể (vuông, tròn) có cách tính diện tích/chu vi khác nhau
  • Cần một khuôn mẫu chung để đảm bảo tất cả hình đều có diện tích và chu vi

2. Điểm quan trọng trong code:

Điểm quan trọng
// Lớp trừu tượng - không thể tạo đối tượng
abstract class Hinh {
    // Phương thức trừu tượng - không có thân
    public abstract double tinhDienTich();
}

// Lớp con - phải triển khai phương thức trừu tượng
class HinhVuong extends Hinh {
    private double canh;
    
    @Override
    public double tinhDienTich() {
        return canh * canh;
    }
}

PHẦN 3: PHƯƠNG THỨC TRỪU TƯỢNG (ABSTRACT METHOD)

3.1. LÝ THUYẾT

Khái niệm

Phương thức trừu tượng là phương thức chỉ có khai báo, không có thân phương thức (implementation).

Đặc điểm:

  • Chỉ có thể khai báo trong lớp trừu tượng hoặc interface
  • Không có dấu ngoặc nhọn {}, kết thúc bằng dấu chấm phẩy ;
  • Lớp con phải triển khai (override) tất cả phương thức trừu tượng

3.2. MỤC ĐÍCH

Mục đích Giải thích
Định nghĩa hợp đồng Bắt buộc lớp con phải có phương thức này
Linh hoạt triển khai Mỗi lớp con có cách triển khai riêng
Đa hình Cho phép gọi phương thức qua tham chiếu lớp cha
Thiết kế mở rộng Dễ dàng thêm chức năng mới

3.3. CÚ PHÁP

// Trong lớp trừu tượng hoặc interface
[access-modifier] abstract kieuTraVe tenPhuongThuc([danh-sach-tham-so]);

// Ví dụ:
public abstract void diChuyen();
public abstract double tinhToan(int a, int b);
protected abstract String layThongTin();

3.4. GIẢI THÍCH CÚ PHÁP

  • abstract: Từ khóa bắt buộc
  • Không có thân phương thức: Chỉ kết thúc bằng ;
  • access-modifier: Thường là public hoặc protected
  • Lớp con phải triển khai: Sử dụng @Override

3.5. VÍ DỤ CHI TIẾT

LÝ THUYẾT

Động vật là khái niệm trừu tượng. Mọi động vật đều phải ăn, di chuyển, nhưng cách thực hiện khác nhau.

LỚP TRỪU TƯỢNG VỚI PHƯƠNG THỨC TRỪU TƯỢNG - DongVat.java

        System.out.println("Tuổi: " + tuoi);

}

PHẦN 4: INTERFACE (GIAO DIỆN)

4.1. LÝ THUYẾT

Khái niệm

Interface là một bản hợp đồng (contract) định nghĩa các phương thức mà lớp triển khai phải có.

Đặc điểm của interface (Java 8+):

  • Phương thức trừu tượng: Chỉ khai báo, không triển khai (trước Java 8)
  • Phương thức default: Có thể có thân phương thức (Java 8+)
  • Phương thức static: Có thể có thân phương thức (Java 8+)
  • Hằng số: Mặc định là public static final
  • Không có constructor, không có thuộc tính (trừ hằng số)

4.2. MỤC ĐÍCH

Mục đích Giải thích
Định nghĩa hợp đồng Ép lớp phải có phương thức nhất định
Đa kế thừa Một lớp có thể triển khai nhiều interface
Tách biệt interface và implementation Dễ thay đổi triển khai
Loose coupling Giảm sự phụ thuộc giữa các lớp

4.3. CÚ PHÁP

// Khai báo interface
[access-modifier] interface TenInterface {
    // HẰNG SỐ
    // Mặc định: public static final
    kieuDuLieu TEN_HANG = giaTri;
    
    // PHƯƠNG THỨC TRỪU TƯỢNG
    // Mặc định: public abstract
    kieuTraVe tenPhuongThuc([tham-so]);
    
    // PHƯƠNG THỨC DEFAULT (Java 8+)
    default kieuTraVe tenPhuongThucDefault([tham-so]) {
        // Thân phương thức
    }
    
    // PHƯƠNG THỨC STATIC (Java 8+)
    static kieuTraVe tenPhuongThucStatic([tham-so]) {
        // Thân phương thức
    }
}

// Lớp triển khai interface
class TenLop implements TenInterface {
    // Phải triển khai tất cả phương thức trừu tượng
}

VD1- QUẢN LÝ CHẤM CÔNG NHÂN VIÊN (JAVA)

Viết chương trình Java quản lý chấm công và tính lương nhân viên, trong đó có một khuôn mẫu chung quy định số giờ làm chuẩn trong ngày là 8 giờ và yêu cầu mọi loại nhân viên đều phải có chức năng tính ngày công và tính lương. Xây dựng hai loại nhân viên: nhân viên hành chính (tính lương theo ngày làm việc) và nhân viên theo giờ (tính ngày công dựa trên tổng số giờ làm). Cuối chương trình, viết hàm chạy thử để quản lý các nhân viên thông qua kiểu chung và in ra số ngày công, tiền lương tương ứng.

INTERFACE CHẤM CÔNG - ChamCong.java
public interface ChamCong {
    // HẰNG SỐ
    public static final int GIO_CHUAN = 8;
    
    // PHƯƠNG THỨC TRỪU TƯỢNG
    public abstract int tinhNgayCong();
    
    public abstract double tinhLuong();
}
LỚP NHÂN VIÊN HÀNH CHÍNH - NhanVienHanhChinh.java
public class NhanVienHanhChinh implements ChamCong {
    private int soNgayLam;
    private double luongNgay;
    
    public NhanVienHanhChinh(int soNgayLam, double luongNgay) {
        this.soNgayLam = soNgayLam;
        this.luongNgay = luongNgay;
    }
    
    @Override
    public int tinhNgayCong() {
        return soNgayLam;
    }
    
    @Override
    public double tinhLuong() {
        return soNgayLam * luongNgay;
    }
}
LỚP NHÂN VIÊN THEO GIỜ - NhanVienTheoGio.java
public class NhanVienTheoGio implements ChamCong {

    public NhanVienTheoGio(int tongSoGio, double luongGio) {
        this.tongSoGio = tongSoGio;
        this.luongGio = luongGio;

    @Override
    public int tinhNgayCong() {

4.4 IMPLEMENTS NHIỀU INTERFACE (JAVA)

Viết chương trình Java quản lý nhân viên FullTime với các chức năng: chấm công, tính lương, tính thưởng và tiền phạt. Mỗi chức năng được chuẩn hóa bằng một khuôn mẫu riêng. Nhân viên FullTime phải thực hiện đầy đủ các khuôn mẫu này và viết chương trình chạy thử để in ra ngày công, lương, thưởng, phạt và tổng thu nhập.

INTERFACE CHẤM CÔNG – ChamCong.java
public interface ChamCong {
    public static final int GIO_CHUAN = 8;
    
    public abstract int tinhNgayCong();
    
    public abstract double tinhLuong();
}
INTERFACE THƯỞNG – Thuong.java
public interface Thuong {
    public static final double TIEN_THUONG_CO_BAN = 500000;
    
    public abstract double tinhThuong();
}
INTERFACE PHẠT – Phat.java
public interface Phat {
    public static final double TIEN_PHAT_MOI_LAN = 100000;
    
    public abstract double tinhTienPhat();
}
LỚP IMPLEMENTS NHIỀU INTERFACE – NhanVienFullTime.java
public class NhanVienFullTime
    implements ChamCong, Thuong, Phat {
    private int tongSoGio;
    private double luongGio;
    private int soLanDiTre;
    
    public NhanVienFullTime(int tongSoGio, double luongGio, int soLanDiTre) {
        this.tongSoGio = tongSoGio;
        this.luongGio = luongGio;
        this.soLanDiTre = soLanDiTre;
    }
    
    // Interface ChamCong
    @Override
    public int tinhNgayCong() {
        return tongSoGio / ChamCong.GIO_CHUAN;
    }
    
    @Override
    public double tinhLuong() {
        return tongSoGio * luongGio;
    }
    
    // Interface Thuong
    @Override
    public double tinhThuong() {
        if (soLanDiTre == 0) {
            return Thuong.TIEN_THUONG_CO_BAN;
        }
        return 0;
    }
    
    // Interface Phat
    @Override
    public double tinhTienPhat() {
        return soLanDiTre * Phat.TIEN_PHAT_MOI_LAN;
    }
    
    // Tổng thu nhập
    public double tinhThuNhap() {
        return tinhLuong() + tinhThuong() - tinhTienPhat();
    }
}

  • Chuẩn hóa hành vi

PHẦN 5: SO SÁNH ABSTRACT CLASS VÀ INTERFACE

5.1. BẢNG SO SÁNH CHI TIẾT

TIÊU CHÍ ABSTRACT CLASS INTERFACE
Từ khóa abstract class interface
Kế thừa extends (1 lớp) implements (nhiều interface)
Constructor Có Không
Thuộc tính Biến instance (có thể thay đổi) Chỉ hằng số (public static final)
Phương thức Trừu tượng + thông thường Trừu tượng + default + static
Phạm vi truy cập public, protected, private Chỉ public (mặc định)
Tạo đối tượng Không thể Không thể
Mục đích Template, chia sẻ code Contract, định nghĩa khả năng
Khi nào dùng IS-A relationship HAS-A/CAN-DO relationship

PHẦN 6: TỔNG KẾT VÀ BÀI TẬP

6.1. TÓM TẮT KIẾN THỨC

1. Tính trừu tượng là gì?

  • Ẩn chi tiết phức tạp, chỉ hiển thị tính năng cần thiết
  • Hai cách triển khai: Abstract class và Interface

2. Abstract Class:

  • abstract class + extends
  • Có constructor, thuộc tính, phương thức
  • Không thể tạo đối tượng
  • Lớp con phải triển khai phương thức trừu tượng

3. Abstract Method:

  • Chỉ khai báo, không có thân
  • Kết thúc bằng dấu ;
  • Bắt buộc lớp con triển khai

4. Interface:

  • interface + implements
  • Chỉ hằng số và phương thức
  • Hỗ trợ đa kế thừa
  • Java 8+: có thể có default và static methods

6.4. TỔNG KẾT CUỐI CÙNG

Những điểm chính cần nhớ:

  1. Tính trừu tượng giúp ẩn chi tiết phức tạp, chỉ hiển thị cần thiết
  2. Abstract class dùng khi có quan hệ IS-A và cần chia sẻ code
  3. Interface dùng khi định nghĩa khả năng (CAN-DO) và cần đa kế thừa
  4. Abstract method chỉ khai báo, lớp con phải triển khai
  5. Một lớp có thể vừa extends một lớp và implements nhiều interface
