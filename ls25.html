PHẦN 1: SET INTERFACE

1.1 SET LÀ GÌ?

Định nghĩa:

Set là interface đại diện cho một tập hợp các phần tử không trùng lặp, giống như tập hợp trong toán học.

Đặc điểm:

  • ✅ Không chứa phần tử trùng lặp
  • ❌ Không đảm bảo thứ tự (tùy loại Set)
  • ❌ Không có chỉ số index
  • ⚠️ Có thể chứa null (tùy loại Set)

Ví dụ trực quan về Set:

  • Tập hợp các số nguyên tố: {2, 3, 5, 7, 11}
  • Tập hợp các chữ cái: {'a', 'b', 'c'}
  • Tập hợp sinh viên có điểm cao

1.2 CÚ PHÁP KHAI BÁO SET

1.2.1 Cú pháp tổng quát

Cú pháp
// Cách 1: Dùng interface Set (khuyến nghị)
Set<KiểuDữLiệu> tênBiến = new HashSet<>();
Set<KiểuDữLiệu> tênBiến = new LinkedHashSet<>();
Set<KiểuDữLiệu> tênBiến = new TreeSet<>();

// Cách 2: Dùng class cụ thể
HashSet<KiểuDữLiệu> tênBiến = new HashSet<>();
LinkedHashSet<KiểuDữLiệu> tênBiến = new LinkedHashSet<>();
TreeSet<KiểuDữLiệu> tênBiến = new TreeSet<>();

1.2.2 Ví dụ khai báo

Ví dụ
// Set với kiểu String
Set<String> stringSet = new HashSet<>();
Set<String> linkedStringSet = new LinkedHashSet<>();
Set<String> treeStringSet = new TreeSet<>();

// Set với kiểu Integer
Set<Integer> intSet = new HashSet<>();
Set<Integer> linkedIntSet = new LinkedHashSet<>();
Set<Integer> treeIntSet = new TreeSet<>();

// Set với kiểu đối tượng tự định nghĩa
Set<Student> studentSet = new HashSet<>();
Set<Student> linkedStudentSet = new LinkedHashSet<>();
Set<Student> treeStudentSet = new TreeSet<>(); // Student phải implements Comparable

1.2.3 Khởi tạo Set với giá trị ban đầu

Ví dụ
// Cách 1: Dùng Arrays.asList()
Set<String> set1 = new HashSet<>(Arrays.asList("Java", "Python", "C++"));

// Cách 2: Dùng Set.of() (Java 9+) - tạo immutable set
Set<String> set2 = Set.of("Java", "Python", "C++");

// Cách 3: Thêm từng phần tử
Set<String> set3 = new HashSet<>();
set3.add("Java");
set3.add("Python");
set3.add("C++");

// Cách 4: Dùng double brace initialization (không khuyến nghị)
Set<String> set4 = new HashSet<>() {{
    add("Java");
    add("Python");
    add("C++");
}};

1.3 CÁC PHƯƠNG THỨC CỦA SET

1.3.1 Phương thức add(E e)

Cú pháp
boolean add(E e)

Giải thích:

Thêm phần tử e vào Set
Trả về true nếu thêm thành công (phần tử chưa tồn tại)
Trả về false nếu phần tử đã tồn tại

Ví dụ
Set<String> set = new HashSet<>();
System.out.println(set.add("Java")); // true
System.out.println(set.add("Python")); // true
System.out.println(set.add("Java")); // false (đã có)
System.out.println(set); // [Java, Python]

1.3.2 Phương thức remove(Object o)

Cú pháp
boolean remove(Object o)

Giải thích:

Xóa phần tử o khỏi Set
Trả về true nếu xóa thành công
Trả về false nếu phần tử không tồn tại

Ví dụ
Set<String> set = new HashSet<>();
set.add("Java");
set.add("Python");
set.add("JavaScript");

System.out.println(set.remove("Python")); // true
System.out.println(set.remove("C++")); // false
System.out.println(set); // [Java, JavaScript]

1.3.3 Phương thức contains(Object o)

Cú pháp
boolean contains(Object o)

Giải thích:

Kiểm tra phần tử o có tồn tại trong Set không

Set<String> set = new HashSet<>();

set.add("Python");

System.out.println(set.contains("Java")); // true
System.out.println(set.contains("C++")); // false

1.3.7 Phương thức addAll(Collection c)

Cú pháp
boolean addAll(Collection<? extends E> c)

Giải thích:

Thêm tất cả phần tử từ Collection c vào Set
Trả về true nếu Set thay đổi

Ví dụ
Set<String> set = new HashSet<>();
set.add("Java");

List<String> list = Arrays.asList("Python", "JavaScript", "Java", "C++");
set.addAll(list);
System.out.println(set); // [Java, C++, Python, JavaScript] (bỏ trùng)

1.3.8 Phương thức retainAll(Collection c)

Cú pháp
boolean retainAll(Collection<?> c)

Giải thích:

Giữ lại các phần tử có trong Collection c
Xóa các phần tử không có trong c
Trả về true nếu Set thay đổi

Ví dụ
Set<Integer> set = new HashSet<>(Arrays.asList(1, 2, 3, 4, 5));
Set<Integer> keep = new HashSet<>(Arrays.asList(2, 4, 6));

boolean changed = set.retainAll(keep);
System.out.println("Set thay đổi? " + changed); // true
System.out.println(set); // [2, 4]

1.3.9 Phương thức removeAll(Collection c)

Cú pháp
boolean removeAll(Collection<?> c)

Giải thích:

Xóa các phần tử có trong Collection c
Trả về true nếu Set thay đổi

Ví dụ
Set<Integer> set = new HashSet<>(Arrays.asList(1, 2, 3, 4, 5));
Set<Integer> remove = new HashSet<>(Arrays.asList(2, 4, 6));

boolean changed = set.removeAll(remove);
System.out.println("Set thay đổi? " + changed); // true
System.out.println(set); // [1, 3, 5]

1.3.10 Phương thức toArray()

Cú pháp
Object[] toArray()
<T> T[] toArray(T[] a)

Giải thích:

Chuyển Set thành mảng
Có hai phiên bản: không tham số và có tham số để chỉ định kiểu mảng

Ví dụ
Set<String> set = new HashSet<>(Arrays.asList("Java", "Python", "C++"));

// Cách 1: toArray() không tham số
Object[] arr1 = set.toArray();
System.out.println(Arrays.toString(arr1)); // [Java, C++, Python]

// Cách 2: toArray(T[] a) với tham số
String[] arr2 = set.toArray(new String[0]);
System.out.println(Arrays.toString(arr2)); // [Java, C++, Python]

// Cách 3: toArray(T[] a) với mảng có kích thước
String[] arr3 = set.toArray(new String[set.size()]);
System.out.println(Arrays.toString(arr3)); // [Java, C++, Python]

1.3.11 Phương thức iterator()

Cú pháp
Iterator<E> iterator()

Giải thích:

Trả về Iterator để duyệt các phần tử trong Set

Ví dụ
Set<String> set = new HashSet<>(Arrays.asList("Java", "Python", "JavaScript"));

Iterator<String> it = set.iterator();
while (it.hasNext()) {
    String element = it.next();
    System.out.println(element);
}

PHẦN 2: HASH SET

2.1 HASH SET LÀ GÌ?

Định nghĩa:

HashSet là class triển khai của Set interface, sử dụng bảng băm (hash table) để lưu trữ phần tử.

Đặc điểm:

  • ✅ Không trùng lặp
  • ❌ Không đảm bảo thứ tự
  • ✅ Cho phép 1 phần tử null
  • ⚡ Hiệu suất O(1) cho thêm/xóa/tìm

2.2 CÚ PHÁP KHAI BÁO HASH SET

2.2.1 Cú pháp tổng quát

Cú pháp
// Cách 1: Dùng interface Set (khuyến nghị)
Set<KiểuDữLiệu> tênBiến = new HashSet<>();

// Cách 2: Dùng class cụ thể
HashSet<KiểuDữLiệu> tênBiến = new HashSet<>();

2.2.2 Các constructor

Constructor
// 1. Constructor mặc định
HashSet<String> set1 = new HashSet<>();

// 2. Constructor với initial capacity
HashSet<String> set2 = new HashSet<>(32);

// 3. Constructor với initial capacity và load factor
HashSet<String> set3 = new HashSet<>(32, 0.8f);

// 4. Constructor từ Collection khác
List<String> list = Arrays.asList("A", "B", "C");
HashSet<String> set4 = new HashSet<>(list);

2.2.3 Ví dụ khai báo

Ví dụ
// HashSet với kiểu String
Set<String> stringSet = new HashSet<>();
stringSet.add("Java");
stringSet.add("Python");

// HashSet với kiểu Integer
Set<Integer> intSet = new HashSet<>();
intSet.add(10);
intSet.add(20);

// HashSet với kiểu đối tượng tự định nghĩa
Set<Student> studentSet = new HashSet<>();
studentSet.add(new Student(1, "Nguyen Van A"));

2.3 CÁC PHƯƠNG THỨC CỦA HASH SET

(HashSet kế thừa tất cả phương thức từ Set interface, đã trình bày ở phần 1.3)

Ví dụ tổng hợp:

HashSetExample.java
public class HashSetExample {
    public static void main(String[] args) {
        // Khai báo và khởi tạo
        Set<String> set = new HashSet<>();
        
        // Thêm phần tử
        set.add("Apple");
        set.add("Banana");
        set.add("Orange");
        set.add("Apple"); // Không thêm
        
        System.out.println("Set: " + set);
        
        // Kiểm tra tồn tại
        if (set.contains("Banana")) {
            System.out.println("Có Banana");
        }
        
        // Xóa phần tử
        set.remove("Orange");
        
        // Duyệt set
        for (String fruit : set) {
            System.out.println(fruit);
        }
        
        // Kích thước
        System.out.println("Size: " + set.size());
    }
}

PHẦN 3: LINKED HASH SET

3.1 LINKED HASH SET LÀ GÌ?

Định nghĩa:

LinkedHashSet là class triển khai Set, kết hợp HashSet và LinkedList để lưu theo thứ tự chèn.

Đặc điểm:

  • ✅ Giữ thứ tự chèn (insertion order)
  • ✅ Không trùng lặp
  • ✅ Cho phép 1 phần tử null
  • ⚡ Hiệu suất gần bằng HashSet

3.2 CÚ PHÁP KHAI BÁO LINKED HASH SET

3.2.1 Cú pháp tổng quát

Cú pháp
// Cách 1: Dùng interface Set (khuyến nghị)
Set<KiểuDữLiệu> tênBiến = new LinkedHashSet<>();

// Cách 2: Dùng class cụ thể
LinkedHashSet<KiểuDữLiệu> tênBiến = new LinkedHashSet<>();

3.2.2 Các constructor

Constructor
// 1. Constructor mặc định
LinkedHashSet<String> set1 = new LinkedHashSet<>();

// 2. Constructor với initial capacity
LinkedHashSet<String> set2 = new LinkedHashSet<>(32);

// 3. Constructor với initial capacity và load factor
LinkedHashSet<String> set3 = new LinkedHashSet<>(32, 0.8f);

// 4. Constructor từ Collection khác
List<String> list = Arrays.asList("A", "B", "C");
LinkedHashSet<String> set4 = new LinkedHashSet<>(list);

3.2.3 Ví dụ khai báo

Ví dụ
// LinkedHashSet với kiểu String
Set<String> stringSet = new LinkedHashSet<>();
stringSet.add("Java");
stringSet.add("Python");
stringSet.add("JavaScript");

// LinkedHashSet với kiểu Integer
Set<Integer> intSet = new LinkedHashSet<>();
intSet.add(30);
intSet.add(10);
intSet.add(20);

3.3 SO SÁNH HASH SET VÀ LINKED HASH SET

CompareHashAndLinked.java
public class CompareHashAndLinked {
    public static void main(String[] args) {
        // HashSet - không đảm bảo thứ tự
        Set<String> hashSet = new HashSet<>();
        hashSet.add("One");
        hashSet.add("Two");
        hashSet.add("Three");
        hashSet.add("Four");
        
        // LinkedHashSet - giữ thứ tự chèn
        Set<String> linkedSet = new LinkedHashSet<>();
        linkedSet.add("One");
        linkedSet.add("Two");
        linkedSet.add("Three");
        linkedSet.add("Four");
        
        System.out.println("HashSet (không thứ tự): " + hashSet);
        System.out.println("LinkedHashSet (có thứ tự): " + linkedSet);
    }
}
Kết quả có thể:
HashSet (không thứ tự): [Three, Four, One, Two]
LinkedHashSet (có thứ tự): [One, Two, Three, Four]

PHẦN 4: TREE SET

4.1 TREE SET LÀ GÌ?

Định nghĩa:

TreeSet là class triển khai Set và SortedSet, sử dụng cây đỏ-đen (Red-Black tree) để lưu theo thứ tự sắp xếp.

Đặc điểm:

  • ✅ Sắp xếp theo thứ tự tự nhiên hoặc Comparator
  • ✅ Không trùng lặp
  • ❌ Không chứa null (từ Java 7)
  • ⚡ Hiệu suất O(log n)

4.2 CÚ PHÁP KHAI BÁO TREE SET

4.2.1 Cú pháp tổng quát

Cú pháp
// Cách 1: Dùng interface Set (khuyến nghị)
Set<KiểuDữLiệu> tênBiến = new TreeSet<>();

// Cách 2: Dùng class cụ thể
TreeSet<KiểuDữLiệu> tênBiến = new TreeSet<>();

// Cách 3: Với Comparator tùy chỉnh
TreeSet<KiểuDữLiệu> tênBiến = new TreeSet<>(comparator);

4.2.2 Các constructor

Constructor
// 1. Constructor mặc định (thứ tự tự nhiên)
TreeSet<String> set1 = new TreeSet<>();

// 2. Constructor với Comparator
TreeSet<String> set2 = new TreeSet<>(Comparator.reverseOrder());

// 3. Constructor từ Collection khác
List<String> list = Arrays.asList("B", "A", "C");
TreeSet<String> set3 = new TreeSet<>(list);

// 4. Constructor từ SortedSet khác
TreeSet<String> set4 = new TreeSet<>(set3);

4.2.3 Ví dụ khai báo

Ví dụ
// TreeSet với thứ tự tự nhiên (tăng dần)
TreeSet<Integer> naturalOrder = new TreeSet<>();
naturalOrder.add(5);
naturalOrder.add(1);
naturalOrder.add(3);
System.out.println(naturalOrder); // [1, 3, 5]

// TreeSet với Comparator giảm dần
TreeSet<Integer> reverseOrder = new TreeSet<>((a, b) -> b - a);
reverseOrder.add(5);
reverseOrder.add(1);
reverseOrder.add(3);
System.out.println(reverseOrder); // [5, 3, 1]

// TreeSet với String (thứ tự alphabet)
TreeSet<String> names = new TreeSet<>();
names.add("John");
names.add("Alice");
names.add("Bob");
System.out.println(names); // [Alice, Bob, John]

PHẦN 5: MAP INTERFACE

5.1 MAP LÀ GÌ?

Định nghĩa:

Map là interface lưu trữ các cặp key-value, mỗi key ánh xạ đến một value.

Đặc điểm:

  • ✅ Key là duy nhất (không trùng)
  • ✅ Value có thể trùng nhau
  • ❌ Không phải Collection (nhưng thuộc Collection Framework)

5.2 CÚ PHÁP KHAI BÁO MAP

5.2.1 Cú pháp tổng quát

Cú pháp
// Cách 1: Dùng interface Map (khuyến nghị)
Map<KiểuKey, KiểuValue> tênBiến = new HashMap<>();
Map<KiểuKey, KiểuValue> tênBiến = new LinkedHashMap<>();
Map<KiểuKey, KiểuValue> tênBiến = new TreeMap<>();

// Cách 2: Dùng class cụ thể
HashMap<KiểuKey, KiểuValue> tênBiến = new HashMap<>();
LinkedHashMap<KiểuKey, KiểuValue> tênBiến = new LinkedHashMap<>();
TreeMap<KiểuKey, KiểuValue> tênBiến = new TreeMap<>();

7.4 ỨNG DỤNG: LRU CACHE

LRUCache.java
class LRUCache<K, V> extends LinkedHashMap<K, V> {
    private final int capacity;
    
    public LRUCache(int capacity) {
        super(capacity, 0.75f, true); // accessOrder = true
        this.capacity = capacity;
    }
    
    @Override
    protected boolean removeEldestEntry(Map.Entry<K, V> eldest) {
        // Tự động xóa entry cũ nhất khi size > capacity
        return size() > capacity;
    }
}

public class LRUCacheExample {
    public static void main(String[] args) {
        LRUCache<Integer, String> cache = new LRUCache<>(3);
        
        cache.put(1, "A");
        cache.put(2, "B");
        cache.put(3, "C");
        System.out.println("Cache: " + cache); // {1=A, 2=B, 3=C}
        
        cache.get(1); // Truy cập 1, nó sẽ được đưa lên cuối
        cache.put(4, "D"); // Thêm 4, vượt capacity, xóa entry cũ nhất (2)
        
        System.out.println("Sau khi thêm 4: " + cache); // {3=C, 1=A, 4=D}
    }
}

PHẦN 8: TREE MAP

8.1 TREE MAP LÀ GÌ?

Định nghĩa:

TreeMap là class triển khai Map interface và SortedMap, sử dụng cây đỏ-đen (Red-Black tree) để lưu trữ các cặp key-value có thứ tự theo key.

Đặc điểm:

  • ✅ Sắp xếp theo key (thứ tự tự nhiên hoặc Comparator)
  • ✅ Key duy nhất
  • ❌ Không chấp nhận key null (từ Java 7)
  • ✅ Value có thể null
  • ⚡ Hiệu suất O(log n)

Cú pháp

8.3.9 Phương thức pollFirstEntry()

Cú pháp

8.3.11 Phương thức subMap(K from, K to)

Cú pháp

PHẦN 9: TỔNG KẾT

9.1 SO SÁNH CÁC LOẠI SET

Đặc điểm HashSet LinkedHashSet TreeSet
Cú pháp khai báo Set<T> set = new HashSet<>() Set<T> set = new LinkedHashSet<>() Set<T> set = new TreeSet<>()
Thứ tự Không Theo thứ tự chèn Theo thứ tự key
Null Cho phép Cho phép Không cho phép
Hiệu suất O(1) O(1) O(log n)
Cấu trúc Hash table Hash table + LinkedList Red-Black tree
Khi nào dùng Tốc độ cao Giữ thứ tự chèn Cần sắp xếp

9.2 SO SÁNH CÁC LOẠI MAP

Đặc điểm HashMap LinkedHashMap TreeMap
Cú pháp khai báo Map<K,V> map = new HashMap<>() Map<K,V> map = new LinkedHashMap<>() Map<K,V> map = new TreeMap<>()
Thứ tự Không Theo chèn/truy cập Theo key
Key null Cho phép 1 Cho phép 1 Không cho phép
Value null Cho phép Cho phép Cho phép
Hiệu suất O(1) O(1) O(log n)
Khi nào dùng Tốc độ cao Cache, LRU Cần sắp xếp theo key

Tóm tắt nhanh:

  • HashSet/HashMap - "Vua tốc độ" - dùng khi cần thao tác nhanh O(1), không quan tâm thứ tự
  • LinkedHashSet/LinkedHashMap - "Giữ trật tự" - dùng khi cần duy trì thứ tự chèn
  • TreeSet/TreeMap - "Sắp xếp tự động" - dùng khi cần dữ liệu có thứ tự
