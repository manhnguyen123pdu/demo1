PHẦN 1: TỔNG QUAN VỀ SO SÁNH TRONG JAVA

1.1 TẠI SAO CẦN SO SÁNH?

Định nghĩa:

Trong lập trình, việc so sánh các đối tượng là nhu cầu cơ bản để:

  • Sắp xếp (sorting)
  • Tìm kiếm (searching)
  • Tìm min/max
  • Xác định thứ tự ưu tiên

Vấn đề với kiểu nguyên thủy:

Ví dụ
int a = 5, b = 10;
System.out.println(a < b); // true - Dễ dàng so sánh
System.out.println(a > b); // false

Vấn đề với đối tượng:

Ví dụ
Student s1 = new Student("Alice", 20);
Student s2 = new Student("Bob", 22);

// Làm thế nào để so sánh?
// s1 < s2? Theo tên? Theo tuổi? Theo điểm?
// Java không thể tự động biết cách so sánh đối tượng do người dùng định nghĩa

1.2 HAI CÁCH SO SÁNH TRONG JAVA

Java cung cấp hai interface để giải quyết vấn đề so sánh:

Interface Mô tả Đặc điểm
Comparable So sánh "tự nhiên" (natural ordering) Một lớp chỉ có một cách so sánh duy nhất
Comparator So sánh tùy chỉnh (custom ordering) Nhiều cách so sánh khác nhau cho cùng một lớp

PHẦN 2: COMPARABLE INTERFACE

2.1 COMPARABLE LÀ GÌ?

Định nghĩa:

Comparable là một interface trong Java, cho phép một lớp định nghĩa thứ tự tự nhiên (natural ordering) của các đối tượng của lớp đó.

Đặc điểm:

  • Nằm trong package java.lang
  • Chỉ có một phương thức duy nhất: compareTo()
  • Được gọi là "thứ tự tự nhiên" của lớp
  • Các lớp trong Java đã implement: String, Integer, Double, Date,...

Cú pháp:

Cú pháp
public interface Comparable<T> {
    public int compareTo(T o);
}

2.2 PHƯƠNG THỨC compareTo()

Cú pháp:

Cú pháp
int compareTo(T o)

Giải thích:

So sánh đối tượng hiện tại (this) với đối tượng o được truyền vào.

Giá trị trả về:

  • Số âm (< 0): nếu đối tượng hiện tại nhỏ hơn đối tượng o
  • Số 0 (== 0): nếu đối tượng hiện tại bằng đối tượng o
  • Số dương (> 0): nếu đối tượng hiện tại lớn hơn đối tượng o

Quy tắc:

this < o  →  trả về số âm
this = o  →  trả về 0
this > o  →  trả về số dương

2.3 CÁCH IMPLEMENT COMPARABLE

2.3.1 Ví dụ với số (Integer đã có sẵn)

Ví dụ
// Integer đã implement Comparable
Integer a = 5;
Integer b = 10;

System.out.println(a.compareTo(b)); // -5 (5 < 10)
System.out.println(b.compareTo(a)); // 5 (10 > 5)
System.out.println(a.compareTo(5)); // 0 (5 = 5)

2.3.2 Ví dụ với String (String đã có sẵn)

Ví dụ
String s1 = "Apple";
String s2 = "Banana";

System.out.println(s1.compareTo(s2)); // -1 (Apple < Banana)
System.out.println(s2.compareTo(s1)); // 1 (Banana > Apple)
System.out.println("Hello".compareTo("Hello")); // 0

2.4 TỰ IMPLEMENT COMPARABLE CHO LỚP CỦA MÌNH

2.4.1 Ví dụ 1: So sánh Student theo tuổi

Student.java
class Student implements Comparable<Student> {
    private String name;
    private int age;
    private double gpa;
    
    public Student(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
    }
    
    @Override
    public int compareTo(Student other) {
        // So sánh theo tuổi (tăng dần)
        if (this.age < other.age) {
            return -1;
        } else if (this.age > other.age) {
            return 1;
        } else {
            return 0;
        }
    }
    
    // Có thể viết ngắn gọn hơn:
    // public int compareTo(Student other) {
    // return this.age - other.age;
    // }
    
    @Override
    public String toString() {
        return name + "(" + age + " tuổi, GPA:" + gpa + ")";
    }
}
ComparableExample1.java
public class ComparableExample1 {
    public static void main(String[] args) {
        Student s1 = new Student("Alice", 20, 8.5);
        Student s2 = new Student("Bob", 22, 7.8);
        Student s3 = new Student("Charlie", 20, 9.0);
        
        System.out.println("s1 so với s2: " + s1.compareTo(s2)); // -1 (20 < 22)
        System.out.println("s2 so với s1: " + s2.compareTo(s1)); // 1 (22 > 20)
        System.out.println("s1 so với s3: " + s1.compareTo(s3)); // 0 (20 = 20)
        
        // Sắp xếp danh sách sinh viên theo tuổi
        List<Student> students = new ArrayList<>(Arrays.asList(s1, s2, s3));
        System.out.println("\nTrước sort: " + students);
        
        Collections.sort(students); // Tự động dùng compareTo
        System.out.println("Sau sort theo tuổi: " + students);
    }
}

class StudentMulti implements Comparable<StudentMulti> {
    private String name;
    private int age;
    private double gpa;
    
    public StudentMulti(String name, int age, double gpa) {
        this.name = name;
        this.age = age;
        this.gpa = gpa;
    }
    
    @Override
    public int compareTo(StudentMulti other) {
        // Ưu tiên 1: GPA giảm dần (cao nhất trước)
        if (this.gpa > other.gpa) {
            return -1;
        } else if (this.gpa < other.gpa) {
            return 1;
        } else {
            // Nếu GPA bằng nhau, so sánh theo tuổi tăng dần
            if (this.age < other.age) {
                return -1;
            } else if (this.age > other.age) {
                return 1;
            } else {
                // Nếu cả GPA và tuổi bằng nhau, so sánh theo tên alphabet
                return this.name.compareTo(other.name);
            }
        }
    }
    
    @Override
    public String toString() {
        return name + "(tuổi:" + age + ", GPA:" + gpa + ")";
    }
}

2.5 LƯU Ý KHI IMPLEMENT COMPARABLE

2.5.1 Quy tắc bắc cầu

Product.java
class Product implements Comparable<Product> {
    private int id;
    private String name;
    
    @Override
    public int compareTo(Product other) {
        return this.id - other.id;
    }
    
    // LƯU Ý: Phải đảm bảo tính bắc cầu
    // Nếu a.compareTo(b) > 0 và b.compareTo(c) > 0
    // Thì a.compareTo(c) > 0
}

2.5.2 Quan hệ với equals()

Khuyến nghị: Nên đảm bảo compareTo() nhất quán với equals()

Student.java
class Student implements Comparable<Student> {
    private int id;
    private String name;
    
    @Override
    public boolean equals(Object obj) {
        if (this == obj) return true;
        if (!(obj instanceof Student)) return false;
        Student other = (Student) obj;
        return this.id == other.id;
    }
    
    @Override
    public int compareTo(Student other) {
        // Nên so sánh dựa trên cùng trường với equals
        return Integer.compare(this.id, other.id);
    }
}

PHẦN 3: COMPARATOR INTERFACE

3.1 COMPARATOR LÀ GÌ?

Định nghĩa:

Comparator là một interface trong Java, cho phép định nghĩa nhiều cách so sánh khác nhau cho cùng một lớp đối tượng.

Đặc điểm:

  • Nằm trong package java.util
  • Có phương thức chính: compare()
  • Cho phép tạo nhiều bộ so sánh khác nhau
  • Thường dùng khi không thể sửa lớp gốc hoặc cần nhiều cách sắp xếp

Cú pháp:

Cú pháp
public interface Comparator<T> {
    int compare(T o1, T o2);
}

3.2 PHƯƠNG THỨC compare()

Cú pháp:

Cú pháp
int compare(T o1, T o2)

Giải thích:

So sánh hai đối tượng o1 và o2 với nhau.

Giá trị trả về:

  • Số âm (< 0): nếu o1 nhỏ hơn o2
  • Số 0 (== 0): nếu o1 bằng o2
  • Số dương (> 0): nếu o1 lớn hơn o2

Quy tắc:

o1 < o2  →  trả về số âm
o1 = o2  →  trả về 0
o1 > o2  →  trả về số dương

3.3 CÁC CÁCH TẠO COMPARATOR

3.3.1 Cách 1: Tạo class riêng implement Comparator

Student.java
class Student {

        this.name = name;

        this.gpa = gpa;

    public String getName() { return name; }

// Comparator so sánh theo tên

}

3.3.4 Cách 4: Dùng Comparator.comparing() (Java 8+)

ComparatorExample4.java
import java.util.Comparator;

public class ComparatorExample4 {
    public static void main(String[] args) {
        List<Student> students = new ArrayList<>();
        students.add(new Student("Alice", 20, 8.5));
        students.add(new Student("Bob", 22, 9.0));
        students.add(new Student("Charlie", 19, 7.5));
        
        // Sắp xếp theo tên
        students.sort(Comparator.comparing(Student::getName));
        System.out.println("Theo tên: " + students);
        
        // Sắp xếp theo tuổi
        students.sort(Comparator.comparingInt(Student::getAge));
        System.out.println("Theo tuổi: " + students);
        
        // Sắp xếp theo GPA
        students.sort(Comparator.comparingDouble(Student::getGpa));
        System.out.println("Theo GPA tăng: " + students);
        
        // Sắp xếp theo GPA giảm dần
        students.sort(Comparator.comparingDouble(Student::getGpa).reversed());
        System.out.println("Theo GPA giảm: " + students);
    }
}

PHẦN 4: CÁC PHƯƠNG THỨC CỦA COMPARATOR (JAVA 8+)

4.1 TỔNG QUAN

Java 8 bổ sung nhiều phương thức tiện ích cho Comparator, cho phép:

  • Tạo Comparator dễ dàng hơn
  • Kết hợp nhiều Comparator
  • Đảo ngược thứ tự
  • Xử lý null

4.2 PHƯƠNG THỨC comparing()

4.2.1 comparing(Function keyExtractor)

Cú pháp
static <T,U extends Comparable<? super U>> Comparator<T> comparing(Function<? super T,? extends U> keyExtractor)

Giải thích:

Tạo Comparator dựa trên một key được trích xuất từ đối tượng
Key phải implement Comparable

Ví dụ
List<Student> students = new ArrayList<>();

// So sánh theo tên
Comparator<Student> nameComp = Comparator.comparing(Student::getName);
students.sort(nameComp);

// So sánh theo tuổi
students.sort(Comparator.comparing(Student::getAge));

// So sánh theo GPA
students.sort(Comparator.comparing(Student::getGpa));

4.2.2 comparing(Function keyExtractor, Comparator keyComparator)

Cú pháp
static <T,U> Comparator<T> comparing(Function<? super T,? extends U> keyExtractor, Comparator<? super U> keyComparator)

Giải thích:

Tạo Comparator dựa trên key, với Comparator riêng cho key

Ví dụ
// So sánh theo tên nhưng không phân biệt hoa thường
students.sort(Comparator.comparing(
    Student::getName,
    String.CASE_INSENSITIVE_ORDER
));

// So sánh theo độ dài tên
students.sort(Comparator.comparing(
    s -> s.getName().length(),
    Comparator.naturalOrder()
));

4.3 PHƯƠNG THỨC thenComparing()

4.3.1 thenComparing(Comparator other)

Cú pháp
default Comparator<T> thenComparing(Comparator<? super T> other)

Giải thích:

Kết hợp Comparator hiện tại với Comparator khác
Dùng khi cần so sánh theo nhiều tiêu chí

Ví dụ
// So sánh theo GPA giảm dần, sau đó theo tuổi tăng dần
Comparator<Student> multiComp = Comparator
    .comparingDouble(Student::getGpa).reversed()
    .thenComparingInt(Student::getAge);

students.sort(multiComp);

// So sánh theo tuổi, sau đó theo tên
students.sort(Comparator
    .comparingInt(Student::getAge)
    .thenComparing(Student::getName)
);

4.3.2 thenComparing(Function keyExtractor)

Cú pháp
default <U extends Comparable<? super U>> Comparator<T> thenComparing(Function<? super T,? extends U> keyExtractor)

Giải thích:

Kết hợp với key mới sau Comparator hiện tại

Ví dụ
// So sánh theo tuổi, sau đó theo tên
students.sort(Comparator
    .comparingInt(Student::getAge)
    .thenComparing(Student::getName)
);

// So sánh theo GPA, sau đó theo tuổi, sau đó theo tên
students.sort(Comparator
    .comparingDouble(Student::getGpa).reversed()
    .thenComparingInt(Student::getAge)
    .thenComparing(Student::getName)
);

4.4 PHƯƠNG THỨC reversed()

Cú pháp
default Comparator<T> reversed()

Giải thích:

Trả về Comparator đảo ngược thứ tự

Ví dụ
// So sánh theo tên giảm dần
students.sort(Comparator.comparing(Student::getName).reversed());

// So sánh theo tuổi giảm dần
students.sort(Comparator.comparingInt(Student::getAge).reversed());

// Kết hợp với thenComparing
students.sort(Comparator
    .comparingDouble(Student::getGpa).reversed()
    .thenComparing(Student::getName)
);

4.5 PHƯƠNG THỨC nullsFirst() và nullsLast()

4.5.1 nullsFirst(Comparator comparator)

Cú pháp
static <T> Comparator<T> nullsFirst(Comparator<? super T> comparator)

Giải thích:

Xử lý null: đưa các giá trị null lên đầu

Ví dụ
List<String> names = new ArrayList<>(Arrays.asList("Alice", null, "Bob", null, "Charlie"));

// Sắp xếp, null lên đầu
names.sort(Comparator.nullsFirst(Comparator.naturalOrder()));
System.out.println(names); // [null, null, Alice, Bob, Charlie]

4.5.2 nullsLast(Comparator comparator)

Cú pháp
static <T> Comparator<T> nullsLast(Comparator<? super T> comparator)

Giải thích:

Xử lý null: đưa các giá trị null xuống cuối

Ví dụ
List<String> names = new ArrayList<>(Arrays.asList("Alice", null, "Bob", null, "Charlie"));

// Sắp xếp, null xuống cuối
names.sort(Comparator.nullsLast(Comparator.naturalOrder()));
System.out.println(names); // [Alice, Bob, Charlie, null, null]

4.6 PHƯƠNG THỨC naturalOrder() và reverseOrder()

4.6.1 naturalOrder()

Cú pháp

Trả về Comparator đảo ngược thứ tự tự nhiên

public class ComparatorMethodsExample {
    public static void main(String[] args) {
        List<Student> students = new ArrayList<>();
        students.add(new Student("Alice", 20, 8.5));
        students.add(new Student("Bob", 22, 9.0));

        students.add(new Student("David", 21, 8.5));
        students.add(new Student("Eve", 20, 9.0));
        students.add(null);
        
        // 1. So sánh đơn giản
        students.stream()
            .filter(s -> s != null)
            .sorted(Comparator.comparing(Student::getName))
            .forEach(s -> System.out.print(s.getName() + " "));
        System.out.println();
        
        // 2. So sánh nhiều tiêu chí
        students.stream()
            .filter(s -> s != null)
            .sorted(Comparator
                .comparingDouble(Student::getGpa).reversed()
                .thenComparingInt(Student::getAge)
                .thenComparing(Student::getName))
            .forEach(s -> System.out.println(s));

        ));

        // 4. Kết hợp nhiều Comparator
        Comparator<Student> complex = Comparator
            .nullsFirst(Comparator.comparingInt(Student::getAge))
            .thenComparing(Student::getName)
            .thenComparingDouble(Student::getGpa).reversed();
        
        students.sort(complex);
    }

public class SortingExample {

        List<Integer> numbers = new ArrayList<>(Arrays.asList(5, 2, 8, 1, 9, 3, 7, 4, 6));

        System.out.println("Tăng dần: " + numbers);
        
        numbers.sort(Comparator.reverseOrder());
        System.out.println("Giảm dần: " + numbers);
        
        // 2. Sắp xếp List<String>
        List<String> words = new ArrayList<>(Arrays.asList(

        ));
        
        words.sort(Comparator.naturalOrder());
        System.out.println("Tự nhiên: " + words);
        
        words.sort(String.CASE_INSENSITIVE_ORDER);
        System.out.println("Không phân biệt hoa thường: " + words);
        
        words.sort(Comparator.comparingInt(String::length));
        System.out.println("Theo độ dài: " + words);
        
        // 3. Sắp xếp List<Student> với nhiều tiêu chí
        List<Student> students = new ArrayList<>();

        students.add(new Student("Eve", 20, 9.0));

            .thenComparing(Student::getName)

PHẦN 7: TỔNG KẾT

7.1 KHI NÀO DÙNG COMPARABLE?

Tình huống Nên dùng Comparable
Có một cách sắp xếp "tự nhiên" duy nhất ✅
Có thể sửa lớp nguồn ✅
Cần sắp xếp mặc định ✅
Ví dụ String, Integer, Date

7.2 KHI NÀO DÙNG COMPARATOR?

Tình huống Nên dùng Comparator
Cần nhiều cách sắp xếp khác nhau ✅
Không thể sửa lớp nguồn ✅
Cần sắp xếp tạm thời ✅
Cần sắp xếp theo tiêu chí phức tạp ✅
Ví dụ Sắp xếp nhân viên theo lương, tên, phòng ban

7.3 SO SÁNH NHANH

Tiêu chí Comparable Comparator (Java 8+)
Cú pháp class A implements Comparable<A> Comparator.comparing(A::getField)
Phương thức compareTo() compare() hoặc lambda
Số lượng 1 Nhiều
Vị trí Trong class Bên ngoài
Lambda Không Có
Method chaining Không Có (thenComparing())
Xử lý null Thủ công nullsFirst(), nullsLast()

Tóm tắt nhanh:

  • Comparable - "Thứ tự tự nhiên" - dùng khi lớp có một cách sắp xếp mặc định duy nhất
  • Comparator - "Thứ tự tùy chỉnh" - dùng khi cần nhiều cách sắp xếp khác nhau
