GIỚI THIỆU VỀ MẢNG VÀ CHUỖI

Tầm quan trọng

Mảng và chuỗi là hai cấu trúc dữ liệu cơ bản và quan trọng nhất trong lập trình Java. Hiểu rõ cách làm việc với chúng là nền tảng để giải quyết các bài toán phức tạp hơn.

Nội dung chính của buổi học

  • PHẦN 1: MẢNG (ARRAY) - Cấu trúc lưu trữ nhiều giá trị cùng kiểu
  • PHẦN 2: CHUỖI (STRING) - Xử lý văn bản và các phương thức làm việc với chuỗi
  • PHẦN 3: STRINGBUILDER & STRINGBUFFER - Xử lý chuỗi hiệu quả
  • BÀI TẬP THỰC HÀNH - Áp dụng kiến thức vào thực tế

ĐỊNH NGHĨA MẢNG

1.1 Định nghĩa Mảng

Định nghĩa: Mảng trong Java là một cấu trúc dữ liệu cho phép lưu trữ nhiều giá trị cùng kiểu dữ liệu trong một biến duy nhất. Các giá trị được lưu theo thứ tự và mỗi giá trị có một chỉ số (index) riêng để truy xuất.

Đặc điểm:

  • Index bắt đầu từ 0
  • Kích thước cố định sau khi khai báo
  • Tất cả phần tử cùng kiểu dữ liệu
  • Có thể chứa các kiểu dữ liệu nguyên thủy hoặc đối tượng

Hình dung mảng:

Giống như một dãy tủ locker, mỗi locker là một phần tử trong mảng:

[0]
10
[1]
20
[2]
30
[3]
40
[4]
50
Khai báo mảng
int[] numbers = {10, 20, 30, 40, 50};

  • Dễ thao tác hàng loạt: Thêm, xóa, sắp xếp, lọc...
  • Phù hợp vòng lặp: Xử lý dữ liệu hiệu quả

1.3 Khai báo Mảng

Cú pháp khai báo mảng
// Cách 1: Sử dụng dấu ngoặc vuông [] (Array literal)
kiểu_dữ_liệu[] tên_mảng = {giá_trị_1, giá_trị_2, ..., giá_trị_n};

// Cách 2: Sử dụng từ khóa new (Array constructor)
kiểu_dữ_liệu[] tên_mảng = new kiểu_dữ_liệu[kích_thước];

// Cách 3: Khai báo mảng rỗng
kiểu_dữ_liệu[] tên_mảng = new kiểu_dữ_liệu[]{giá_trị_1, giá_trị_2, ...};

Giải thích cú pháp:

  • kiểu_dữ_liệu: int, double, String, boolean,...
  • tên_mảng: Tên biến mảng, đặt theo quy tắc Java
  • kích_thước: Số lượng phần tử tối đa
  • new: Toán tử cấp phát bộ nhớ
  • {} Khởi tạo với giá trị cụ thể
ArrayDeclaration.java
public class ArrayDeclaration {
    public static void main(String[] args) {
        System.out.println("=== KHAI BÁO MẢNG ===");
        
        // Cách 1: Array literal
        int[] numbers1 = {1, 2, 3, 4, 5};
        String[] fruits = {"Táo", "Cam", "Chuối"};
        
        // Cách 2: Array constructor
        int[] numbers2 = new int[3]; // Mảng 3 phần tử
        double[] prices = new double[5];
        
        // Cách 3: Khai báo và khởi tạo với new
        char[] letters = new char[]{'A', 'B', 'C'};
        
        // Gán giá trị cho mảng
        numbers2[0] = 10;
        numbers2[1] = 20;
        numbers2[2] = 30;
        
        prices[0] = 1.5;
        prices[1] = 2.3;
        prices[2] = 3.7;
        prices[3] = 4.1;
        prices[4] = 5.9;
        
        // Hiển thị mảng
        System.out.println("numbers1: " + java.util.Arrays.toString(numbers1));
        System.out.println("fruits: " + java.util.Arrays.toString(fruits));
        System.out.println("numbers2: " + java.util.Arrays.toString(numbers2));
        System.out.println("prices: " + java.util.Arrays.toString(prices));
        System.out.println("letters: " + java.util.Arrays.toString(letters));
    }
}

1.4 Cấp phát bộ nhớ trong mảng

Quy trình cấp phát bộ nhớ mảng trong java

B1: Khai báo mảng: int[] arr;

➡ Khi đó, Java chỉ khai báo một biến tham chiếu (arr) nhưng chưa cấp phát bộ nhớ thực sự cho các phần tử. arr lúc này chưa trỏ đến vùng nhớ nào (null).

B2: Yêu cầu cấp phát

Khi dùng từ khóa new: arr = new int[5];
➡ Trình JVM (Java Virtual Machine) gửi yêu cầu cấp phát bộ nhớ cho vùng nhớ heap.

B3: JVM kiểm tra vùng nhớ Heap

JVM sẽ kiểm tra xem vùng nhớ Heap còn đủ chỗ để lưu mảng kích thước 5 phần tử.
Nếu đủ bộ nhớ → chuyển sang bước 6.
Nếu không đủ bộ nhớ → sang bước 4.

B4: Kích hoạt bộ dọn rác (Garbage Collector – GC)

JVM kích hoạt GC để giải phóng những đối tượng không còn được tham chiếu trong chương trình.
Sau khi GC hoạt động, nếu có đủ bộ nhớ trống thì tiếp tục cấp phát.

B5: GC kiểm tra kết quả

Nếu GC giải phóng đủ bộ nhớ → sang bước 6.
Nếu vẫn không đủ → JVM ném lỗi: OutOfMemoryError

B6: Cấp phát bộ nhớ thành công

JVM tạo ra một vùng nhớ mới trong heap để chứa mảng.
Vùng nhớ này gồm:
- Thông tin về kiểu dữ liệu (int, double, String, …)
- Số lượng phần tử
- Các ô nhớ liên tiếp để lưu giá trị

B7: Khởi tạo giá trị mặc định

Tất cả phần tử trong mảng sẽ được gán giá trị mặc định tùy theo kiểu dữ liệu:

Kiểu dữ liệu Giá trị mặc định
int, byte, short, long 0
float, double 0.0
char '\u0000' (ký tự rỗng)
boolean false
Đối tượng (String, Object, …) null
B8: Gán địa chỉ cho biến tham chiếu

Biến mảng (ví dụ arr) sẽ nhận địa chỉ vùng nhớ của mảng vừa được tạo trong Heap.
Lúc này arr trỏ tới vùng nhớ mảng [0, 0, 0, 0, 0].

B9: Mảng sẵn sàng sử dụng

Giờ ta có thể truy cập và gán giá trị cho từng phần tử:
arr[0] = 10;
arr[1] = 20;

[0]
10
[1]
20
[2]
0
[3]
0
[4]
0

TRUY CẬP PHẦN TỬ TRONG MẢNG

1.5 Truy cập Phần tử trong Mảng

Cú pháp truy cập
tên_mảng[vị_trí]

Giải thích:

  • vị_trí: Chỉ số index (bắt đầu từ 0)
  • Muốn truy cập phần tử thứ i → dùng arr[i-1]
  • Truy cập cuối mảng → arr[arr.length - 1]
ArrayAccess.java
public class ArrayAccess {
    public static void main(String[] args) {
        System.out.println("=== TRUY CẬP PHẦN TỬ MẢNG ===");
        String[] fruits = {"Táo", "Cam", "Chuối", "Xoài", "Ổi"};
        
        // Truy cập phần tử
        System.out.println("Phần tử đầu: " + fruits[0]); // Táo
        System.out.println("Phần tử thứ 2: " + fruits[1]); // Cam
        System.out.println("Phần tử thứ 3: " + fruits[2]); // Chuối
        
        // Truy cập phần tử cuối
        System.out.println("Phần tử cuối: " + fruits[fruits.length - 1]); // Ổi
        
        // Thay đổi phần tử
        fruits[1] = "Quýt";
        System.out.println("Sau thay đổi: " + fruits[1]); // Quýt
        
        // Lỗi thường gặp
        try {
            System.out.println(fruits[-1]); // Lỗi: ArrayIndexOutOfBoundsException
        } catch (Exception e) {
            System.out.println("Lỗi truy cập index -1: " + e.getMessage());
        }
        
        try {
            System.out.println(fruits[10]); // Lỗi: ArrayIndexOutOfBoundsException
        } catch (Exception e) {
            System.out.println("Lỗi truy cập index 10: " + e.getMessage());
        }
        
        // Duyệt toàn bộ mảng
        System.out.println("\nDuyệt mảng fruits:");
        for (int i = 0; i < fruits.length; i++) {
            System.out.println("fruits[" + i + "] = " + fruits[i]);
        }
    }
}

Lỗi thường gặp

ArrayIndexOutOfBoundsException: Xảy ra khi truy cập index ngoài phạm vi mảng (index âm hoặc lớn hơn length-1)

CÁC PHƯƠNG THỨC VỚI MẢNG TRONG JAVA

2.1 Các phương thức cơ bản của mảng

2.1.1. length - Độ dài mảng

Định nghĩa: length là thuộc tính (property) của mảng, trả về số lượng phần tử trong mảng.

Cú pháp
tên_mảng.length

Giải thích:

  • Không có dấu ngoặc đơn vì là thuộc tính, không phải phương thức
  • Trả về số nguyên (int)
  • Chỉ đọc (read-only), không thể thay đổi
ArrayLength.java
public class ArrayLength {
    public static void main(String[] args) {
        int[] numbers = {10, 20, 30, 40, 50};
        String[] names = {"An", "Bình", "Chi"};
        
        System.out.println("Độ dài mảng numbers: " + numbers.length); // 5
        System.out.println("Độ dài mảng names: " + names.length); // 3
        
        // Sử dụng trong vòng lặp
        for (int i = 0; i < numbers.length; i++) {
            System.out.println("numbers[" + i + "] = " + numbers[i]);
        }
    }
}

2.2 Các phương thức với lớp Arrays

2.2.1. Arrays.toString() - Chuyển mảng thành chuỗi

Định nghĩa: Phương thức tĩnh của lớp Arrays để chuyển mảng thành chuỗi dễ đọc.

Cú pháp
Arrays.toString(tên_mảng)

Giải thích:

  • Phương thức tĩnh → gọi trực tiếp qua lớp Arrays
  • Trả về String định dạng: [element1, element2, ...]
  • Cần import java.util.Arrays
ToStringMethod.java
import java.util.Arrays;

public class ToStringMethod {
    public static void main(String[] args) {
        int[] numbers = {1, 2, 3, 4, 5};
        String[] fruits = {"Táo", "Cam", "Chuối"};
        boolean[] flags = {true, false, true};
        
        System.out.println("Mảng numbers: " + Arrays.toString(numbers));
        System.out.println("Mảng fruits: " + Arrays.toString(fruits));
        System.out.println("Mảng flags: " + Arrays.toString(flags));
        
        // So sánh với in trực tiếp
        System.out.println("In trực tiếp: " + numbers); // [I@hashcode
    }
}

2.2.2. Arrays.sort() - Sắp xếp mảng

Định nghĩa: Sắp xếp các phần tử trong mảng theo thứ tự tăng dần.

Cú pháp
Arrays.sort(tên_mảng) // Sắp xếp toàn bộ
Arrays.sort(tên_mảng, fromIndex, toIndex) // Sắp xếp một phần

Giải thích:

  • Sắp xếp tại chỗ (in-place) → thay đổi mảng gốc
  • fromIndex: vị trí bắt đầu (bao gồm)
  • toIndex: vị trí kết thúc (không bao gồm)
  • Sắp xếp tăng dần theo thứ tự tự nhiên
SortMethod.java
import java.util.Arrays;

public class SortMethod {
    public static void main(String[] args) {
        int[] numbers = {5, 2, 8, 1, 9, 3};
        String[] names = {"Mai", "An", "Bình", "Chi"};
        
        System.out.println("Mảng gốc: " + Arrays.toString(numbers));
        
        // Sắp xếp toàn bộ
        Arrays.sort(numbers);
        System.out.println("Sau sort toàn bộ: " + Arrays.toString(numbers));
        
        // Sắp xếp một phần (từ index 1 đến 4)
        int[] partial = {5, 2, 8, 1, 9, 3};
        Arrays.sort(partial, 1, 4);
        System.out.println("Sort từ index 1-4: " + Arrays.toString(partial));
        
        // Sắp xếp chuỗi
        System.out.println("Names gốc: " + Arrays.toString(names));
        Arrays.sort(names);
        System.out.println("Names sau sort: " + Arrays.toString(names));
    }
}

TÓM TẮT CÁC PHƯƠNG THỨC MẢNG

Phương thức Mục đích Có thay đổi mảng gốc?
arr.length Lấy độ dài Không
Arrays.toString(arr) Chuyển thành chuỗi Không
Arrays.sort(arr) Sắp xếp Có
Arrays.binarySearch(arr, value) Tìm kiếm nhị phân Không
Arrays.copyOf(arr, newLength) Sao chép mảng Không
Arrays.copyOfRange(arr, start, end) Sao chép đoạn Không
Arrays.fill(arr, value) Điền giá trị Có
Arrays.equals(arr1, arr2) So sánh mảng Không

Lưu ý quan trọng:

  • Luôn import java.util.Arrays khi sử dụng các phương thức
  • length là thuộc tính, không phải phương thức
  • Các phương thức có từ "copy" tạo mảng mới
  • sort() và fill() thay đổi mảng gốc
