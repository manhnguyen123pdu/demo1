MỤC 1: COLLECTIONS FRAMEWORK - TỔNG QUAN

1.1 Collections Framework là gì?

Định nghĩa:

Collections Framework là một bộ khung (framework) thống nhất trong Java, cung cấp các lớp và interface để lưu trữ, quản lý và xử lý các nhóm đối tượng một cách hiệu quả.

Giải thích đơn giản:

Thay vì tự viết code để quản lý mảng, thêm/xóa phần tử, sắp xếp,... Collections Framework đã làm sẵn hết cho bạn.

Cú pháp khai báo:

Import Collections Framework
import java.util.*; // Import toàn bộ Collections Framework

Ví dụ:

ViDuCollection.java
import java.util.*;

public class ViDuCollection {
    public static void main(String[] args) {
        // Không cần tự viết code quản lý mảng
        List danhSach = new ArrayList<>();
        
        // Thêm phần tử - Java tự động mở rộng mảng
        danhSach.add("Java");
        danhSach.add("Python");
        
        // Lấy phần tử
        String ngonNgu = danhSach.get(0);
        System.out.println("Ngôn ngữ đầu: " + ngonNgu);
    }
}

1.2 Lợi ích của Collections Framework

Các lợi ích chính:

  • Không phải tự viết lại cấu trúc dữ liệu
  • Hiệu suất cao (đã được tối ưu)
  • Có sẵn các thuật toán (sắp xếp, tìm kiếm,...)
  • Tương thích với nhau (có thể chuyển đổi qua lại)

Ví dụ về thuật toán có sẵn:

LoiIchCollection.java
import java.util.*;

public class LoiIchCollection {
    public static void main(String[] args) {
        List so = new ArrayList<>();
        so.add(5);
        so.add(2);
        so.add(8);
        so.add(1);
        
        // Sắp xếp - có sẵn
        Collections.sort(so);
        System.out.println("Sau sắp xếp: " + so); // [1, 2, 5, 8]
        
        // Tìm kiếm nhị phân - có sẵn
        int viTri = Collections.binarySearch(so, 5);
        System.out.println("Số 5 ở vị trí: " + viTri); // 2
        
        // Đảo ngược - có sẵn
        Collections.reverse(so);
        System.out.println("Sau đảo ngược: " + so); // [8, 5, 2, 1]
    }
}

1.3 Các interface chính

Sơ đồ phân cấp:

Collection (interface gốc)
    ├── List (interface)    - Danh sách có thứ tự, cho phép trùng
    ├── Set (interface)     - Tập hợp không trùng
    └── Queue (interface)   - Hàng đợi (FIFO)

Ví dụ so sánh các interface:

SoSanhInterface.java
import java.util.*;

public class SoSanhInterface {
    public static void main(String[] args) {
        // 1. LIST - có thứ tự, cho phép trùng
        List list = new ArrayList<>();
        list.add("A"); // index 0
        list.add("B"); // index 1
        list.add("A"); // index 2 - cho phép trùng
        System.out.println("List: " + list); // [A, B, A]
        
        // 2. SET - không trùng
        Set set = new HashSet<>();
        set.add("A");
        set.add("B");
        set.add("A"); // bị bỏ qua vì trùng
        System.out.println("Set: " + set); // [A, B]
        
        // 3. QUEUE - vào trước ra trước
        Queue queue = new LinkedList<>();
        queue.add("Người 1");
        queue.add("Người 2");
        queue.add("Người 3");
        
        String raTruoc = queue.poll(); // lấy phần tử đầu
        System.out.println("Ra trước: " + raTruoc); // Người 1
        System.out.println("Còn lại: " + queue); // [Người 2, Người 3]
    }
}

MỤC 2: LIST INTERFACE

2.1 Định nghĩa List

Định nghĩa:

List là một interface trong Java Collections Framework, đại diện cho một danh sách có thứ tự (ordered collection).

Cú pháp khai báo List:

Cú pháp
// Cú pháp
List tenBien = new Implementation<>();

// Ví dụ
List list = new ArrayList<>();
List numbers = new LinkedList<>();

Giải thích cú pháp:

  • List: Interface cha
  • <KiểuDữLiệu>: Kiểu dữ liệu của các phần tử (Wrapper class)
  • tenBien: Tên biến
  • new Implementation<>(): Khởi tạo đối tượng cụ thể (ArrayList, LinkedList,...)

2.2 Đặc điểm của List

Các đặc điểm chính:

  • Có thứ tự - Phần tử được lưu theo thứ tự thêm vào
  • Cho phép trùng - Có thể thêm phần tử giống nhau
  • Truy cập bằng index - Mỗi phần tử có vị trí (bắt đầu từ 0)
  • Cho phép null - Có thể lưu giá trị null

Ví dụ minh họa các đặc điểm:

DacDiemList.java
import java.util.*;

public class DacDiemList {
    public static void main(String[] args) {
        List list = new ArrayList<>();
        
        // 1. Có thứ tự
        list.add("Java"); // index 0
        list.add("Python"); // index 1
        list.add("JavaScript"); // index 2
        System.out.println("Thứ tự: " + list); // [Java, Python, JavaScript]
        
        // 2. Cho phép trùng
        list.add("Java"); // Thêm thêm một lần nữa
        System.out.println("Cho phép trùng: " + list); // [Java, Python, JavaScript, Java]
        
        // 3. Truy cập bằng index
        String phanTuThu2 = list.get(1); // Lấy phần tử ở index 1
        System.out.println("Phần tử thứ 2: " + phanTuThu2); // Python
        
        // 4. Cho phép null
        list.add(null);
        System.out.println("Cho phép null: " + list); // [Java, Python, JavaScript, Java, null]
    }
}

3.1.2 add(int index, E element) - Thêm vào vị trí chỉ định

Cú pháp
void add(int index, E element)

Giải thích:

  • Thêm phần tử vào vị trí index chỉ định
  • Các phần tử từ index trở đi được đẩy sang phải
  • index: Vị trí cần thêm (0 đến size)
  • element: Phần tử cần thêm
AddViDu2.java
import java.util.*;

        System.out.println("Sau khi thêm C++ vào index 1: " + list);
        // [Java, C++, Python, JavaScript]
        
        // Thêm vào đầu danh sách
        list.add(0, "Ruby");
        System.out.println("Sau khi thêm Ruby vào đầu: " + list);
        // [Ruby, Java, C++, Python, JavaScript]
        
        // Thêm vào cuối (tương đương add(element))
        list.add(list.size(), "Go");
        System.out.println("Sau khi thêm Go vào cuối: " + list);
        // [Ruby, Java, C++, Python, JavaScript, Go]
    }
}

3.2 Phương thức get() - Lấy phần tử

Cú pháp
E get(int index)

Giải thích:

  • Trả về phần tử tại vị trí index chỉ định
  • index: Vị trí cần lấy (0 đến size-1)
  • Ném IndexOutOfBoundsException nếu index không hợp lệ
GetViDu.java
import java.util.*;

public class GetViDu {
    public static void main(String[] args) {
        List list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("JavaScript");
        
        // Lấy phần tử
        String first = list.get(0);
        String second = list.get(1);
        String last = list.get(list.size() - 1);
        
        System.out.println("Phần tử đầu: " + first); // Java
        System.out.println("Phần tử thứ hai: " + second); // Python
        System.out.println("Phần tử cuối: " + last); // JavaScript
        
        // Lỗi nếu index không hợp lệ
        try {
            String error = list.get(10); // index 10 không tồn tại
        } catch (IndexOutOfBoundsException e) {
            System.out.println("Lỗi: Index không hợp lệ");
        }
    }
}

3.9 Phương thức subList() - Lấy danh sách con

Cú pháp
List subList(int fromIndex, int toIndex)

Giải thích:

  • Trả về một view của List từ fromIndex đến toIndex - 1
  • fromIndex: Vị trí bắt đầu (bao gồm)
  • toIndex: Vị trí kết thúc (không bao gồm)
  • Thay đổi trên subList sẽ ảnh hưởng đến List gốc
SubListViDu.java
import java.util.*;

public class SubListViDu {
    public static void main(String[] args) {
        List list = new ArrayList<>();
        list.add("Java");
        list.add("Python");
        list.add("JavaScript");
        list.add("C++");
        list.add("Ruby");
        list.add("Go");
        
        System.out.println("List gốc: " + list);
        // [Java, Python, JavaScript, C++, Ruby, Go]
        
        // Lấy từ index 1 đến 3 (1, 2)
        List sub = list.subList(1, 4);
        System.out.println("subList(1,4): " + sub);
        // [Python, JavaScript, C++]

MỤC 4: ARRAYLIST

4.1 Định nghĩa ArrayList

Định nghĩa:

ArrayList là một class implements List, sử dụng mảng động (dynamic array) để lưu trữ dữ liệu. Khi mảng đầy, nó tự động tăng kích thước.

Cú pháp tạo ArrayList:

Cú pháp
// Cú pháp
ArrayList tenBien = new ArrayList<>();

// Ví dụ

ArrayList list2 = new ArrayList<>(20); // Chỉ định capacity = 20
ArrayList list3 = new ArrayList<>(list1); // Copy từ ArrayList khác

Giải thích cú pháp:

  • ArrayList: Tên class
  • <KiểuDữLiệu>: Kiểu dữ liệu của phần tử (phải là Wrapper class)
  • new ArrayList<>(): Khởi tạo với capacity mặc định (10)
  • new ArrayList<>(20): Khởi tạo với capacity chỉ định
  • new ArrayList<>(collection): Copy từ collection khác

Ví dụ tạo ArrayList:

MỤC 6: ARRAYLIST VS LINKEDLIST

6.1 So sánh về cấu trúc

Tiêu chí ArrayList LinkedList
Cấu trúc Mảng động (dynamic array) Danh sách liên kết kép
Bộ nhớ Lưu liên tiếp trong RAM Lưu rải rác, mỗi node có thêm 2 con trỏ

6.2 So sánh về hiệu suất (Big O Notation)

Thao tác ArrayList LinkedList
get(int index) O(1) - Rất nhanh O(n) - Chậm
add(E element) O(1)* - Nhanh O(1) - Nhanh
add(int index, E) O(n) - Trung bình O(n) - Trung bình
remove(int index) O(n) - Trung bình O(n) - Trung bình
addFirst O(n) - Chậm O(1) - Rất nhanh
removeFirst O(n) - Chậm O(1) - Rất nhanh
addLast O(1)* - Nhanh O(1) - Nhanh
removeLast O(1) - Nhanh O(1) - Nhanh

6.3 Ví dụ so sánh cụ thể

SoSanhArrayListLinkedList.java
import java.util.*;

public class SoSanhArrayListLinkedList {
    
    public static void main(String[] args) {
        int size = 100000;
        
        // TEST 1: Thêm vào cuối
        System.out.println("=== TEST 1: Thêm vào cuối ===");
        
        // ArrayList thêm cuối
        long start = System.currentTimeMillis();
        ArrayList arrayList = new ArrayList<>();
        for (int i = 0; i < size; i++) {
            arrayList.add(i);
        }
        long end = System.currentTimeMillis();
        System.out.println("ArrayList thêm cuối: " + (end - start) + " ms");
        
        // LinkedList thêm cuối
        start = System.currentTimeMillis();
        LinkedList linkedList = new LinkedList<>();
        for (int i = 0; i < size; i++) {
            linkedList.add(i);
        }
        end = System.currentTimeMillis();
        System.out.println("LinkedList thêm cuối: " + (end - start) + " ms");
        
        // TEST 2: Thêm vào đầu
        System.out.println("\n=== TEST 2: Thêm vào đầu ===");
        
        // ArrayList thêm đầu

6.5 Khi nào dùng LinkedList?

KhiNaoDungLinkedList.java
import java.util.*;

public class KhiNaoDungLinkedList {
    public static void main(String[] args) {
        // 1. Thêm/xóa ở đầu nhiều
        LinkedList history = new LinkedList<>();
        
        // Lưu lịch sử truy cập - gần nhất ở đầu
        history.addFirst("Trang chủ"); // Nhanh O(1)
        history.addFirst("Sản phẩm"); // Nhanh O(1)
        history.addFirst("Chi tiết"); // Nhanh O(1)
        System.out.println("Lịch sử: " + history);
        
        // 2. Làm Queue (hàng đợi) - FIFO
        Queue queue = new LinkedList<>();
        queue.offer("Khách 1"); // Thêm vào cuối
        queue.offer("Khách 2");
        queue.offer("Khách 3");
        
        String phucVu = queue.poll(); // Lấy từ đầu - Khách 1
        System.out.println("Đang phục vụ: " + phucVu);
        
        // 3. Làm Stack (ngăn xếp) - LIFO
        Deque stack = new LinkedList<>();
        stack.push("Sách 1");
        stack.push("Sách 2");
        stack.push("Sách 3");
        
        String layRa = stack.pop(); // Lấy từ đầu - Sách 3
        System.out.println("Lấy ra: " + layRa);
        
        // 4. Cần xóa phần tử ở giữa nhiều (với Iterator)
        LinkedList numbers = new LinkedList<>();
        numbers.addAll(Arrays.asList(1, 2, 3, 4, 5, 6));
        
        // Xóa các số chẵn - LinkedList xóa nhanh hơn khi đã có Iterator
        Iterator it = numbers.iterator();
        while (it.hasNext()) {
            if (it.next() % 2 == 0) {
                it.remove(); // O(1) với LinkedList
            }
        }
        System.out.println("Sau khi xóa số chẵn: " + numbers);
    }
}

6.6 Bảng tóm tắt chọn lựa

Tình huống Nên chọn Lý do
Truy cập ngẫu nhiên nhiều ArrayList get(index) là O(1)
Thêm/xóa ở đầu nhiều LinkedList addFirst/removeFirst là O(1)
Thêm/xóa ở cuối nhiều Cả hai đều tốt add/remove cuối O(1)
Làm Queue/Stack LinkedList Có sẵn các phương thức
Sắp xếp, tìm kiếm nhiều ArrayList Collections.sort() hiệu quả hơn
Bộ nhớ hạn chế ArrayList Tốn ít bộ nhớ hơn
Duyệt tuần tự là chính Cả hai đều tốt Iterator cho cả hai

Tóm tắt nhanh:

  • ArrayList - "Vua truy cập ngẫu nhiên" - dùng khi cần get/set nhiều
  • LinkedList - "Vua thêm xóa đầu/cuối" - dùng khi cần add/remove ở đầu nhiều, làm Queue/Stack
