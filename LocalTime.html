PHẦN MỞ ĐẦU

1. GIỚI THIỆU TỔNG QUAN

1.1 Vấn đề với Date/Time cũ (trước Java 8)

Các lớp java.util.Date, java.util.Calendar, java.text.SimpleDateFormat khó sử dụng

Không thread-safe: SimpleDateFormat không an toàn trong môi trường đa luồng

API không nhất quán: Phương thức đặt tên khó hiểu, dễ nhầm lẫn

Xử lý timezone phức tạp: Thiếu hỗ trợ cho các múi giờ

Không tuân thủ chuẩn ISO-8601

Vấn đề với Date/Time cũ
// Ví dụ cũ - PROBLEMATIC
Date date = new Date(); // Mutable, không thread-safe
Calendar cal = Calendar.getInstance();
cal.set(2024, Calendar.DECEMBER, 25); // Tháng bắt đầu từ 0!
SimpleDateFormat sdf = new SimpleDateFormat("dd/MM/yyyy"); // Not thread-safe
Vấn đề chính:
  • Thread-safety: SimpleDateFormat không an toàn trong đa luồng
  • API thiết kế kém: Tháng bắt đầu từ 0, năm từ 1900
  • Mutable objects: Có thể thay đổi trạng thái sau khi tạo
  • Timezone issues: Xử lý timezone phức tạp và dễ sai

1.2 Giới thiệu java.time API (Java 8+)

Package java.time (JSR-310): Thiết kế dựa trên thư viện Joda-Time

Đặc điểm nổi bật: Thread-safe, immutable, API rõ ràng

Tuân thủ ISO-8601: Chuẩn quốc tế về biểu diễn ngày giờ

Phân chia rõ ràng: Mỗi lớp phục vụ một mục đích cụ thể

Tích hợp tốt với Java Stream API

Ví dụ với java.time API
// Ví dụ mới - BETTER
LocalDate date = LocalDate.of(2024, 12, 25); // Immutable, thread-safe
Ưu điểm của java.time:
  • Immutability: Tất cả các lớp đều immutable → thread-safe
  • API nhất quán: Phương thức đặt tên rõ ràng, dễ hiểu
  • ISO-8601 compliant: Tuân theo chuẩn quốc tế
  • Phân chia rõ ràng: Mỗi lớp có mục đích cụ thể
  • Timezone hỗ trợ tốt: Xử lý timezone dễ dàng

PHẦN 3: LOCALTIME - CHỈ GIỜ PHÚT GIÂY

3.1 KHÁI NIỆM LOCALTIME

Mô tả: Đại diện cho thời gian (giờ-phút-giây-nanosecond) không có ngày và timezone

Phân tích:

LocalTime là một lớp immutable đại diện cho thời gian mà không có ngày và múi giờ.

Nó thường được sử dụng để đại diện cho thời gian như giờ làm việc, giờ hẹn, lịch trình...

Định dạng mặc định theo ISO-8601: HH:mm:ss.SSS

PHẦN 4: LOCALDATETIME - NGÀY VÀ GIỜ

4.1 KHÁI NIỆM LOCALDATETIME

Mô tả: Kết hợp LocalDate và LocalTime, không có timezone

Phân tích:

LocalDateTime là sự kết hợp của LocalDate và LocalTime, đại diện cho ngày và giờ mà không có múi giờ.

Thích hợp cho các sự kiện trong nước, lịch hẹn nội bộ, thời điểm không phụ thuộc vào múi giờ.

static LocalDateTime of(int year, int month, int dayOfMonth,

PHẦN 5: ZONEDDATETIME - VỚI TIMEZONE

5.1 KHÁI NIỆM ZONEDDATETIME

Mô tả: Kết hợp LocalDateTime với ZoneId (múi giờ)

Phân tích:

ZonedDateTime là sự kết hợp của LocalDateTime với thông tin múi giờ (ZoneId).

Thích hợp cho các ứng dụng đa múi giờ, lịch hẹn quốc tế, hệ thống phân tán.

Tự động xử lý Daylight Saving Time (DST) và các quy tắc múi giờ phức tạp.

5.3 PHƯƠNG THỨC CHUYỂN ĐỔI TIMEZONE

5.3.1 Phương thức withZoneSameInstant()

Mô tả: Chuyển đổi sang timezone khác, giữ nguyên thời điểm

Cú pháp: ZonedDateTime withZoneSameInstant(ZoneId zone)

Ví dụ:

Ví dụ withZoneSameInstant()
ZonedDateTime vnTime = ZonedDateTime.of(
    2024, 12, 25, 14, 30, 0, 0,
    ZoneId.of("Asia/Ho_Chi_Minh")
);

ZonedDateTime nyTime = vnTime.withZoneSameInstant(
    ZoneId.of("America/New_York")
);
// 14:30 VN (UTC+7) = 02:30 NY (UTC-5) cùng ngày
Phân tích:

Phương thức này giữ nguyên thời điểm thực (instant) trên timeline.

Chỉ thay đổi múi giờ và điều chỉnh giờ địa phương cho phù hợp.

Đây là cách chính xác để chuyển đổi múi giờ.

5.3.2 Phương thức withZoneSameLocal()

Mô tả: Giữ nguyên giờ địa phương, thay đổi timezone

Cú pháp: ZonedDateTime withZoneSameLocal(ZoneId zone)

Ví dụ:

Ví dụ withZoneSameLocal()
ZonedDateTime vnTime = ZonedDateTime.of(
    2024, 12, 25, 14, 30, 0, 0,
    ZoneId.of("Asia/Ho_Chi_Minh")
);

ZonedDateTime wrongNyTime = vnTime.withZoneSameLocal(
    ZoneId.of("America/New_York")
);
// 14:30 NY (UTC-5) ≠ 14:30 VN (UTC+7) - KHÁC thời điểm
Cảnh báo:

Phương thức này giữ nguyên giờ địa phương nhưng thay đổi múi giờ.

Điều này tạo ra một thời điểm KHÁC trên timeline.

Chỉ sử dụng khi bạn thực sự muốn thay đổi múi giờ mà không thay đổi giờ hiển thị.

PHẦN 6: INSTANT - THỜI ĐIỂM TRÊN TIMELINE

6.1 KHÁI NIỆM INSTANT

Mô tả: Đại diện cho thời điểm trên timeline (từ Unix epoch 1970-01-01T00:00:00Z)

Phân tích:

Instant đại diện cho một thời điểm cụ thể trên timeline, tính từ epoch (00:00:00 UTC ngày 1 tháng 1 năm 1970).

Thường được sử dụng để ghi nhật ký, timestamp trong database, và tính toán khoảng thời gian.

Luôn ở múi giờ UTC (Coordinated Universal Time).

Bài tập 1: Chuyển đổi múi giờ

TimezoneConverter.java
import java.time.LocalDateTime;
import java.time.ZoneId;
import java.time.ZonedDateTime;

public class TimezoneConverter {
    public static void main(String[] args) {
        // Tạo thời điểm ở Việt Nam
        ZonedDateTime vnTime = ZonedDateTime.of(
            LocalDateTime.of(2024, 12, 25, 14, 30),
            ZoneId.of("Asia/Ho_Chi_Minh")
        );

        // Chuyển sang các múi giờ khác
        ZonedDateTime nyTime = vnTime.withZoneSameInstant(
            ZoneId.of("America/New_York")
        );

        ZonedDateTime londonTime = vnTime.withZoneSameInstant(
            ZoneId.of("Europe/London")
        );

        System.out.println("Việt Nam: " + vnTime);
        System.out.println("New York: " + nyTime);
        System.out.println("London: " + londonTime);
    }
}

Bài tập 2: Tính thời gian làm việc

WorkingHoursCalculator.java
import java.time.Duration;
import java.time.LocalTime;

public class WorkingHoursCalculator {
    public static void main(String[] args) {
        LocalTime startWork = LocalTime.of(9, 0);
        LocalTime endWork = LocalTime.of(17, 30);
        LocalTime lunchStart = LocalTime.of(12, 0);
        LocalTime lunchEnd = LocalTime.of(13, 0);

        // Tính tổng thời gian làm việc
        Duration morning = Duration.between(startWork, lunchStart);
        Duration afternoon = Duration.between(lunchEnd, endWork);
        Duration totalWork = morning.plus(afternoon);

        System.out.println("Thời gian làm buổi sáng: " + morning.toHours() + " giờ");
        System.out.println("Thời gian làm buổi chiều: " + afternoon.toHours() + " giờ");
        System.out.println("Tổng thời gian làm việc: " + totalWork.toHours() + " giờ " +
            (totalWork.toMinutes() % 60) + " phút");
    }
}

        DateTimeFormatter formatter4 = DateTimeFormatter.ofPattern("hh:mm a");

        System.out.println("Format 1: " + now.format(formatter1));
        System.out.println("Format 2: " + now.format(formatter2));
        System.out.println("Format 3: " + now.format(formatter3));
        System.out.println("Format 4: " + now.format(formatter4));
    }
