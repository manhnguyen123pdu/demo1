GIỚI THIỆU VÀ CÀI ĐẶT

0.1 JDBC LÀ GÌ?

Định nghĩa:

JDBC (Java Database Connectivity) là API (Application Programming Interface) trong Java cho phép các chương trình Java kết nối và tương tác với cơ sở dữ liệu.

Mục đích:

  • Kết nối Java application với database
  • Gửi câu lệnh SQL từ Java xuống database
  • Nhận kết quả từ database về Java
  • Xử lý giao dịch (transaction)

Kiến trúc JDBC:

Java Application
       ↓
   JDBC API
       ↓
JDBC Driver Manager
       ↓
   JDBC Driver
       ↓
   Database

0.2 CÁC THÀNH PHẦN CỦA JDBC

Thành phần Mô tả
JDBC API Các interface và class trong java.sql và javax.sql
Driver Manager Quản lý các driver, kết nối đến database
JDBC Driver Cầu nối giữa Java và database cụ thể
Database Hệ quản trị CSDL (MySQL, Oracle, SQL Server...)

0.3 CÁC LOẠI JDBC DRIVER

Loại Tên Mô tả
Type 1 JDBC-ODBC Bridge Dùng ODBC driver, hiện đã deprecated
Type 2 Native-API Dùng thư viện native của database
Type 3 Network Protocol Dùng middleware server
Type 4 Pure Java Driver thuần Java, kết nối trực tiếp (khuyên dùng)

Trong khóa học này, chúng ta dùng Type 4 driver cho MySQL.

PHẦN 1: JDBC ARCHITECTURE

1.1 KIẾN TRÚC TỔNG QUAN

Định nghĩa:

JDBC Architecture là kiến trúc nhiều lớp (layered architecture) giúp tách biệt ứng dụng Java khỏi database cụ thể.

Sơ đồ chi tiết:

┌─────────────────────┐
│   Java Application  │
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│     JDBC API        │
│  (java.sql.*)       │
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│  DriverManager      │
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│   JDBC Driver       │
└──────────┬──────────┘
           │
┌──────────▼──────────┐
│     Database        │
└─────────────────────┘

1.2 CÁC INTERFACE CHÍNH TRONG JDBC

Interface Mô tả
Driver Đăng ký driver, kết nối database
Connection Đại diện cho một kết nối đến database
Statement Thực thi câu lệnh SQL
PreparedStatement Statement đã được biên dịch trước, chống SQL injection
CallableStatement Gọi stored procedure
ResultSet Kết quả trả về từ câu lệnh SELECT
ResultSetMetaData Thông tin về ResultSet (số cột, tên cột)
DatabaseMetaData Thông tin về database

1.3 CÁC BƯỚC KẾT NỐI JDBC

Quy trình 7 bước:

1. Load driver
         ↓
2. Tạo kết nối (Connection)
         ↓
3. Tạo Statement
         ↓
4. Thực thi câu lệnh SQL
         ↓
5. Xử lý kết quả (ResultSet)
         ↓
6. Đóng kết nối

PHẦN 2: JDBC STEPS

2.1 BƯỚC 1: LOAD DRIVER

Mục đích:

Đăng ký driver với DriverManager để có thể kết nối đến database.

Cú pháp:

Java
// Cách 1: Class.forName() (cổ điển)
Class.forName("com.mysql.cj.jdbc.Driver");

// Cách 2: DriverManager.registerDriver() (ít dùng)
DriverManager.registerDriver(new com.mysql.cj.jdbc.Driver());

// Cách 3: Không cần load (từ JDBC 4.0 trở đi - tự động tìm driver)
// Chỉ cần có driver trong classpath

Ví dụ:

LoadDriverExample.java
import java.sql.*;

public class LoadDriverExample {
    public static void main(String[] args) {
        try {
            // Load MySQL driver
            Class.forName("com.mysql.cj.jdbc.Driver");
            System.out.println("Đã load driver thành công!");
            
        } catch (ClassNotFoundException e) {
            System.out.println("Không tìm thấy driver: " + e.getMessage());
        }
    }
}

Giải thích:

  • com.mysql.cj.jdbc.Driver là tên class driver của MySQL
  • Class.forName() nạp class vào bộ nhớ, tự động đăng ký với DriverManager
  • Từ Java 6, không bắt buộc phải gọi Class.forName() nếu driver có trong classpath

2.2 BƯỚC 2: TẠO KẾT NỐI (CONNECTION)

Mục đích:

Tạo kết nối từ Java application đến database.

Cú pháp:

Java
String url = "jdbc:mysql://host:port/database";
String username = "username";
String password = "password";

Connection conn = DriverManager.getConnection(url, username, password);

Cấu trúc URL kết nối MySQL:

URL
jdbc:mysql://localhost:3306/QuanLySinhVien?useSSL=false&serverTimezone=UTC
↑ ↑ ↑ ↑ ↑
Giao thức Server Port Database Parameters

Ví dụ:

ConnectionExample.java
import java.sql.*;

public class ConnectionExample {
    public static void main(String[] args) {
        Connection conn = null;
        
        try {
            // Thông tin kết nối
            String url = "jdbc:mysql://localhost:3306/QuanLySinhVien?useSSL=false&serverTimezone=UTC";
            String username = "root";
            String password = "123456";
            
            // Tạo kết nối
            conn = DriverManager.getConnection(url, username, password);
            System.out.println("Kết nối thành công!");
            
        } catch (SQLException e) {
            System.out.println("Lỗi kết nối: " + e.getMessage());
        } finally {
            // Đóng kết nối
            try {
                if (conn != null) conn.close();
            } catch (SQLException e) {
                e.printStackTrace();
            }
        }
    }
}

Giải thích:

  • localhost: máy chủ database (có thể là IP)
  • 3306: port mặc định của MySQL
  • QuanLySinhVien: tên database
  • useSSL=false: tắt SSL (cho đơn giản)
  • serverTimezone=UTC: chỉ định múi giờ

Mục đích:

Statement stmt = conn.createStatement();

2.4 BƯỚC 4: THỰC THI CÂU LỆNH SQL

Mục đích:

Gửi câu lệnh SQL xuống database và nhận kết quả.

Các phương thức thực thi:

Phương thức Dùng cho Trả về
executeQuery(sql) SELECT ResultSet
executeUpdate(sql) INSERT, UPDATE, DELETE int (số hàng bị ảnh hưởng)
execute(sql) Mọi loại SQL boolean (true nếu có ResultSet)

Ví dụ 1: Thực thi SELECT với executeQuery()

Java
String sql = "SELECT * FROM SinhVien";
ResultSet rs = stmt.executeQuery(sql);

Ví dụ 2: Thực thi INSERT với executeUpdate()

Java
String sql = "INSERT INTO SinhVien (HoTen, Email) VALUES ('Nguyen Van A', 'a@gmail.com')";
int rows = stmt.executeUpdate(sql);
System.out.println(rows + " hàng đã được thêm");

Ví dụ 3: Thực thi UPDATE với executeUpdate()

Java
String sql = "UPDATE SinhVien SET DiaChi = 'Hà Nội' WHERE MaSV = 1";
int rows = stmt.executeUpdate(sql);
System.out.println(rows + " hàng đã được cập nhật");

Ví dụ 4: Thực thi DELETE với executeUpdate()

Java
String sql = "DELETE FROM SinhVien WHERE MaSV = 5";
int rows = stmt.executeUpdate(sql);
System.out.println(rows + " hàng đã được xóa");

2.5 BƯỚC 5: XỬ LÝ KẾT QUẢ (RESULTSET)

Mục đích:

Đọc dữ liệu từ ResultSet trả về từ câu lệnh SELECT.

Các phương thức của ResultSet:

Phương thức Mô tả
next() Di chuyển đến hàng tiếp theo, true nếu còn hàng
getString(columnName) Lấy giá trị String theo tên cột
getInt(columnName) Lấy giá trị int theo tên cột
getDouble(columnName) Lấy giá trị double theo tên cột
getDate(columnName) Lấy giá trị Date theo tên cột
getBoolean(columnName) Lấy giá trị boolean theo tên cột
getString(index) Lấy giá trị theo chỉ số cột (bắt đầu từ 1)

Ví dụ:

ResultSetExample.java
import java.sql.*;
import java.util.Date;

public class ResultSetExample {
    public static void main(String[] args) {
        String url = "jdbc:mysql://localhost:3306/QuanLySinhVien?useSSL=false";
        String username = "root";
        String password = "123456";
        
        try (Connection conn = DriverManager.getConnection(url, username, password);
             Statement stmt = conn.createStatement()) {
            
            String sql = "SELECT * FROM SinhVien";
            ResultSet rs = stmt.executeQuery(sql);
            
            System.out.println("=== DANH SÁCH SINH VIÊN ===");
            while (rs.next()) {
                // Lấy theo tên cột
                int maSV = rs.getInt("MaSV");
                String hoTen = rs.getString("HoTen");
                Date ngaySinh = rs.getDate("NgaySinh");
                String gioiTinh = rs.getString("GioiTinh");
                String diaChi = rs.getString("DiaChi");
                String dienThoai = rs.getString("DienThoai");
                String email = rs.getString("Email");
                
                System.out.println("Mã SV: " + maSV);
                System.out.println("Họ tên: " + hoTen);
                System.out.println("Ngày sinh: " + ngaySinh);
                System.out.println("Giới tính: " + gioiTinh);
                System.out.println("Địa chỉ: " + diaChi);
                System.out.println("Điện thoại: " + dienThoai);
                System.out.println("Email: " + email);
                System.out.println("------------------------");
            }
            
        } catch (SQLException e) {
            e.printStackTrace();
        }
    }
}

Ví dụ lấy theo chỉ số cột:

Java
while (rs.next()) {
    int maSV = rs.getInt(1); // Cột 1: MaSV
    String hoTen = rs.getString(2); // Cột 2: HoTen
    // ... các cột khác
}

2.6 BƯỚC 6: ĐÓNG KẾT NỐI

Mục đích:

Giải phóng tài nguyên (kết nối, statement, result set) sau khi sử dụng.

Cú pháp:

Java
rs.close(); // Đóng ResultSet
stmt.close(); // Đóng Statement
conn.close(); // Đóng Connection

Ví dụ với try-catch-finally:

Java
Connection conn = null;
Statement stmt = null;
ResultSet rs = null;

try {
    // Kết nối và thực thi
    conn = DriverManager.getConnection(url, username, password);
    stmt = conn.createStatement();
    rs = stmt.executeQuery(sql);
    
    // Xử lý kết quả
    while (rs.next()) {
        // ...
    }
    
} catch (SQLException e) {
    e.printStackTrace();
} finally {
    // Đóng theo thứ tự ngược lại: rs → stmt → conn
    try { if (rs != null) rs.close(); } catch (SQLException e) {}
    try { if (stmt != null) stmt.close(); } catch (SQLException e) {}
    try { if (conn != null) conn.close(); } catch (SQLException e) {}
}

Ví dụ với try-with-resources (Java 7+):

Java
try (Connection conn = DriverManager.getConnection(url, username, password);
     Statement stmt = conn.createStatement();
     ResultSet rs = stmt.executeQuery(sql)) {
    
    while (rs.next()) {
        // Xử lý kết quả
    }
    
} catch (SQLException e) {
    e.printStackTrace();
}
// Tự động đóng rs, stmt, conn

Loại Interface Mô tả

3.2 STATEMENT

Định nghĩa:

Statement là loại cơ bản nhất, dùng để thực thi câu lệnh SQL tĩnh (không có tham số).

Đặc điểm:

  • SQL được tạo bằng cách nối chuỗi
  • Dễ bị SQL injection
  • Chậm hơn PreparedStatement vì phải biên dịch lại mỗi lần

Ví dụ:

StatementExample.java
public class StatementExample {
    public static void main(String[] args) {

        String username = "root";
        String password = "123456";
        
        try (Connection conn = DriverManager.getConnection(url, username, password);
             Statement stmt = conn.createStatement()) {
            
            // SELECT
            String selectSQL = "SELECT * FROM SinhVien WHERE MaSV = 1";
            ResultSet rs = stmt.executeQuery(selectSQL);
            
            // INSERT
            String insertSQL = "INSERT INTO SinhVien (HoTen, Email) VALUES ('Test', 'test@gmail.com')";
            int rows = stmt.executeUpdate(insertSQL);
            
            // UPDATE
            String updateSQL = "UPDATE SinhVien SET DiaChi = 'HCM' WHERE MaSV = 2";
            rows = stmt.executeUpdate(updateSQL);
            
            // DELETE
            String deleteSQL = "DELETE FROM SinhVien WHERE MaSV = 3";

        } catch (SQLException e) {

        }
    }

3.3 PREPAREDSTATEMENT

Định nghĩa:

PreparedStatement là Statement đã được biên dịch trước, cho phép truyền tham số bằng dấu ?.

Đặc điểm:

  • Chống SQL injection
  • Nhanh hơn khi thực thi nhiều lần
  • SQL rõ ràng, dễ đọc hơn

Cú pháp:

Java
String sql = "INSERT INTO table VALUES (?, ?, ?)";
PreparedStatement pstmt = conn.prepareStatement(sql);
pstmt.setString(1, value1);
pstmt.setInt(2, value2);
pstmt.setDate(3, dateValue);
ResultSet rs = pstmt.executeQuery(); // cho SELECT
int rows = pstmt.executeUpdate(); // cho INSERT, UPDATE, DELETE

3.4 CALLABLESTATEMENT

Định nghĩa:

CallableStatement dùng để gọi stored procedure trong database.

Cú pháp:

Java
// Stored procedure không tham số
CallableStatement cstmt = conn.prepareCall("{call procedure_name}");

// Stored procedure có tham số IN
CallableStatement cstmt = conn.prepareCall("{call procedure_name(?, ?)}");

// Stored procedure có tham số OUT
CallableStatement cstmt = conn.prepareCall("{call procedure_name(?, ?)}");
cstmt.registerOutParameter(2, Types.INTEGER);

Ví dụ: Tạo stored procedure trong MySQL

SQL
-- Stored procedure tính tổng số sinh viên
DELIMITER //
CREATE PROCEDURE sp_count_students(OUT total INT)
BEGIN
    SELECT COUNT(*) INTO total FROM SinhVien;
END //
DELIMITER ;

-- Stored procedure tìm sinh viên theo giới tính
DELIMITER //
CREATE PROCEDURE sp_get_students_by_gender(IN p_gender VARCHAR(10))
BEGIN
    SELECT * FROM SinhVien WHERE GioiTinh = p_gender;
END //
DELIMITER ;

        try { if (stmt != null) stmt.close(); } catch (SQLException e) {}

    }

PHẦN 5: TỔNG KẾT

5.1 CÁC BƯỚC JDBC

Bước Mô tả Code
1 Load driver Class.forName("com.mysql.cj.jdbc.Driver")
2 Tạo kết nối DriverManager.getConnection(url, user, pass)
3 Tạo statement conn.createStatement() hoặc conn.prepareStatement(sql)
4 Thực thi SQL stmt.executeQuery(sql) hoặc stmt.executeUpdate(sql)
5 Xử lý kết quả while(rs.next()) { ... }
6 Đóng kết nối rs.close(); stmt.close(); conn.close();

5.2 SO SÁNH CÁC LOẠI STATEMENT

Loại Cú pháp Khi nào dùng
Statement conn.createStatement() SQL tĩnh, không tham số (không nên dùng)
PreparedStatement conn.prepareStatement(sql) Có tham số, chống SQL injection
CallableStatement conn.prepareCall(sql) Gọi stored procedure

Tóm tắt nhanh:

  • JDBC - Kết nối Java với Database
  • Statement - SQL tĩnh (không nên dùng)
  • PreparedStatement - SQL động, chống SQL injection (nên dùng)
  • CallableStatement - Gọi stored procedure
