PHẦN 1: QUAN HỆ GIỮA CÁC BẢNG (RELATIONSHIPS)

1.1. LÝ THUYẾT VỀ CÁC LOẠI QUAN HỆ

Trong cơ sở dữ liệu quan hệ, có 3 loại quan hệ chính giữa các bảng:

Loại quan hệ Ký hiệu Ý nghĩa Ví dụ thực tế
One-to-One (1-1) 1 ↔ 1 Một bản ghi của bảng A tương ứng với một và chỉ một bản ghi của bảng B Mỗi người có một CMND duy nhất
One-to-Many (1-n) 1 ↔ N Một bản ghi của bảng A tương ứng với nhiều bản ghi của bảng B Một lớp có nhiều học sinh
Many-to-Many (n-n) N ↔ N Nhiều bản ghi của bảng A tương ứng với nhiều bản ghi của bảng B Học sinh học nhiều môn, mỗi môn có nhiều học sinh

1.2. QUAN HỆ 1-1 (ONE-TO-ONE)

1.2.1. Khái niệm

Ý tưởng: Mỗi bản ghi ở bảng A chỉ liên kết với chính xác một bản ghi ở bảng B.

Mục đích:

  • Chia dữ liệu thành 2 bảng để tránh dư thừa.
  • Dữ liệu liên quan nhưng không phải lúc nào cũng cần truy vấn cùng.

Ví dụ thực tế:

  • Mỗi HocSinh có một HoKhau.
  • Mỗi NhanVien có một ThongTinLuong.

1.2.2. Cú pháp và giải thích

Tạo bảng cha (Parent table)
CREATE TABLE BảngCha (
   MaCha INT PRIMARY KEY,
   ThuocTinh1 VARCHAR(50),
   ThuocTinh2 INT
);

Giải thích: PRIMARY KEY: Khóa chính, duy nhất mỗi bản ghi. Bảng cha là bảng chứa thông tin cơ bản, bảng con sẽ liên kết với bảng này.

Tạo bảng phụ (Child table) và liên kết 1-1
CREATE TABLE BảngCon (
   MaCon INT PRIMARY KEY,
   ThuocTinh1 VARCHAR(50),
   ThuocTinh2 DATE,
   MaCha INT UNIQUE,
   FOREIGN KEY (MaCha) REFERENCES BảngCha(MaCha)
);
Thành phần Chức năng
PRIMARY KEY Khóa chính cho bảng con, mỗi bản ghi duy nhất
MaCha INT UNIQUE Mỗi bản ghi trong bảng con chỉ liên kết với 1 bản ghi duy nhất của bảng cha
FOREIGN KEY (MaCha) REFERENCES BảngCha(MaCha) Khóa ngoại, liên kết bảng con → bảng cha
UNIQUE Quan trọng để đảm bảo tính 1-1, không bị nhiều bản ghi con cùng tham chiếu 1 bản ghi cha

1.2.3. Ví dụ chi tiết

QuanLyNhanKhau.sql
-- Tạo cơ sở dữ liệu
CREATE DATABASE QuanLyNhanKhau;
USE QuanLyNhanKhau;

-- Bảng thông tin cơ bản
CREATE TABLE Nguoi (
   MaNguoi INT PRIMARY KEY AUTO_INCREMENT,
   HoTen VARCHAR(100) NOT NULL,
   NgaySinh DATE,
   DiaChi VARCHAR(200),
   SoDienThoai VARCHAR(15)
);

-- Bảng CMND (quan hệ 1-1 với Nguoi)
CREATE TABLE CMND (
   MaCMND INT PRIMARY KEY AUTO_INCREMENT,
   SoCMND VARCHAR(12) UNIQUE NOT NULL,
   NgayCap DATE,
   NoiCap VARCHAR(100),
   MaNguoi INT UNIQUE,
   FOREIGN KEY (MaNguoi) REFERENCES Nguoi(MaNguoi)
);

-- Thêm dữ liệu
INSERT INTO Nguoi (HoTen, NgaySinh, DiaChi, SoDienThoai) VALUES
('Nguyễn Văn A', '1990-01-01', 'Hà Nội', '0123456789'),
('Trần Thị B', '1995-02-02', 'TP.HCM', '0987654321');

INSERT INTO CMND (SoCMND, NgayCap, NoiCap, MaNguoi) VALUES
('001203000001', '2010-01-01', 'Hà Nội', 1),
('001203000002', '2015-02-02', 'TP.HCM', 2);

-- Truy vấn lấy thông tin kết hợp
SELECT n.HoTen, n.NgaySinh, c.SoCMND, c.NgayCap
FROM Nguoi n
INNER JOIN CMND c ON n.MaNguoi = c.MaNguoi;

1.3. QUAN HỆ 1-n (ONE-TO-MANY)

1.3.1. Khái niệm

Ý tưởng: Một bản ghi ở bảng cha liên kết với nhiều bản ghi ở bảng con.

Mục đích:

  • Liên kết dữ liệu logic: ví dụ mỗi lớp có nhiều học sinh, mỗi phòng có nhiều thiết bị.
  • Tránh trùng lặp dữ liệu bảng cha.

Ví dụ thực tế:

  • Lop – HocSinh: mỗi lớp có nhiều học sinh.
  • Phong – ThietBi: mỗi phòng có nhiều thiết bị.

1.3.2. Cú pháp và giải thích

Tạo bảng cha (Parent table)
CREATE TABLE BảngCha (
   MaCha INT PRIMARY KEY,
   TenCha VARCHAR(50)
);

Giải thích: PRIMARY KEY: Mỗi bản ghi trong bảng cha duy nhất. Bảng cha chứa dữ liệu cơ bản, được nhiều bản ghi con tham chiếu.

Tạo bảng con (Child table) và liên kết 1-n
CREATE TABLE BảngCon (
   MaCon INT PRIMARY KEY,
   TenCon VARCHAR(50),
   MaCha INT,
   FOREIGN KEY (MaCha) REFERENCES BảngCha(MaCha)
);
Thành phần Chức năng
PRIMARY KEY Khóa chính bảng con, mỗi bản ghi duy nhất
MaCha INT Khóa ngoại tham chiếu đến bảng cha
FOREIGN KEY (MaCha) REFERENCES BảngCha(MaCha) Bảo đảm giá trị MaCha tồn tại trong bảng cha
Không dùng UNIQUE Vì nhiều bản ghi con có thể cùng tham chiếu một bản ghi cha → đúng 1-n

1.4. QUAN HỆ n-n (MANY-TO-MANY)

1.4.1. Mục đích

Ý tưởng: Nhiều bản ghi ở bảng A có thể liên kết với nhiều bản ghi ở bảng B.

Mục đích:

  • Liên kết các dữ liệu phức tạp: ví dụ học sinh học nhiều môn, mỗi môn có nhiều học sinh.
  • Không lặp dữ liệu: bảng trung gian lưu quan hệ, không cần lặp lại thông tin bảng A hoặc B.

Ví dụ thực tế:

  • HocSinh – MonHoc: học sinh học nhiều môn, mỗi môn có nhiều học sinh.
  • SanPham – DonHang: mỗi đơn hàng có nhiều sản phẩm, mỗi sản phẩm xuất hiện trong nhiều đơn hàng.

1.4.2. Cú pháp và giải thích

Tạo bảng A (Parent table 1)
CREATE TABLE BangA (
   MaA INT PRIMARY KEY,
   TenA VARCHAR(50)
);
Tạo bảng B (Parent table 2)
CREATE TABLE BangB (
   MaB INT PRIMARY KEY,
   TenB VARCHAR(50)
);
Tạo bảng trung gian (Mapping table)
CREATE TABLE BangA_BangB (
   MaA INT,
   MaB INT,
   ThuocTinh FLOAT, -- ví dụ điểm, số lượng...
   PRIMARY KEY (MaA, MaB),
   FOREIGN KEY (MaA) REFERENCES BangA(MaA),
   FOREIGN KEY (MaB) REFERENCES BangB(MaB)
);
Thành phần Chức năng PRIMARY KEY(MaA, MaB) Khóa chính kết hợp → mỗi cặp A-B chỉ xuất hiện 1 lần FOREIGN KEY(MaA) Liên kết bảng trung gian → bảng A FOREIGN KEY(MaB) Liên kết bảng trung gian → bảng B ThuocTinh Lưu thông tin liên quan (ví dụ điểm, số lượng)

Lưu ý: Không cần UNIQUE trên từng cột riêng lẻ, vì nhiều bản ghi A có thể liên kết nhiều B và ngược lại. Bảng trung gian là bắt buộc để thực hiện quan hệ n-n.

1.4.3. Ví dụ chi tiết

QuanLyDiem.sql
-- Tạo cơ sở dữ liệu Quản lý điểm
CREATE DATABASE QuanLyDiem;
USE QuanLyDiem;

-- Bảng 1: HocSinh
CREATE TABLE HocSinh (
   MaHS INT PRIMARY KEY AUTO_INCREMENT,
   TenHS VARCHAR(100) NOT NULL,
   Lop VARCHAR(20)
);

-- Bảng 2: MonHoc
CREATE TABLE MonHoc (
   MaMon INT PRIMARY KEY AUTO_INCREMENT,
   TenMon VARCHAR(50) NOT NULL,
   SoTinChi INT
);

-- Bảng trung gian: BangDiem (liên kết HocSinh và MonHoc)
CREATE TABLE BangDiem (
   MaHS INT,
   MaMon INT,
   Diem FLOAT,
   NgayThi DATE,
   PRIMARY KEY (MaHS, MaMon), -- Khóa hỗn hợp
   FOREIGN KEY (MaHS) REFERENCES HocSinh(MaHS),
   FOREIGN KEY (MaMon) REFERENCES MonHoc(MaMon)
);

-- Chèn dữ liệu vào HocSinh
INSERT INTO HocSinh (TenHS, Lop) VALUES
('Nguyễn Văn A', '10A1'),
('Trần Thị B', '10A1'),
('Lê Văn C', '10A2');

-- Chèn dữ liệu vào MonHoc
INSERT INTO MonHoc (TenMon, SoTinChi) VALUES
('Toán', 4),
('Văn', 4),
('Anh', 3),
('Lý', 3),
('Hóa', 3);

-- Chèn dữ liệu vào BangDiem (quan hệ n-n)
-- Học sinh A học 3 môn: Toán, Văn, Anh
INSERT INTO BangDiem (MaHS, MaMon, Diem, NgayThi) VALUES
(1, 1, 8.5, '2024-01-15'), -- A - Toán
(1, 2, 7.5, '2024-01-16'), -- A - Văn
(1, 3, 9.0, '2024-01-17'); -- A - Anh

-- Học sinh B học 4 môn: Toán, Văn, Lý, Hóa
INSERT INTO BangDiem (MaHS, MaMon, Diem, NgayThi) VALUES
(2, 1, 9.0, '2024-01-15'), -- B - Toán
(2, 2, 8.0, '2024-01-16'), -- B - Văn
(2, 4, 7.5, '2024-01-18'), -- B - Lý
(2, 5, 8.0, '2024-01-19'); -- B - Hóa

-- Học sinh C học 2 môn: Văn, Anh
INSERT INTO BangDiem (MaHS, MaMon, Diem, NgayThi) VALUES
(3, 2, 6.5, '2024-01-16'), -- C - Văn
(3, 3, 7.0, '2024-01-17'); -- C - Anh

-- Truy vấn: Lấy danh sách điểm của từng học sinh
SELECT hs.TenHS, mh.TenMon, bd.Diem
FROM HocSinh hs
INNER JOIN BangDiem bd ON hs.MaHS = bd.MaHS
INNER JOIN MonHoc mh ON bd.MaMon = mh.MaMon
ORDER BY hs.TenHS, mh.TenMon;

1.5. TỔNG KẾT VỀ CÁC LOẠI QUAN HỆ

Mối quan hệ Ý nghĩa Bảng tham gia Cách thiết lập Ví dụ Lưu ý
1-1 (One-to-One) Mỗi bản ghi bảng A liên kết với 1 bản ghi duy nhất bảng B BảngCha → BảngCon BảngCon có MaCha INT UNIQUE, FOREIGN KEY(MaCha) REFERENCES BảngCha(MaCha) HocSinh – HoKhau: mỗi học sinh có 1 hộ khẩu Dùng UNIQUE trên khóa ngoại để đảm bảo 1-1
1-n (One-to-Many) Mỗi bản ghi bảng cha liên kết với nhiều bản ghi bảng con BảngCha → BảngCon BảngCon có MaCha INT, FOREIGN KEY(MaCha) REFERENCES BảngCha(MaCha) Lop – HocSinh: 1 lớp có nhiều học sinh Không dùng UNIQUE trên khóa ngoại vì nhiều bản ghi con liên kết cùng cha
n-n (Many-to-Many) Nhiều bản ghi bảng A liên kết với nhiều bản ghi bảng B BảngA ↔ BảngB qua bảng trung gian Tạo bảng trung gian BangA_BangB có MaA, MaB, PRIMARY KEY(MaA, MaB), FOREIGN KEY(MaA) REFERENCES BangA(MaA), FOREIGN KEY(MaB) REFERENCES BangB(MaB) HocSinh – MonHoc: mỗi học sinh học nhiều môn, mỗi môn có nhiều học sinh Bắt buộc bảng trung gian; khóa chính kết hợp để tránh trùng lặp

PHẦN 2: JOIN TRONG SQL

2.1. LÝ THUYẾT VỀ JOIN

JOIN là phép toán dùng để kết hợp dữ liệu từ hai hoặc nhiều bảng dựa trên một điều kiện liên quan giữa chúng.

Loại JOIN Mô tả Ví dụ minh họa
INNER JOIN Chỉ lấy các dòng có dữ liệu khớp ở cả hai bảng Lấy học sinh có lớp
LEFT JOIN Lấy TẤT CẢ dòng từ bảng bên trái + dòng khớp từ bảng phải Lấy tất cả học sinh (kể cả chưa có lớp)
RIGHT JOIN Lấy TẤT CẢ dòng từ bảng bên phải + dòng khớp từ bảng trái Lấy tất cả lớp (kể cả lớp chưa có học sinh)
CROSS JOIN Tích Descartes - mỗi dòng bảng A ghép với mỗi dòng bảng B Mọi cặp có thể có

2.2. INNER JOIN

2.2.1. Mục đích

Lấy những bản ghi có dữ liệu khớp nhau ở cả hai bảng.

2.2.2. Cú pháp và giải thích

Cú pháp INNER JOIN
-- Cú pháp cơ bản
SELECT column1, column2, ...
FROM table1
INNER JOIN table2 ON table1.key = table2.key;

-- Hoặc viết tắt
SELECT column1, column2, ...
FROM table1
JOIN table2 ON table1.key = table2.key;

Giải thích: INNER JOIN kết hợp hai bảng. ON chỉ định điều kiện kết nối (thường là khóa chính = khóa ngoại). Chỉ lấy những dòng thỏa mãn điều kiện ở ON.

2.2.3. Ví dụ chi tiết

INNER JOIN Example
-- 1. Tạo bảng students
DROP TABLE IF EXISTS students;
CREATE TABLE students (
   student_id INT PRIMARY KEY,
   name VARCHAR(50),
   age INT
);

-- 2. Chèn dữ liệu vào students
INSERT INTO students (student_id, name, age) VALUES
(1, 'Nam', 20),
(2, 'Lan', 21),
(3, 'Hoa', 19),
(4, 'Huy', 22);

-- 3. Tạo bảng scores
DROP TABLE IF EXISTS scores;
CREATE TABLE scores (
   score_id INT PRIMARY KEY,
   student_id INT,
   subject VARCHAR(50),
   score FLOAT,
   FOREIGN KEY (student_id) REFERENCES students(student_id)
);

-- 4. Chèn dữ liệu vào scores
INSERT INTO scores (score_id, student_id, subject, score) VALUES
(1, 1, 'Math', 8),
(2, 1, 'English', 7.5),
(3, 2, 'Math', 9),
(4, 3, 'English', 8),
(5, 5, 'Math', 6); -- student_id 5 không tồn tại trong students

-- 5. INNER JOIN: Lấy danh sách sinh viên kèm điểm
SELECT s.student_id, s.name, sc.subject, sc.score
FROM students s
INNER JOIN scores sc ON s.student_id = sc.student_id;

-- 6. INNER JOIN với điều kiện: chỉ Math >= 8
SELECT s.name, sc.subject, sc.score
FROM students s
INNER JOIN scores sc ON s.student_id = sc.student_id
WHERE sc.subject = 'Math' AND sc.score >= 8;

Giải thích: DROP TABLE IF EXISTS → xóa bảng nếu đã tồn tại, tránh lỗi khi chạy nhiều lần. FOREIGN KEY → đảm bảo student_id trong scores phải có trong students. INNER JOIN → chỉ lấy các cặp khớp giữa students và scores. Truy vấn thứ hai có WHERE để lọc môn Math ≥ 8.

2.3. LEFT JOIN (LEFT OUTER JOIN)

2.3.1. Mục đích

Lấy tất cả bản ghi từ bảng bên trái, kể cả khi không có dữ liệu khớp ở bảng bên phải.

2.3.2. Cú pháp và giải thích

Cú pháp LEFT JOIN
SELECT column1, column2, ...
FROM table1 -- Bảng bên trái
LEFT JOIN table2 ON table1.key = table2.key;
  • table1: bảng bên trái (bảng chính, luôn được giữ lại toàn bộ dữ liệu)
  • table2: bảng bên phải
  • ON table1.key = table2.key: điều kiện nối

Giải thích: Bảng bên trái: Tất cả dòng đều được lấy. Bảng bên phải: Chỉ lấy dòng khớp, nếu không khớp thì giá trị là NULL.

2.3.3. Ví dụ chi tiết

DROP TABLE IF EXISTS students;
CREATE TABLE students (

   name VARCHAR(50),

   FOREIGN KEY (student_id) REFERENCES students(student_id)
);

-- 4. Chèn dữ liệu vào scores

(3, 2, 'Math', 9),

2.4. RIGHT JOIN (RIGHT OUTER JOIN)

2.4.1. Mục đích

Lấy tất cả bản ghi từ bảng bên phải, kể cả khi không có dữ liệu khớp ở bảng bên trái.

2.4.2. Cú pháp và giải thích

Cú pháp RIGHT JOIN
SELECT column1, column2, ...
FROM table1
RIGHT JOIN table2 ON table1.key = table2.key;

Giải thích: Ngược với LEFT JOIN. Thường có thể thay bằng LEFT JOIN bằng cách đổi vị trí bảng.

2.4.3. Ví dụ chi tiết

-- 4. Chèn dữ liệu vào scores
INSERT INTO scores (score_id, student_id, subject, score) VALUES
(1, 1, 'Math', 8),
(2, 1, 'English', 7.5),
(3, 2, 'Math', 9),
(4, 3, 'English', 8),
(5, 5, 'Math', 6); -- student_id 5 không tồn tại trong students

-- 5. RIGHT JOIN: Lấy tất cả điểm kèm thông tin sinh viên nếu có
SELECT s.student_id, s.name, sc.subject, sc.score
FROM students s
RIGHT JOIN scores sc ON s.student_id = sc.student_id;
student_id name subject score
1 Nam Math 8
1 Nam English 7.5
2 Lan Math 9
3 Hoa English 8
NULL NULL Math 6

Giải thích RIGHT JOIN: RIGHT JOIN lấy tất cả bản ghi bên phải (scores), kể cả khi bên trái (students) không có dữ liệu khớp. Nếu không có sinh viên tương ứng → cột student_id và name sẽ là NULL. Sinh viên không có điểm (Huy = 4) bị loại, nhưng điểm không có sinh viên (student_id = 5) vẫn hiển thị.

2.6. BẢNG TỔNG HỢP CÁC LOẠI JOIN

Loại JOIN Lấy dữ liệu từ bảng Điều kiện khớp Dữ liệu không khớp có hiển thị? Ví dụ SQL
INNER JOIN Cả hai bảng ON <cột chung> Không (chỉ lấy các bản ghi khớp) SELECT s.name, sc.subject, sc.score FROM students s INNER JOIN scores sc ON s.student_id = sc.student_id;
LEFT JOIN Bảng bên trái ON <cột chung> Có (bên trái vẫn hiển thị, bên phải NULL nếu không khớp) SELECT s.name, sc.subject, sc.score FROM students s LEFT JOIN scores sc ON s.student_id = sc.student_id;
RIGHT JOIN Bảng bên phải ON <cột chung> Có (bên phải vẫn hiển thị, bên trái NULL nếu không khớp) SELECT s.name, sc.subject, sc.score FROM students s RIGHT JOIN scores sc ON s.student_id = sc.student_id;
FULL OUTER JOIN Cả hai bảng ON <cột chung> Có (lấy tất cả các bản ghi, NULL nếu không khớp) SELECT s.name, sc.subject, sc.score FROM students s LEFT JOIN scores sc ON s.student_id = sc.student_id UNION SELECT s.name, sc.subject, sc.score FROM students s RIGHT JOIN scores sc ON s.student_id = sc.student_id;

PHẦN 3: CÁC LỆNH NÂNG CAO

3.1. GROUP BY và HAVING

3.1.1. Mục đích

  • GROUP BY: Nhóm các dòng có cùng giá trị thành một nhóm
  • HAVING: Lọc kết quả sau khi nhóm (giống WHERE nhưng áp dụng cho GROUP BY)

3.1.2. Cú pháp và giải thích

Cú pháp GROUP BY và HAVING
SELECT column1, aggregate_function(column2)
FROM table
[WHERE condition]
GROUP BY column1
[HAVING condition]
ORDER BY column1;

Giải thích: aggregate_function: SUM(), AVG(), COUNT(), MAX(), MIN(). WHERE: Lọc trước khi nhóm. HAVING: Lọc sau khi nhóm.

3.1.3. Ví dụ chi tiết

GROUP BY và HAVING Examples
-- Sử dụng database QuanLyDiem
USE QuanLyDiem;

-- Ví dụ 1: Đếm số học sinh theo từng lớp
SELECT
   Lop,
   COUNT(*) AS SoHocSinh
FROM HocSinh
GROUP BY Lop;

-- Kết quả:
-- Lop | SoHocSinh
-- 10A1 | 2
-- 10A2 | 1

-- Ví dụ 2: Tính điểm trung bình theo môn học
SELECT
   mh.TenMon,
   COUNT(bd.Diem) AS SoBaiThi,
   ROUND(AVG(bd.Diem), 2) AS DiemTrungBinh,
   MIN(bd.Diem) AS DiemThapNhat,
   MAX(bd.Diem) AS DiemCaoNhat
FROM MonHoc mh
INNER JOIN BangDiem bd ON mh.MaMon = bd.MaMon
GROUP BY mh.TenMon;

-- Ví dụ 3: HAVING - chỉ lấy lớp có số học sinh > 1
SELECT
   Lop,
   COUNT(*) AS SoHocSinh
FROM HocSinh
GROUP BY Lop
HAVING COUNT(*) > 1;

-- Kết quả: Chỉ hiển thị lớp 10A1 (có 2 học sinh)

-- Ví dụ 4: HAVING với nhiều điều kiện
SELECT
   hs.Lop,
   COUNT(bd.Diem) AS SoBaiThi,
   ROUND(AVG(bd.Diem), 2) AS DiemTB
FROM HocSinh hs
INNER JOIN BangDiem bd ON hs.MaHS = bd.MaHS
GROUP BY hs.Lop
HAVING COUNT(bd.Diem) >= 3 AND AVG(bd.Diem) >= 7.5
ORDER BY DiemTB DESC;

-- Ví dụ 5: Phân biệt WHERE và HAVING
-- WHERE lọc trước khi group, HAVING lọc sau khi group
SELECT
   Lop,
   COUNT(*) AS SoHocSinh
FROM HocSinh
WHERE Tuoi >= 15 -- Lọc học sinh từ 15 tuổi trở lên
GROUP BY Lop
HAVING COUNT(*) >= 1; -- Lọc lớp có ít nhất 1 học sinh

3.4. DISTINCT

3.4.1. Mục đích

Loại bỏ các dòng trùng lặp trong kết quả truy vấn.

3.4.2. Cú pháp và giải thích

Cú pháp DISTINCT
SELECT DISTINCT column1, column2, ...
FROM table
[WHERE condition];

Giải thích: DISTINCT áp dụng cho tất cả các cột trong SELECT. Chỉ giữ lại một bản ghi duy nhất cho mỗi tổ hợp giá trị.

3.4.3. Ví dụ chi tiết

DISTINCT Examples
-- Ví dụ 1: Lấy danh sách các lớp (không trùng)
SELECT DISTINCT Lop FROM HocSinh;

-- Kết quả: chỉ hiển thị 10A1, 10A2 (không hiển thị trùng)

-- Ví dụ 2: Lấy danh sách các cặp (Lớp, Giới tính) duy nhất
SELECT DISTINCT Lop, GioiTinh FROM HocSinh;

-- Ví dụ 3: Đếm số lớp khác nhau
SELECT COUNT(DISTINCT Lop) AS SoLop FROM HocSinh;

-- Ví dụ 4: Lấy danh sách học sinh từng thi (không trùng tên)
SELECT DISTINCT hs.HoTen
FROM HocSinh hs
INNER JOIN BangDiem bd ON hs.MaHS = bd.MaHS;

-- Ví dụ 5: DISTINCT với nhiều cột
SELECT DISTINCT
   hs.Lop,
   mh.TenMon
FROM HocSinh hs
INNER JOIN BangDiem bd ON hs.MaHS = bd.MaHS
INNER JOIN MonHoc mh ON bd.MaMon = mh.MaMon
ORDER BY hs.Lop, mh.TenMon;
