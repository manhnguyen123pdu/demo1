PHẦN 1: DATABASE CONCEPTS

1.1 DATABASE LÀ GÌ?

Định nghĩa:

Database (Cơ sở dữ liệu) là một tập hợp có tổ chức của dữ liệu được lưu trữ và truy cập điện tử từ hệ thống máy tính.

Mục đích:

  • Lưu trữ dữ liệu một cách có hệ thống, không bị trùng lặp
  • Truy xuất dữ liệu nhanh chóng khi cần
  • Đảm bảo tính toàn vẹn và nhất quán của dữ liệu
  • Cho phép nhiều người dùng truy cập đồng thời
  • Bảo mật dữ liệu, phân quyền người dùng

Ví dụ thực tế:

  • Hệ thống quản lý sinh viên của trường học: lưu thông tin sinh viên, điểm số, môn học
  • Hệ thống bán hàng của siêu thị: lưu sản phẩm, đơn hàng, khách hàng
  • Hệ thống ngân hàng: lưu tài khoản, giao dịch, khách hàng
  • Hệ thống đặt vé máy bay: lưu chuyến bay, hành khách, vé đặt

1.2 CÁC THÀNH PHẦN CỦA DATABASE

1.2.1 Table (Bảng)

Định nghĩa:

Table là cấu trúc cơ bản nhất để lưu trữ dữ liệu trong cơ sở dữ liệu quan hệ. Nó giống như một bảng tính Excel, gồm các hàng và cột.

Cấu trúc của một bảng:
Bảng: SinhVien
+------------+----------------+-----+------------+
| MaSV (PK) | HoTen | Tuoi| Lop |
+------------+----------------+-----+------------+
| 1 | Nguyễn Văn An | 20 | CNTT1 |
| 2 | Trần Thị Bích | 21 | CNTT2 |
| 3 | Lê Văn Cường | 19 | CNTT1 |
| 4 | Phạm Thị Dung | 22 | CNTT3 |
+------------+----------------+-----+------------+
↑ ↑ ↑ ↑
Khóa chính Cột Cột Cột

Giải thích:

  • Hàng (Row/Record): Mỗi hàng là một bản ghi, đại diện cho một đối tượng cụ thể
  • Cột (Column/Field): Mỗi cột là một thuộc tính của đối tượng
  • Ô (Cell): Giao điểm của hàng và cột, chứa giá trị cụ thể

1.2.2 Primary Key (Khóa chính)

Định nghĩa:

Khóa chính là một cột hoặc tổ hợp các cột dùng để xác định duy nhất mỗi hàng trong bảng.

Đặc điểm:
  • Giá trị duy nhất, không trùng lặp giữa các hàng
  • Không được NULL (bắt buộc phải có giá trị)
  • Mỗi bảng chỉ có một Primary Key
  • Dữ liệu tự động được đánh chỉ mục (index) để tìm nhanh
Ví dụ:
SQL
-- Cách 1: Khai báo khi tạo cột
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY,
    HoTen VARCHAR(100),
    Tuoi INT
);

-- Cách 2: Khai báo riêng ở cuối
CREATE TABLE SinhVien (
    MaSV INT,
    HoTen VARCHAR(100),
    Tuoi INT,
    PRIMARY KEY (MaSV)
);

-- Cách 3: Khóa chính gồm nhiều cột
CREATE TABLE Diem (
    MaSV INT,
    MaMon INT,
    Diem FLOAT,
    PRIMARY KEY (MaSV, MaMon) -- Cả hai cột hợp thành khóa chính
);

Giải thích:

  • MaSV INT PRIMARY KEY: cột MaSV là khóa chính, kiểu số nguyên
  • PRIMARY KEY (MaSV, MaMon): tổ hợp (MaSV + MaMon) là khóa chính

1.2.3 Foreign Key (Khóa ngoại)

Định nghĩa:

Khóa ngoại là một cột tạo liên kết giữa hai bảng, tham chiếu đến khóa chính của bảng khác.

Mục đích:
  • Tạo mối quan hệ giữa các bảng
  • Đảm bảo toàn vẹn dữ liệu (không thể tham chiếu đến dữ liệu không tồn tại)
Ví dụ:
SQL
-- Bảng 1: SinhVien
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY,
    HoTen VARCHAR(100)
);

-- Bảng 2: KetQua (có khóa ngoại tham chiếu đến SinhVien)
CREATE TABLE KetQua (
    MaKetQua INT PRIMARY KEY,
    MaSV INT,
    MonHoc VARCHAR(50),
    Diem FLOAT,
    FOREIGN KEY (MaSV) REFERENCES SinhVien(MaSV)
);

Giải thích:

  • FOREIGN KEY (MaSV): cột MaSV trong bảng KetQua là khóa ngoại
  • REFERENCES SinhVien(MaSV): tham chiếu đến cột MaSV của bảng SinhVien
  • Điều này đảm bảo mọi MaSV trong bảng KetQua phải tồn tại trong bảng SinhVien

1.2.4 Index (Chỉ mục)

Định nghĩa:

Index là cấu trúc dữ liệu đặc biệt giúp tăng tốc độ truy vấn dữ liệu, giống như mục lục của cuốn sách.

Cách hoạt động:
  • Không có index: phải duyệt toàn bộ bảng để tìm (chậm)
  • Có index: tìm trực tiếp qua chỉ mục (nhanh)
Ví dụ:
SQL
-- Tạo index trên cột HoTen
CREATE INDEX idx_hoten ON SinhVien(HoTen);

-- Tạo index trên nhiều cột
CREATE INDEX idx_tuoi_lop ON SinhVien(Tuoi, Lop);

-- Xóa index
DROP INDEX idx_hoten ON SinhVien;

1.3 CÁC MỐI QUAN HỆ TRONG DATABASE

1.3.1 One-to-One (1-1)

Định nghĩa:

Mỗi hàng trong bảng A liên kết với một và chỉ một hàng trong bảng B.

Ví dụ thực tế:
  • Một người có một căn cước công dân
  • Một tài khoản có một thông tin bảo mật
  • Một nhân viên có một hợp đồng lao động
Cách tạo:
SQL
CREATE TABLE Nguoi (
    NguoiID INT PRIMARY KEY,
    HoTen VARCHAR(100)
);

CREATE TABLE CCCD (
    CCCDID INT PRIMARY KEY,
    SoCCCD VARCHAR(20),
    NguoiID INT UNIQUE, -- UNIQUE đảm bảo 1-1
    FOREIGN KEY (NguoiID) REFERENCES Nguoi(NguoiID)
);

1.3.2 One-to-Many (1-N)

Định nghĩa:

Một hàng trong bảng A liên kết với nhiều hàng trong bảng B.

Ví dụ thực tế:
  • Một lớp học có nhiều sinh viên
  • Một khách hàng có nhiều đơn hàng
  • Một tác giả có nhiều sách
Cách tạo:
SQL
-- Bảng 1: Lop (1)
CREATE TABLE Lop (
    MaLop INT PRIMARY KEY,
    TenLop VARCHAR(50)
);

-- Bảng 2: SinhVien (N) - chứa khóa ngoại
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY,
    HoTen VARCHAR(100),
    MaLop INT,
    FOREIGN KEY (MaLop) REFERENCES Lop(MaLop)
);

1.3.3 Many-to-Many (N-N)

Định nghĩa:

Nhiều hàng trong bảng A liên kết với nhiều hàng trong bảng B.

Ví dụ thực tế:
  • Một sinh viên học nhiều môn, một môn có nhiều sinh viên
  • Một sản phẩm thuộc nhiều danh mục, một danh mục có nhiều sản phẩm
  • Một diễn viên đóng nhiều phim, một phim có nhiều diễn viên
Cách tạo:

Cần bảng trung gian (junction table) để kết nối.

SQL
-- Bảng 1: SinhVien
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY,
    HoTen VARCHAR(100)
);

-- Bảng 2: MonHoc
CREATE TABLE MonHoc (
    MaMon INT PRIMARY KEY,
    TenMon VARCHAR(100)
);

-- Bảng trung gian: DangKy
CREATE TABLE DangKy (
    MaSV INT,
    MaMon INT,
    NgayDangKy DATE,
    PRIMARY KEY (MaSV, MaMon), -- Khóa chính là tổ hợp
    FOREIGN KEY (MaSV) REFERENCES SinhVien(MaSV),
    FOREIGN KEY (MaMon) REFERENCES MonHoc(MaMon)
);

PHẦN 2: SQL BASICS

2.1 SQL LÀ GÌ?

Định nghĩa:

SQL (Structured Query Language) là ngôn ngữ chuẩn để giao tiếp với cơ sở dữ liệu quan hệ.

Mục đích:

  • Truy vấn dữ liệu (lấy dữ liệu ra xem)
  • Thêm dữ liệu mới vào database
  • Cập nhật dữ liệu đã có
  • Xóa dữ liệu không cần thiết
  • Tạo và thay đổi cấu trúc database

Phân loại câu lệnh SQL:

Nhóm Tên đầy đủ Chức năng Các lệnh
DDL Data Definition Language Định nghĩa cấu trúc CREATE, ALTER, DROP, TRUNCATE
DML Data Manipulation Language Thao tác dữ liệu SELECT, INSERT, UPDATE, DELETE
DCL Data Control Language Kiểm soát quyền GRANT, REVOKE
TCL Transaction Control Language Quản lý giao dịch COMMIT, ROLLBACK, SAVEPOINT

2.2 CÁC KIỂU DỮ LIỆU TRONG SQL

Mục đích:

Xác định loại dữ liệu có thể lưu trong mỗi cột.

Các kiểu dữ liệu thường dùng:

Kiểu dữ liệu Mô tả Kích thước Ví dụ
INT Số nguyên 4 bytes 1, 100, -25
VARCHAR(n) Chuỗi có độ dài thay đổi n ký tự 'Nguyễn Văn A'
CHAR(n) Chuỗi có độ dài cố định n ký tự 'M', 'F'
DATE Ngày tháng 3 bytes '2024-12-25'
DATETIME Ngày giờ 8 bytes '2024-12-25 14:30:00'
DECIMAL(p,s) Số thập phân p số, s lẻ 99.99
BOOLEAN Đúng/sai 1 byte TRUE, FALSE
TEXT Văn bản dài 65KB-4GB Mô tả sản phẩm
Ví dụ:
SQL
CREATE TABLE SanPham (
    MaSP INT,
    TenSP VARCHAR(100),
    Gia DECIMAL(10,2),
    NgayTao DATE,
    ConHang BOOLEAN
);

2.3 CÁC RÀNG BUỘC (CONSTRAINTS)

Mục đích:

Ràng buộc dùng để giới hạn loại dữ liệu có thể được đưa vào bảng, đảm bảo tính chính xác và toàn vẹn của dữ liệu.

Các ràng buộc thường dùng:

Ràng buộc Mô tả Ví dụ
NOT NULL Không được để trống HoTen VARCHAR(100) NOT NULL
UNIQUE Giá trị không được trùng lặp Email VARCHAR(255) UNIQUE
PRIMARY KEY Khóa chính (NOT NULL + UNIQUE) MaSV INT PRIMARY KEY
FOREIGN KEY Khóa ngoại FOREIGN KEY (MaSV) REFERENCES SinhVien(MaSV)
CHECK Kiểm tra điều kiện Tuoi INT CHECK (Tuoi >= 18)
DEFAULT Giá trị mặc định TrangThai VARCHAR(20) DEFAULT 'active'
Ví dụ tổng hợp:
SQL
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY, -- Khóa chính
    HoTen VARCHAR(100) NOT NULL, -- Không được NULL
    Email VARCHAR(255) UNIQUE, -- Không trùng
    Tuoi INT CHECK (Tuoi >= 18), -- Phải >= 18
    NgayNhapHoc DATE DEFAULT CURRENT_DATE, -- Mặc định là ngày hiện tại
    GioiTinh CHAR(1) DEFAULT 'M' -- Mặc định là 'M'
);

2.4 CÁC LỆNH SQL CƠ BẢN

2.4.1 CREATE DATABASE - Tạo cơ sở dữ liệu

Mục đích:

Tạo một database mới để chứa các bảng.

Cú pháp:
Cú pháp
CREATE DATABASE ten_database;
Ví dụ:
SQL
-- Tạo database QuanLySinhVien

-- Hiển thị danh sách database
SHOW DATABASES;

-- Chọn database để làm việc

2.4.2 CREATE TABLE - Tạo bảng

Mục đích:

Tạo một bảng mới trong database để lưu trữ dữ liệu.

Cú pháp:
Cú pháp
CREATE TABLE ten_bang (
    ten_cot1 kieu_du_lieu rang_buoc,
    ten_cot2 kieu_du_lieu rang_buoc,
    ten_cot3 kieu_du_lieu rang_buoc,
    ...
);
Ví dụ:
SQL
-- Tạo bảng Khoa
CREATE TABLE Khoa (
    MaKhoa INT PRIMARY KEY,
    TenKhoa VARCHAR(100) NOT NULL,
    SoDienThoai VARCHAR(15)
);

-- Tạo bảng SinhVien
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY,
    HoTen VARCHAR(100) NOT NULL,
    NgaySinh DATE,
    GioiTinh CHAR(1) CHECK (GioiTinh IN ('M', 'F')),
    MaKhoa INT,
    Email VARCHAR(255) UNIQUE,
    FOREIGN KEY (MaKhoa) REFERENCES Khoa(MaKhoa)
);

2.4.3 INSERT - Thêm dữ liệu

Mục đích:

Thêm một hoặc nhiều hàng mới vào bảng.

Cú pháp:
Cú pháp
-- Thêm một hàng
INSERT INTO ten_bang (cot1, cot2, cot3, ...)
VALUES (gia_tri1, gia_tri2, gia_tri3, ...);

-- Thêm nhiều hàng
INSERT INTO ten_bang (cot1, cot2, cot3, ...)
VALUES
    (gia_tri1, gia_tri2, gia_tri3, ...),
    (gia_tri1, gia_tri2, gia_tri3, ...),
    (gia_tri1, gia_tri2, gia_tri3, ...);
Ví dụ:
SQL
-- Thêm một khoa
INSERT INTO Khoa (MaKhoa, TenKhoa, SoDienThoai)
VALUES (1, 'Công nghệ thông tin', '0123456789');

-- Thêm nhiều sinh viên
INSERT INTO SinhVien (MaSV, HoTen, NgaySinh, GioiTinh, MaKhoa, Email) VALUES
(1, 'Nguyễn Văn An', '2000-05-15', 'M', 1, 'an@gmail.com'),

(5, 'Hoàng Văn Em', '2001-07-30', 'M', 3, 'em@gmail.com');

2.4.4 SELECT - Truy vấn dữ liệu

Mục đích:

Lấy dữ liệu từ database để xem hoặc xử lý.

Cú pháp cơ bản:
Cú pháp
SELECT cot1, cot2, ...
FROM ten_bang
WHERE dieu_kien
ORDER BY cot ASC|DESC
LIMIT so_luong;
Ví dụ:
SQL
-- Lấy tất cả sinh viên
SELECT * FROM SinhVien;

-- Lấy tên và email sinh viên
SELECT HoTen, Email FROM SinhVien;

-- Lấy sinh viên khoa CNTT
SELECT * FROM SinhVien
WHERE MaKhoa = 1;

-- Lấy sinh viên trên 20 tuổi, sắp xếp theo tên
SELECT * FROM SinhVien
WHERE Tuoi > 20
ORDER BY HoTen ASC;

-- Lấy 5 sinh viên đầu tiên
SELECT * FROM SinhVien
LIMIT 5;

2.4.5 UPDATE - Cập nhật dữ liệu

Mục đích:

Sửa đổi giá trị của các hàng đã có trong bảng.

Cú pháp:
Cú pháp
UPDATE ten_bang
SET cot1 = gia_tri_moi1, cot2 = gia_tri_moi2, ...
WHERE dieu_kien;
Ví dụ:
SQL
-- Sửa tuổi sinh viên có MaSV = 1
UPDATE SinhVien
SET Tuoi = 21
WHERE MaSV = 1;

-- Sửa nhiều cột
UPDATE SinhVien
SET Tuoi = 22, Email = 'an_moi@gmail.com'
WHERE MaSV = 1;

-- Tăng tuổi tất cả sinh viên lên 1 (cẩn thận!)
UPDATE SinhVien
SET Tuoi = Tuoi + 1;

2.4.6 DELETE - Xóa dữ liệu

Mục đích:

Xóa một hoặc nhiều hàng khỏi bảng.

Cú pháp:
Cú pháp
DELETE FROM ten_bang
WHERE dieu_kien;
Ví dụ:
SQL
-- Xóa sinh viên có MaSV = 1
DELETE FROM SinhVien
WHERE MaSV = 1;

-- Xóa sinh viên dưới 18 tuổi
DELETE FROM SinhVien
WHERE Tuoi < 18;

-- Xóa tất cả sinh viên (cẩn thận!)
DELETE FROM SinhVien;

2.4.7 ALTER TABLE - Thay đổi cấu trúc bảng

Mục đích:

Thêm, sửa, xóa cột trong bảng đã có.

Cú pháp:
Cú pháp
-- Thêm cột
ALTER TABLE ten_bang
ADD ten_cot kieu_du_lieu rang_buoc;

-- Xóa cột
ALTER TABLE ten_bang
DROP COLUMN ten_cot;

-- Sửa cột
ALTER TABLE ten_bang
MODIFY COLUMN ten_cot kieu_du_lieu_moi;

-- Đổi tên cột (MySQL)
ALTER TABLE ten_bang
CHANGE COLUMN ten_cu ten_moi kieu_du_lieu;
Ví dụ:
SQL
-- Thêm cột DiaChi
ALTER TABLE SinhVien
ADD DiaChi VARCHAR(200);

-- Xóa cột DiaChi
ALTER TABLE SinhVien
DROP COLUMN DiaChi;

-- Sửa kiểu dữ liệu cột Tuoi
ALTER TABLE SinhVien
MODIFY COLUMN Tuoi INT;

2.4.8 DROP TABLE - Xóa bảng

Mục đích:

Xóa hoàn toàn bảng (cả cấu trúc và dữ liệu).

Cú pháp:
Cú pháp
DROP TABLE ten_bang;
DROP TABLE IF EXISTS ten_bang; -- Chỉ xóa nếu tồn tại
Ví dụ:
SQL
-- Xóa bảng SinhVien
DROP TABLE SinhVien;

-- Xóa nếu tồn tại
DROP TABLE IF EXISTS SinhVien;

2.4.9 TRUNCATE TABLE - Xóa dữ liệu giữ cấu trúc

Mục đích:

Xóa tất cả dữ liệu trong bảng nhưng giữ nguyên cấu trúc.

Cú pháp:
Cú pháp
TRUNCATE TABLE ten_bang;
So sánh DELETE và TRUNCATE:
DELETE TRUNCATE
Xóa từng hàng, chậm Xóa tất cả cùng lúc, nhanh

SQL

PHẦN 3: OPERATORS

3.1 TOÁN TỬ SỐ HỌC

Mục đích:

Thực hiện các phép tính toán học trên dữ liệu số.

Các toán tử:

Toán tử Mô tả Ví dụ Kết quả
+ Cộng SELECT 10 + 3; 13
- Trừ SELECT 10 - 3; 7
* Nhân SELECT 10 * 3; 30
/ Chia SELECT 10 / 3; 3.3333
% Chia lấy dư SELECT 10 % 3; 1
Ví dụ trong bảng:
SQL
-- Tính tổng điểm
SELECT
    MaSV,
    DiemToan,
    DiemLy,
    DiemHoa,
    (DiemToan + DiemLy + DiemHoa) AS TongDiem
FROM BangDiem;

-- Tính điểm trung bình
SELECT
    MaSV,
    (DiemToan + DiemLy + DiemHoa) / 3 AS DiemTB
FROM BangDiem;

-- Tính thuế 10% trên giá sản phẩm
SELECT
    TenSP,
    Gia,
    Gia * 0.1 AS Thue,
    Gia * 1.1 AS GiaSauThue
FROM SanPham;

3.2 TOÁN TỬ SO SÁNH

Mục đích:

So sánh các giá trị để lọc dữ liệu.

Các toán tử:

Toán tử Mô tả Ví dụ
= Bằng WHERE Tuoi = 20
!= hoặc <> Không bằng WHERE Tuoi != 20
> Lớn hơn WHERE Tuoi > 20
<< /span> Nhỏ hơn WHERE Tuoi < 20
>= Lớn hơn hoặc bằng WHERE Tuoi >= 20
<=< /span> Nhỏ hơn hoặc bằng WHERE Tuoi <= 20
Ví dụ:
SQL
-- Sinh viên bằng đúng 20 tuổi
SELECT * FROM SinhVien WHERE Tuoi = 20;

-- Sinh viên không phải 20 tuổi
SELECT * FROM SinhVien WHERE Tuoi != 20;

-- Sinh viên trên 20 tuổi
SELECT * FROM SinhVien WHERE Tuoi > 20;

-- Sinh viên từ 20 tuổi trở lên
SELECT * FROM SinhVien WHERE Tuoi >= 20;

3.3 TOÁN TỬ LOGIC

Mục đích:

Kết hợp nhiều điều kiện với nhau.

3.3.1 AND

Cú pháp:
Cú pháp
WHERE dieu_kien1 AND dieu_kien2

Giải thích:

Cả hai điều kiện phải đúng thì hàng mới được chọn.

Ví dụ:
SQL
-- Sinh viên trên 20 tuổi VÀ học lớp CNTT1
SELECT * FROM SinhVien
WHERE Tuoi > 20 AND Lop = 'CNTT1';

-- Sinh viên từ 18-25 tuổi
SELECT * FROM SinhVien
WHERE Tuoi >= 18 AND Tuoi <= 25;

3.3.2 OR

Cú pháp:
Cú pháp
WHERE dieu_kien1 OR dieu_kien2

Giải thích:

Một trong hai điều kiện đúng là hàng được chọn.

Ví dụ:
SQL
-- Sinh viên học lớp CNTT1 HOẶC CNTT2
SELECT * FROM SinhVien
WHERE Lop = 'CNTT1' OR Lop = 'CNTT2';

3.3.3 NOT

Cú pháp:
Cú pháp
WHERE NOT dieu_kien

Giải thích:

Lấy các hàng không thỏa mãn điều kiện.

Ví dụ:
SQL
-- Sinh viên không học lớp CNTT1
SELECT * FROM SinhVien
WHERE NOT Lop = 'CNTT1';

3.4 TOÁN TỬ ĐẶC BIỆT

3.4.1 BETWEEN ... AND ...

Mục đích:

Lấy giá trị trong một khoảng (bao gồm cả hai đầu).

Cú pháp:
Cú pháp
WHERE cot BETWEEN gia_tri1 AND gia_tri2
Ví dụ:
SQL
-- Sinh viên từ 20 đến 25 tuổi
SELECT * FROM SinhVien
WHERE Tuoi BETWEEN 20 AND 25;

-- Tương đương với:
SELECT * FROM SinhVien
WHERE Tuoi >= 20 AND Tuoi <= 25;

3.4.2 IN

Mục đích:

Lấy giá trị nằm trong một tập hợp.

Cú pháp:
Cú pháp
WHERE cot IN (gia_tri1, gia_tri2, gia_tri3, ...)
Ví dụ:
SQL
-- Sinh viên học lớp CNTT1, CNTT2, hoặc CNTT3
SELECT * FROM SinhVien
WHERE Lop IN ('CNTT1', 'CNTT2', 'CNTT3');

-- Sinh viên có mã 1, 3, 5
SELECT * FROM SinhVien
WHERE MaSV IN (1, 3, 5);

3.4.3 LIKE

Mục đích:

Tìm kiếm theo mẫu, thường dùng cho chuỗi.

Ký tự đại diện:
  • %: đại diện cho 0 hoặc nhiều ký tự
  • _: đại diện cho 1 ký tự bất kỳ
Cú pháp:
Cú pháp
WHERE cot LIKE 'mau'
Ví dụ:
SQL
-- Tên bắt đầu bằng 'Nguyen'
SELECT * FROM SinhVien
WHERE HoTen LIKE 'Nguyen%';

-- Tên kết thúc bằng 'Anh'
SELECT * FROM SinhVien
WHERE HoTen LIKE '%Anh';

-- Tên có chứa 'Van'
SELECT * FROM SinhVien
WHERE HoTen LIKE '%Van%';

-- Email có đuôi @gmail.com
SELECT * FROM SinhVien
WHERE Email LIKE '%@gmail.com';

3.4.4 IS NULL và IS NOT NULL

Mục đích:

Kiểm tra giá trị NULL (không xác định).

Lưu ý:

  • Không thể dùng = NULL vì NULL không phải giá trị
  • Phải dùng IS NULL hoặc IS NOT NULL
Cú pháp:
Cú pháp
WHERE cot IS NULL
WHERE cot IS NOT NULL
Ví dụ:
SQL
-- Sinh viên chưa có email
SELECT * FROM SinhVien
WHERE Email IS NULL;

-- Sinh viên đã có email
SELECT * FROM SinhVien
WHERE Email IS NOT NULL;

PHẦN 4: VÍ DỤ TỔNG HỢP

4.1 Tạo cơ sở dữ liệu quản lý sinh viên

SQL
-- 1. Tạo database
CREATE DATABASE QuanLySinhVien;
USE QuanLySinhVien;

-- 2. Tạo bảng Khoa
CREATE TABLE Khoa (
    MaKhoa INT PRIMARY KEY,
    TenKhoa VARCHAR(100) NOT NULL,
    SoDienThoai VARCHAR(15)
);

-- 3. Tạo bảng SinhVien
CREATE TABLE SinhVien (
    MaSV INT PRIMARY KEY,
    HoTen VARCHAR(100) NOT NULL,
    NgaySinh DATE,
    GioiTinh CHAR(1) CHECK (GioiTinh IN ('M', 'F')),
    MaKhoa INT,
    Email VARCHAR(255) UNIQUE,
    FOREIGN KEY (MaKhoa) REFERENCES Khoa(MaKhoa)
);

-- 4. Tạo bảng MonHoc
CREATE TABLE MonHoc (
    MaMon INT PRIMARY KEY,
    TenMon VARCHAR(100) NOT NULL,
    SoTinChi INT DEFAULT 3
);

-- 5. Tạo bảng KetQua (bảng điểm)
CREATE TABLE KetQua (
    MaSV INT,
    MaMon INT,
    Diem DECIMAL(4,2) CHECK (Diem >= 0 AND Diem <= 10),
    NgayThi DATE,
    PRIMARY KEY (MaSV, MaMon),
    FOREIGN KEY (MaSV) REFERENCES SinhVien(MaSV),
    FOREIGN KEY (MaMon) REFERENCES MonHoc(MaMon)
);

-- 7.4. Lấy điểm của sinh viên, kèm tên môn học
SELECT
    sv.HoTen,
    mh.TenMon,
    kq.Diem
FROM SinhVien sv
JOIN KetQua kq ON sv.MaSV = kq.MaSV
JOIN MonHoc mh ON kq.MaMon = mh.MaMon;

-- 7.5. Tính điểm trung bình của từng sinh viên
SELECT
    sv.MaSV,
    sv.HoTen,
    AVG(kq.Diem) AS DiemTrungBinh
FROM SinhVien sv
LEFT JOIN KetQua kq ON sv.MaSV = kq.MaSV
GROUP BY sv.MaSV, sv.HoTen;

SELECT
    sv.HoTen,
    kq.Diem

WHERE kq.MaMon = 1
ORDER BY kq.Diem DESC
