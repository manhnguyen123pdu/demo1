1. Ví dụ bảng dữ liệu

2. GROUP BY – Nhóm dữ liệu

Tính điểm trung bình mỗi học sinh

SELECT name, AVG(score) AS avg_score FROM diem GROUP BY name;
SELECT name, AVG(score) AS avg_score
FROM diem
GROUP BY name;
nameavg_score
Nam7.75
Lan8.75

Giải thích:

GROUP BY name: gom các dòng cùng tên lại
AVG(score): tính trung bình trong từng nhóm

3. HAVING – Lọc sau khi GROUP

Chỉ lấy học sinh có điểm trung bình > 7

SELECT name, AVG(score) AS avg_score FROM diem GROUP BY name HAVING AVG(score) > 7;
SELECT name, AVG(score) AS avg_score
FROM diem
GROUP BY name
HAVING AVG(score) > 7;
nameavg_score
Nam7.75
Lan8.75
Giải thích:

HAVING dùng để lọc sau khi đã GROUP
Không dùng WHERE vì AVG(score) là hàm tổng hợp

4. So sánh WHERE vs HAVING

WHERE (lọc trước GROUP)

SELECT name, AVG(score) FROM diem WHERE score > 6 GROUP BY name;
SELECT name, AVG(score) FROM diem WHERE score > 6 GROUP BY name;
Lọc từng dòng trước rồi mới tính trung bình

HAVING (lọc sau GROUP)

SELECT name, AVG(score) FROM diem GROUP BY name HAVING AVG(score) > 7;
SELECT name, AVG(score) FROM diem GROUP BY name HAVING AVG(score) > 7;
Tính trung bình xong rồi mới lọc

5. Ví dụ nâng cao (đếm số môn > 1)

Tìm học sinh học hơn 1 môn

SELECT name, COUNT(subject) AS so_mon FROM diem GROUP BY name HAVING COUNT(subject) > 1;
SELECT name, COUNT(subject) AS so_mon FROM diem GROUP BY name HAVING COUNT(subject) > 1;

II. DISTINCT

1. DISTINCT là gì?

DISTINCT dùng để loại bỏ dữ liệu trùng lặp
Nói đơn giản: Chỉ lấy giá trị duy nhất (không lặp lại)

2. Ví dụ bảng dữ liệu

Bảng diem

student_idnamesubject
1NamMath
1NamEnglish
2LanMath
2LanMath
3MinhEnglish

3. Không dùng DISTINCT

SELECT subject FROM diem;
SELECT subject FROM diem;
Kết quả: Math, English, Math, Math, English
Bị lặp

4. Dùng DISTINCT

SELECT DISTINCT subject FROM diem;
SELECT DISTINCT subject FROM diem;
Kết quả: Math, English
Đã loại bỏ trùng

5. DISTINCT với nhiều cột

SELECT DISTINCT name, subject FROM diem;
SELECT DISTINCT name, subject FROM diem;
Kết quả là cặp (name, subject) không trùng nhau

6. DISTINCT + COUNT

Đếm số môn học khác nhau

SELECT COUNT(DISTINCT subject) AS so_mon FROM diem;
SELECT COUNT(DISTINCT subject) AS so_mon FROM diem;

7. DISTINCT vs GROUP BY

DISTINCT
SELECT DISTINCT subject FROM diem;

GROUP BY
SELECT subject FROM diem GROUP BY subject;

Kết quả giống nhau
Nhưng: DISTINCT → dùng để loại trùng, GROUP BY → dùng để gom nhóm + tính toán

8. Khi nào dùng DISTINCT?

  • ✔ Khi cần: Danh sách không trùng (tên, môn, email…)
  • ✔ Lọc dữ liệu sạch
  • ✔ Đếm giá trị khác nhau

9. Ví dụ thực tế

Lấy danh sách học sinh không trùng

SELECT DISTINCT name FROM diem;
SELECT DISTINCT name FROM diem;

III. HÀM SỐ HỌC (AGGREGATE FUNCTIONS)

1. Giới thiệu chung

Năm hàm SUM(), AVG(), COUNT(), MIN(), MAX() được gọi là hàm tổng hợp (aggregate functions). Chúng nhận đầu vào là một tập hợp các giá trị (thường là một cột) và trả về một giá trị duy nhất sau khi tính toán.

Đặc điểm chung:

  • Tất cả các hàm này (trừ COUNT(*)) đều bỏ qua giá trị NULL.
  • Có thể kết hợp với DISTINCT (ví dụ: COUNT(DISTINCT cột)).
  • Thường được sử dụng với GROUP BY để tính toán theo từng nhóm.
  • Không thể dùng trực tiếp trong mệnh đề WHERE (vì WHERE xử lý từng hàng, còn aggregate cần tập hợp).

2. Hàm SUM()

2.1. Công dụng

Tính tổng của các giá trị trong một cột số.

2.2. Cú pháp

SELECT SUM(column_name) FROM table_name;
SELECT SUM(column_name) FROM table_name;

Giải thích: SUM(column_name) → cộng tất cả giá trị trong cột, Bỏ qua giá trị NULL

2.3. Ví dụ chi tiết

Dữ liệu mẫu: Bảng students (điểm thi)

CREATE TABLE students (id INT PRIMARY KEY, name VARCHAR(50), subject VARCHAR(50), score INT);
CREATE TABLE students (
id INT PRIMARY KEY,
name VARCHAR(50),
subject VARCHAR(50),
score INT
);

INSERT INTO students VALUES
(1, 'An', 'Math', 85),
(2, 'An', 'English', 78),
(3, 'An', 'Science', 92),
(4, 'Binh', 'Math', 64),
(5, 'Binh', 'English', 70),
(6, 'Chi', 'Math', 88),
(7, 'Chi', 'Science', NULL); -- Chi nghỉ thi Science, điểm NULL

Ví dụ 1: Tổng điểm của tất cả học sinh (tất cả môn)

SELECT SUM(score) FROM students;
SELECT SUM(score) FROM students;
Kết quả: 85 + 78 + 92 + 64 + 70 + 88 = 477
(NULL của Chi bị bỏ qua, không tính)

Ví dụ 2: Tổng điểm môn Math

SELECT SUM(score) FROM students WHERE subject = 'Math';
SELECT SUM(score) FROM students WHERE subject = 'Math';
Kết quả: 85 + 64 + 88 = 237

Lưu ý: Nếu tất cả giá trị đều NULL, SUM() trả về NULL (không phải 0).

3. Hàm AVG()

3.1. Công dụng

Tính giá trị trung bình của các giá trị trong một cột số.

3.2. Cú pháp

SELECT AVG(column_name) FROM table_name;
SELECT AVG(column_name) FROM table_name;

Giải thích: Tính trung bình cộng, Bỏ qua NULL

3.3. Ví dụ chi tiết

Ví dụ 1: Điểm trung bình của tất cả học sinh (tất cả môn)

SELECT AVG(score) FROM students;
SELECT AVG(score) FROM students;
Kết quả: Tổng 477 / 6 (vì có 6 giá trị không NULL) = 79.5
→ Không phải 477/7 (vì bỏ qua NULL)

Ví dụ 2: Điểm trung bình môn Science

SELECT AVG(score) AS avg_science FROM students WHERE subject = 'Science';
SELECT AVG(score) AS avg_science FROM students WHERE subject = 'Science';
Kết quả: (92) / 1 = 92 (bỏ qua NULL của Chi)

Ví dụ 3: Kết hợp GROUP BY – Điểm trung bình theo từng học sinh

SELECT name, AVG(score) AS avg_score FROM students GROUP BY name;
SELECT name, AVG(score) AS avg_score FROM students GROUP BY name;
nameavg_score
An(85+78+92)/3 = 85.0
Binh(64+70)/2 = 67.0
Chi88 (chỉ có Math, Science NULL)

Lưu ý: AVG() chỉ tính trên các hàng không NULL. Nếu muốn coi NULL là 0, phải dùng AVG(COALESCE(score, 0)).

4. Hàm COUNT()

4.1. Công dụng

Đếm số lượng hàng hoặc số lượng giá trị không NULL trong một cột.

4.2. Các dạng cú pháp

SELECT COUNT(column_name) FROM table_name;
SELECT COUNT(column_name) FROM table_name;

Các cách dùng:
✔ Đếm tất cả dòng: SELECT COUNT(*) FROM students; → Đếm toàn bộ bản ghi (kể cả NULL)
✔ Đếm cột (bỏ NULL): SELECT COUNT(score) FROM students; → Chỉ đếm dòng có giá trị score
✔ Đếm không trùng: SELECT COUNT(DISTINCT subject) FROM students; → Đếm số môn học khác nhau

4.3. Ví dụ chi tiết

Ví dụ 1: COUNT(*) – Tổng số sinh viên (mỗi hàng là một bài thi)

SELECT COUNT(*) FROM students;
SELECT COUNT(*) FROM students;
Kết quả: 7 (có 7 hàng, kể cả hàng có score = NULL)

Ví dụ 2: COUNT(cột) – Số bài thi có điểm (không NULL)

SELECT COUNT(score) FROM students;
SELECT COUNT(score) FROM students;
Kết quả: 6 (bỏ qua hàng score NULL của Chi môn Science)

Ví dụ 3: COUNT(DISTINCT) – Số môn học khác nhau

SELECT COUNT(DISTINCT subject) FROM students;
SELECT COUNT(DISTINCT subject) FROM students; -- Kết quả: 3 (Math, English, Science)

Ví dụ 4: Đếm số học sinh có điểm > 80

SELECT COUNT(*) FROM students WHERE score > 80;
SELECT COUNT(*) FROM students WHERE score > 80;
Kết quả: 4 (An Math 85, An Science 92, Chi Math 88)

Ví dụ 5: Kết hợp GROUP BY – Số môn đã thi của mỗi học sinh

SELECT name, COUNT(subject) AS subjects_taken FROM students GROUP BY name;
SELECT name, COUNT(subject) AS subjects_taken FROM students GROUP BY name;
namesubjects_taken
An3 (cả 3 môn)
Binh2
Chi1 (Science NULL nên không đếm)

Lưu ý: COUNT(*) nhanh hơn COUNT(cột) nếu bạn chỉ cần đếm số hàng, vì không cần kiểm tra NULL.

5. Hàm MIN() và MAX()

5.1. Công dụng

MIN(): Giá trị nhỏ nhất trong tập hợp. MAX(): Giá trị lớn nhất trong tập hợp.

5.2. Cú pháp

SELECT MIN(column_name) FROM table_name; SELECT MAX(column_name) FROM table_name;
SELECT MIN(column_name) FROM table_name;
SELECT MAX(column_name) FROM table_name;

Có thể dùng với kiểu số, ngày tháng, chuỗi (theo thứ tự từ điển).

5.3. Ví dụ chi tiết

Ví dụ 1: Điểm cao nhất và thấp nhất của tất cả học sinh

SELECT MIN(score), MAX(score) FROM students;
SELECT MIN(score), MAX(score) FROM students;
Kết quả: min = 64, max = 92 (NULL bị bỏ qua)

Ví dụ 2: Với kiểu chuỗi – Tên đầu tiên và cuối cùng theo bảng chữ cái

SELECT MIN(name), MAX(name) FROM students;
SELECT MIN(name), MAX(name) FROM students;
Kết quả: first_name = 'An', last_name = 'Chi' (dựa trên thứ tự từ điển)

7. Sử dụng với GROUP BY và HAVING

Bài toán: Tính tổng điểm, điểm trung bình của mỗi học sinh, chỉ lấy những học sinh có tổng điểm > 150.

SELECT name, SUM(score) AS total, AVG(score) AS average, COUNT(*) AS subjects_taken, MIN(score) AS lowest, MAX(score) AS highest FROM students GROUP BY name HAVING total > 150;
SELECT
name,
SUM(score) AS total,
AVG(score) AS average,
COUNT(*) AS subjects_taken,
MIN(score) AS lowest,
MAX(score) AS highest
FROM students
GROUP BY name
HAVING total > 150;

Kết quả: Chỉ An (tổng 255) thỏa mãn.

8. Các lỗi thường gặp và mẹo

Lỗi 1: Dùng hàm tổng hợp trong WHERE
-- SAI: Không được dùng AVG trong WHERE
SELECT * FROM students WHERE score > AVG(score);

-- ĐÚNG: Dùng truy vấn con
SELECT * FROM students WHERE score > (SELECT AVG(score) FROM students);
Lỗi 2: Quên GROUP BY khi kết hợp cột thường và hàm tổng hợp
-- SAI (nếu chế độ ONLY_FULL_GROUP_BY bật):
SELECT name, AVG(score) FROM students;

-- ĐÚNG:
SELECT name, AVG(score) FROM students GROUP BY name;
Lỗi 3: Hiểu nhầm COUNT(*) và COUNT(cột)
-- COUNT(*) = 7 (kể cả dòng NULL)
-- COUNT(score) = 6 (bỏ qua NULL)
Mẹo 1: Xử lý NULL thành 0 khi tính AVG
SELECT AVG(COALESCE(score, 0)) FROM students; -- (477+0)/7 = 68.14
Mẹo 2: Đếm tỷ lệ phần trăm
-- Tỷ lệ học sinh có điểm >= 80
SELECT (COUNT(CASE WHEN score >= 80 THEN 1 END) * 100.0 / COUNT(*)) AS percent_above_80 FROM students;

IV. HÀM XỬ LÝ CHUỖI (STRING FUNCTIONS)

1. Giới thiệu chung

Hàm xử lý chuỗi (string functions) trong MySQL cho phép thao tác trên dữ liệu dạng văn bản: nối chuỗi, thay đổi chữ hoa/thường, cắt, loại bỏ khoảng trắng, tìm kiếm, thay thế,...

Đặc điểm chung: Hầu hết các hàm chuỗi đều trả về kết quả là một chuỗi mới (hoặc số với LENGTH). Nếu đầu vào là NULL, hầu hết các hàm đều trả về NULL (trừ CONCAT có thể có ngoại lệ). Có thể lồng các hàm với nhau (ví dụ: UPPER(TRIM(' hello '))).

2. Hàm CONCAT() – Nối chuỗi

2.1. Công dụng

Nối (ghép) hai hoặc nhiều chuỗi lại với nhau thành một chuỗi duy nhất.

2.2. Cú pháp

CONCAT(chuỗi1, chuỗi2, ...) CONCAT_WS(dấu_phân_cách, chuỗi1, chuỗi2, ...)
CONCAT(chuỗi1, chuỗi2, ...)
CONCAT_WS(dấu_phân_cách, chuỗi1, chuỗi2, ...) -- CONCAT With Separator

2.3. Ví dụ chi tiết

Ví dụ 1: Nối chuỗi đơn giản

SELECT CONCAT('Xin chào ', 'Anh') AS greeting;
SELECT CONCAT('Xin chào ', 'Anh') AS greeting; -- Kết quả: 'Xin chào Anh'

Ví dụ 2: Nối nhiều hơn 2 chuỗi

SELECT CONCAT('MySQL', ' ', 'is', ' ', 'powerful') AS message;
SELECT CONCAT('MySQL', ' ', 'is', ' ', 'powerful') AS message; -- 'MySQL is powerful'

Ví dụ 3: Nối với cột trong bảng

SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM users;
SELECT CONCAT(first_name, ' ', last_name) AS full_name FROM users;

Ví dụ 4: CONCAT với số và ngày tháng (tự động chuyển đổi)

SELECT CONCAT('Order #', 1001, ' created on ', CURDATE()) AS order_info;
SELECT CONCAT('Order #', 1001, ' created on ', CURDATE()) AS order_info;

Ví dụ 5: Xử lý NULL – CONCAT trả về NULL nếu có bất kỳ tham số nào là NULL

SELECT CONCAT('Hello', NULL, 'World') AS result;
SELECT CONCAT('Hello', NULL, 'World') AS result; -- Kết quả: NULL

Khắc phục: Dùng CONCAT_WS() hoặc COALESCE()

SELECT CONCAT_WS(' ', 'Hello', NULL, 'World') AS result;
SELECT CONCAT_WS(' ', 'Hello', NULL, 'World') AS result; -- 'Hello World'

Ví dụ 6: Ứng dụng thực tế – Tạo địa chỉ email từ tên

SELECT CONCAT(LOWER(first_name), '.', LOWER(last_name), '@company.com') AS email FROM users;
SELECT CONCAT(LOWER(first_name), '.', LOWER(last_name), '@company.com') AS email FROM users;

3. Hàm LENGTH() – Độ dài chuỗi

3.1. Công dụng

Trả về độ dài của chuỗi theo số byte. Đối với chuỗi UTF-8 (tiếng Việt có dấu), mỗi ký tự có thể chiếm 2-3 byte.

Lưu ý: Để đếm số ký tự (không phải byte), dùng CHAR_LENGTH() hoặc CHARACTER_LENGTH().

3.2. Cú pháp

LENGTH(chuỗi) CHAR_LENGTH(chuỗi)
LENGTH(chuỗi)
CHAR_LENGTH(chuỗi) -- đếm số ký tự

3.3. Ví dụ chi tiết

Ví dụ 1: Chuỗi tiếng Anh (1 ký tự = 1 byte)

SELECT LENGTH('Hello') AS byte_length;
SELECT LENGTH('Hello') AS byte_length; -- Kết quả: 5

Ví dụ 2: Chuỗi tiếng Việt có dấu

SELECT LENGTH('Xin chào') AS byte_length, CHAR_LENGTH('Xin chào') AS char_length;
SELECT LENGTH('Xin chào') AS byte_length, CHAR_LENGTH('Xin chào') AS char_length;

Ví dụ 3: Kiểm tra dữ liệu bị rỗng hoặc chỉ toàn khoảng trắng

SELECT * FROM users WHERE LENGTH(TRIM(first_name)) = 0;
SELECT * FROM users WHERE LENGTH(TRIM(first_name)) = 0;

Ví dụ 4: Ứng dụng – Kiểm tra độ dài mật khẩu tối thiểu

SELECT username FROM accounts WHERE LENGTH(password) < 8;
SELECT username FROM accounts WHERE LENGTH(password) < 8;

4. Hàm UPPER() và LOWER()

4.1. Công dụng

UPPER(): Chuyển đổi toàn bộ chuỗi thành chữ hoa. LOWER(): Chuyển đổi toàn bộ chuỗi thành chữ thường.

4.2. Cú pháp

UPPER(chuỗi) LOWER(chuỗi)
UPPER(chuỗi)
LOWER(chuỗi)

4.3. Ví dụ chi tiết

Ví dụ 1: Chuyển đổi cơ bản

SELECT UPPER('hello') AS upper_case, LOWER('HELLO') AS lower_case;
SELECT UPPER('hello') AS upper_case, LOWER('HELLO') AS lower_case;
Kết quả: upper_case = 'HELLO', lower_case = 'hello'

Ví dụ 2: Xử lý tiếng Việt (có dấu)

SELECT UPPER('Xin chào Việt Nam') AS vietnamese_upper;
SELECT UPPER('Xin chào Việt Nam') AS vietnamese_upper;
Kết quả: 'XIN CHÀO VIỆT NAM'

Ví dụ 3: Chuẩn hóa dữ liệu trước khi so sánh (không phân biệt chữ hoa/thường)

SELECT * FROM users WHERE UPPER(email) = UPPER('vana@example.com');
SELECT * FROM users WHERE UPPER(email) = UPPER('vana@example.com');

Ví dụ 4: Ứng dụng – Tạo mã code tự động

SELECT CONCAT(UPPER(SUBSTRING(product_name, 1, 3)), LPAD(id, 4, '0')) AS product_code FROM products;
SELECT CONCAT(UPPER(SUBSTRING(product_name, 1, 3)), LPAD(id, 4, '0')) AS product_code FROM products;

Ví dụ 5: Định dạng tên riêng (viết hoa chữ cái đầu)

SELECT CONCAT(UPPER(SUBSTRING(first_name, 1, 1)), LOWER(SUBSTRING(first_name, 2))) AS proper_name FROM users;
SELECT CONCAT(UPPER(SUBSTRING(first_name, 1, 1)), LOWER(SUBSTRING(first_name, 2))) AS proper_name FROM users;

5. Hàm SUBSTRING() – Cắt chuỗi

5.1. Công dụng

Trích xuất một phần của chuỗi (substring), bắt đầu từ vị trí xác định và có độ dài tùy chọn.

5.2. Cú pháp

SUBSTRING(chuỗi, vị_trí_bắt_đầu, [độ_dài])
SUBSTRING(chuỗi, vị_trí_bắt_đầu, [độ_dài])
SUBSTR(chuỗi, vị_trí_bắt_đầu, [độ_dài]) -- dạng viết tắt

vị_trí_bắt_đầu: bắt đầu từ 1 (không phải 0). Nếu là số âm, tính từ cuối chuỗi.
độ_dài: (tùy chọn) số ký tự cần lấy. Nếu bỏ qua, lấy từ vị trí bắt đầu đến hết.

5.3. Ví dụ chi tiết

Ví dụ 1: Cắt từ vị trí đầu

SELECT SUBSTRING('Hello World', 2, 3) AS substring_result;
SELECT SUBSTRING('Hello World', 2, 3) AS substring_result;
Kết quả: 'ell'

Ví dụ 2: Bỏ qua tham số độ dài (lấy đến hết)

SELECT SUBSTRING('Hello World', 7) AS substring_result;
SELECT SUBSTRING('Hello World', 7) AS substring_result;
Kết quả: 'World'

Ví dụ 3: Sử dụng vị trí âm (đếm từ cuối lên)

SELECT SUBSTRING('MySQL', -2) AS last_two;
SELECT SUBSTRING('MySQL', -2) AS last_two;
Kết quả: 'QL'
SELECT SUBSTRING('MySQL', -3, 2) AS result;
SELECT SUBSTRING('MySQL', -3, 2) AS result;
Kết quả: 'SQ'

Ví dụ 4: Ứng dụng – Lấy tên miền từ email

SELECT email, SUBSTRING(email, LOCATE('@', email) + 1) AS domain FROM users;
SELECT email, SUBSTRING(email, LOCATE('@', email) + 1) AS domain FROM users;

Lỗi 1: So sánh DATETIME với DATE không chính xác
-- SAI: created_at = '2024-01-15' sẽ không khớp với '2024-01-15 10:30:00'
SELECT * FROM orders WHERE created_at = '2024-01-15';

-- ĐÚNG: Chuyển về DATE để so sánh
SELECT * FROM orders WHERE DATE(created_at) = '2024-01-15';
-- HOẶC: Dùng khoảng
SELECT * FROM orders WHERE created_at >= '2024-01-15' AND created_at < '2024-01-16';
Lỗi 2: Cộng trừ tháng vào ngày 31
-- Kết quả bất ngờ: 2024-01-31 + 1 tháng = 2024-02-29 (không phải 2024-03-02)
SELECT DATE_ADD('2024-01-31', INTERVAL 1 MONTH); -- 2024-02-29
Mẹo 1: Lấy đầu tháng, cuối tháng
-- Đầu tháng hiện tại
SELECT DATE_SUB(CURDATE(), INTERVAL DAY(CURDATE())-1 DAY);

-- Cuối tháng hiện tại
SELECT LAST_DAY(CURDATE());
Mẹo 2: Lấy tuần trước – tuần này
-- Đầu tuần (thứ 2)
SELECT DATE_SUB(CURDATE(), INTERVAL WEEKDAY(CURDATE()) DAY);

-- Cuối tuần (chủ nhật)
SELECT DATE_ADD(CURDATE(), INTERVAL (6 - WEEKDAY(CURDATE())) DAY);
