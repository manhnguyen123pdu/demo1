PHẦN 1: GIỚI THIỆU MYSQL

1.1. MySQL là gì?

MySQL là một hệ quản trị cơ sở dữ liệu quan hệ (Relational Database Management System – RDBMS).

  • Dữ liệu được lưu trữ trong các bảng (tables), mỗi bảng gồm cột (columns) và hàng (rows).
  • MySQL sử dụng ngôn ngữ SQL (Structured Query Language) để thao tác dữ liệu.
  • Đây là phần mềm mã nguồn mở (open-source), miễn phí, nhưng cũng có phiên bản thương mại do Oracle phát triển.
  • Được dùng phổ biến trong:
    • Website (WordPress, Laravel, PHP…)
    • Ứng dụng doanh nghiệp
    • Game server
    • Hệ thống phân tích dữ liệu

1.2. Đặc điểm chính

Đặc điểm Mô tả
Mã nguồn mở Có thể sử dụng miễn phí, tùy chỉnh mã nguồn nếu cần.
Hiệu năng cao Xử lý nhanh, ổn định với dữ liệu lớn.
Tính tương thích Hoạt động trên nhiều hệ điều hành: Windows, Linux, macOS.
Hỗ trợ đa ngôn ngữ lập trình PHP, Java, Python, C#, Node.js…
Bảo mật Hỗ trợ xác thực người dùng, phân quyền, mã hóa.

1.3. Kiến trúc MySQL

MySQL hoạt động theo mô hình Client - Server:

  • MySQL Server:
    • Chạy nền (background) trên máy chủ.
    • Quản lý dữ liệu, xử lý câu lệnh SQL.
  • MySQL Client:
    • Có thể là MySQL CLI (dòng lệnh) hoặc ứng dụng như MySQL Workbench, PHPMyAdmin.
    • Gửi câu lệnh SQL đến server để thực thi.

1.4. Cú pháp SQL cơ bản

  • SQL (Structured Query Language) là ngôn ngữ để giao tiếp với MySQL.
  • Một câu lệnh SQL có dạng: LỆNH SQL từ khóa [tùy chọn] ...;
Ví dụ
SELECT * FROM customers;

Giải thích:

  • SELECT: Lệnh lấy dữ liệu.
  • *: Lấy tất cả các cột.
  • FROM customers: Lấy từ bảng customers.
  • ;: Kết thúc câu lệnh.

1.5. Phân loại lệnh SQL

Nhóm lệnh Mục đích Ví dụ
DDL (Data Definition Language) Định nghĩa cấu trúc dữ liệu CREATE, ALTER, DROP
DML (Data Manipulation Language) Thao tác dữ liệu INSERT, UPDATE, DELETE
DQL (Data Query Language) Truy vấn dữ liệu SELECT
DCL (Data Control Language) Phân quyền GRANT, REVOKE
TCL (Transaction Control Language) Quản lý giao dịch COMMIT, ROLLBACK

PHẦN 2: CÀI ĐẶT & KẾT NỐI MYSQL

1.1. Cài MySQL trực tiếp vào máy

Windows

  • Vào trang tải MySQL: https://dev.mysql.com/downloads/.
  • Tải MySQL Installer bản .msi.
  • Mở file cài → Chọn Developer Default (bao gồm MySQL Server + MySQL Workbench).
  • Chọn Next → Cài đặt.
  • Đặt mật khẩu cho user root.
  • Hoàn tất cài đặt.
  • Sau khi cài: Mở Command Prompt: mysql -u root -p → Nhập mật khẩu đã đặt.

1.2. Cài MySQL bằng Docker

Nhanh, không ảnh hưởng hệ thống

Bước 1 – Kéo image MySQL trên cmd

Docker pull
docker pull mysql:latest

Bước 2 – Tạo container MySQL

Docker run
docker run --name mysql-server \ -e MYSQL_ROOT_PASSWORD=1234 \ -p 3306:3306 \ -d mysql:latest

Giải thích:

  • --name mysql-server → Tên container.
  • -e MYSQL_ROOT_PASSWORD=1234 → Mật khẩu root là 1234.
  • -p 3306:3306 → Mở cổng MySQL ra ngoài.
  • -d → Chạy nền.
  • mysql:latest → Image MySQL mới nhất.

Nếu muốn bản Oracle Linux 9:

Docker run with Oracle Linux 9
docker run --name mysql-server \ -e MYSQL_ROOT_PASSWORD=1234 \ -p 3306:3306 \ -d mysql:oraclelinux9

1.3. Cài MySQL Workbench

MySQL Workbench là phần mềm giao diện để quản lý database trực quan.

  • Windows / macOS: Khi cài MySQL Installer, chọn luôn MySQL Workbench.
  • Hoặc tải riêng tại: https://dev.mysql.com/downloads/workbench/

Mở Workbench và kết nối

  • Mở MySQL Workbench.
  • Nhấn + để tạo kết nối mới.
  • Nhập thông tin:
    • Connection Name: MySQL Local
    • Hostname: 127.0.0.1
    • Port: 3306
    • Username: root
    • Password: nhập mật khẩu root
  • Nhấn Test Connection → Nếu OK thì Save.

PHẦN 3: QUẢN LÝ DATABASE

Trong MySQL, Database (Cơ sở dữ liệu) là nơi chứa các bảng (tables), và mỗi bảng sẽ chứa dữ liệu theo cấu trúc hàng (rows) và cột (columns).

Việc quản lý database bao gồm: Tạo mới (CREATE DATABASE), Xem danh sách (SHOW DATABASES), Chọn database để làm việc (USE), Xóa database (DROP DATABASE).

3.1. Tạo Database

Cú pháp: CREATE DATABASE database_name;

Giải thích: CREATE DATABASE → Lệnh tạo mới cơ sở dữ liệu. database_name → Tên cơ sở dữ liệu (không dấu cách, thường dùng chữ thường hoặc gạch dưới _).

Quy tắc đặt tên: Chỉ gồm chữ, số, dấu gạch dưới _. Không trùng với tên database đã có. Không dùng ký tự đặc biệt.

Ví dụ
CREATE DATABASE cre;

→ Tạo database tên cre.

3.2. Xem danh sách Database

Cú pháp: SHOW DATABASES;

Giải thích: SHOW DATABASES → Liệt kê tất cả database hiện có trên server MySQL.

Ví dụ
SHOW DATABASES;

Kết quả mẫu:

Database
information_schema
mysql
performance_schema
sys
cre

3.3. Chọn Database để làm việc

Cú pháp: USE database_name;

Giải thích: USE → Lệnh chọn database. Sau khi chọn, tất cả các thao tác (tạo bảng, thêm dữ liệu…) sẽ áp dụng trên database đó.

Ví dụ
USE cre;

→ Chọn cre làm database hiện tại.

3.4. Xóa Database

Cú pháp: DROP DATABASE database_name;

Giải thích: DROP DATABASE → Xóa toàn bộ database và dữ liệu bên trong. Cẩn thận: Lệnh này không thể hoàn tác.

Ví dụ
DROP DATABASE cre;

→ Xóa database cre.

3.5. Các lệnh đầy đủ minh họa

SQL Commands
-- 1. Tạo database
CREATE DATABASE cre;

-- 2. Xem danh sách database
SHOW DATABASES;

-- 3. Chọn database để làm việc
USE cre;

-- 4. Xóa database
DROP DATABASE cre;

3.6. Ghi chú quan trọng

Lệnh Mô tả
CREATE DATABASE Tạo cơ sở dữ liệu mới
SHOW DATABASES Liệt kê tất cả cơ sở dữ liệu
USE Chọn cơ sở dữ liệu hiện tại
DROP DATABASE Xóa cơ sở dữ liệu

PHẦN 4: QUẢN LÝ TABLE

4.1. Mục đích

Bảng (Table) trong MySQL là nơi lưu trữ dữ liệu của một ứng dụng, hệ thống hoặc website.

Mỗi bảng sẽ chứa nhiều cột (columns) để mô tả các thuộc tính của dữ liệu và nhiều hàng (rows) chứa dữ liệu thực tế.

Tạo bảng (CREATE TABLE) thường được thực hiện khi:

  • Bắt đầu một dự án mới (tạo bảng users để lưu thông tin người dùng).
  • Thêm chức năng mới cần bảng dữ liệu riêng.
  • Chuẩn bị cơ sở dữ liệu cho hệ thống quản lý, bán hàng, học sinh, nhân viên…

4.1. Tạo Table

Cú pháp:

CREATE TABLE Syntax
CREATE TABLE table_name (
   column1 datatype constraints,
   column2 datatype constraints,
   ...
);

Giải thích:

  • CREATE TABLE → Lệnh tạo bảng mới.
  • table_name → Tên bảng (nên viết thường, không dấu, dùng _ để phân cách).
  • column1, column2 → Tên các cột trong bảng.
  • datatype → Kiểu dữ liệu (INT, VARCHAR, DATE…).
  • constraints → Ràng buộc dữ liệu (PRIMARY KEY, NOT NULL, DEFAULT…).

Các kiểu dữ liệu MySQL thường dùng

Nhóm kiểu dữ liệu Tên kiểu Mô tả
Số nguyên TINYINT Số nguyên rất nhỏ (-128 → 127)
SMALLINT Số nguyên nhỏ (-32,768 → 32,767)
INT / INTEGER Số nguyên thường dùng
BIGINT Số nguyên rất lớn
Số thực FLOAT Số thực chính xác thấp (4 byte)
DOUBLE / REAL Số thực chính xác cao (8 byte)
DECIMAL(m, d) Số thập phân chính xác tuyệt đối (dùng cho tiền tệ)
Chuỗi ký tự CHAR(n) Chuỗi cố định độ dài n (tối đa 255 ký tự)
VARCHAR(n) Chuỗi biến độ dài, tối đa n (tối đa 65535 ký tự)
TEXT Chuỗi văn bản dài (~65,535 ký tự)
Ngày & Thời gian DATE Ngày (YYYY-MM-DD)
TIME Giờ, phút, giây (HH:MM:SS)
DATETIME Ngày & giờ
TIMESTAMP Ngày & giờ (Unix timestamp)
Boolean BOOLEAN / BOOL Giá trị TRUE/FALSE (lưu như TINYINT(1))

Constraints – Ràng buộc dữ liệu trong MySQL

Định nghĩa: Constraints là các quy tắc áp dụng trên cột hoặc bảng để đảm bảo tính toàn vẹn và hợp lệ của dữ liệu.

Constraint Mục đích Ví dụ
PRIMARY KEY Định danh duy nhất mỗi bản ghi, không trùng và không NULL. Tự động tạo chỉ mục giúp truy vấn nhanh hơn. id INT AUTO_INCREMENT PRIMARY KEY
FOREIGN KEY Liên kết dữ liệu giữa các bảng, đảm bảo giá trị cột phải tồn tại ở bảng khác. FOREIGN KEY (user_id) REFERENCES users(id)
NOT NULL Bắt buộc cột phải có giá trị, không được để trống. name VARCHAR(50) NOT NULL
UNIQUE Đảm bảo giá trị trong cột là duy nhất. email VARCHAR(100) UNIQUE
DEFAULT Gán giá trị mặc định nếu không nhập giá trị. status VARCHAR(10) DEFAULT 'active'
CHECK Giới hạn giá trị cột theo điều kiện. age INT CHECK (age >= 18)
Ví dụ CREATE TABLE users
CREATE TABLE users (
   id INT AUTO_INCREMENT PRIMARY KEY, -- Khóa chính, tự tăng
   name VARCHAR(50) NOT NULL, -- Không được để trống
   email VARCHAR(100) UNIQUE, -- Giá trị duy nhất
   age INT CHECK (age >= 18), -- Tuổi phải >= 18
   status VARCHAR(10) DEFAULT 'active', -- Giá trị mặc định
   created_date TIMESTAMP DEFAULT NOW() -- Thời gian mặc định
);

Giải thích chi tiết từng phần:

  • CREATE TABLE users → Lệnh tạo bảng tên users.
  • id INT AUTO_INCREMENT PRIMARY KEY → id: Tên cột. INT: Kiểu số nguyên. AUTO_INCREMENT: Tự động tăng giá trị mỗi bản ghi mới. PRIMARY KEY: Khóa chính, duy nhất, không được NULL.
  • name VARCHAR(50) NOT NULL → VARCHAR(50): Chuỗi tối đa 50 ký tự. NOT NULL: Không được để trống.
  • email VARCHAR(100) UNIQUE → VARCHAR(100): Chuỗi tối đa 100 ký tự. UNIQUE: Giá trị không được trùng lặp giữa các bản ghi.
  • age INT CHECK (age >= 18) → INT: Kiểu số nguyên. CHECK (age >= 18): Ràng buộc, chỉ chấp nhận tuổi từ 18 trở lên.
  • status VARCHAR(10) DEFAULT 'active' → VARCHAR(10): Chuỗi tối đa 10 ký tự. DEFAULT 'active': Nếu không nhập, mặc định là "active".
  • created_date TIMESTAMP DEFAULT NOW() → TIMESTAMP: Lưu thời gian. DEFAULT NOW(): Mặc định là thời điểm hiện tại khi thêm dữ liệu.

4.2. Xem danh sách các Table

Cú pháp: SHOW TABLES;

Giải thích: SHOW TABLES → Liệt kê toàn bộ bảng có trong database hiện tại. Phải chọn database trước bằng USE database_name;.

4.3. Xem cấu trúc Table

Cú pháp: DESCRIBE table_name; hoặc SHOW COLUMNS FROM table_name;

Giải thích: DESCRIBE → Hiển thị thông tin chi tiết từng cột trong bảng (tên, kiểu dữ liệu, khóa, mặc định…).

Ví dụ
DESCRIBE users;

Kết quả mẫu:

Field Type Null Key Default Extra
id int NO PRI - auto_increment
name varchar(50) NO - - -
email varchar(100) YES UNI - -
age int YES - - -
status varchar(10) YES - active -
created_date timestamp YES - CURRENT_TIMESTAMP DEFAULT_GENERATED

4.4. Xóa Table

Cú pháp: DROP TABLE table_name;

Giải thích: DROP TABLE → Xóa hoàn toàn bảng và dữ liệu bên trong. Không thể hoàn tác, nên cần cẩn thận.

Ví dụ
DROP TABLE users;

→ Xóa bảng users khỏi database.

4.5. Ví dụ tổng hợp

SQL Commands

   created_date TIMESTAMP DEFAULT NOW()

-- 2. Xem danh sách bảng

-- 3. Xem cấu trúc bảng users

PHẦN 5. INSERT – Thêm dữ liệu vào bảng

Mục đích: Dùng để chèn (thêm) một hoặc nhiều bản ghi (record) vào bảng. Đây là bước để đưa dữ liệu thực tế vào cơ sở dữ liệu sau khi đã tạo cấu trúc bảng.

5.1. Cú pháp đầy đủ

INSERT Syntax
INSERT INTO table_name (column1, column2, column3, ...)
VALUES (value1, value2, value3, ...);

Giải thích: INSERT INTO table_name → Chọn bảng để thêm dữ liệu. (column1, column2, ...) → Danh sách các cột muốn chèn dữ liệu. VALUES (value1, value2, ...) → Giá trị tương ứng với các cột. Các giá trị chuỗi (text) phải đặt trong dấu ' ' (hoặc " "), giá trị số thì không cần.

5.2. Chèn dữ liệu vào tất cả các cột

INSERT INTO all columns
INSERT INTO table_name
VALUES (value1, value2, value3, ...);

Cẩn thận: Phải nhập đúng số lượng và thứ tự cột.

5.3. Ví dụ với bảng users

INSERT Examples
-- 1. Tạo database tên 'users'
CREATE DATABASE users;

-- 2. Chọn database để làm việc
USE users;

-- 3. Tạo bảng 'users'
CREATE TABLE users (
   id INT AUTO_INCREMENT PRIMARY KEY,
   name VARCHAR(50) NOT NULL,
   email VARCHAR(100) UNIQUE,
   age INT CHECK (age >= 18),
   status VARCHAR(10) DEFAULT 'active',
   created_date TIMESTAMP DEFAULT NOW()
);

-- 4. Chèn dữ liệu (cách 1: vào một số cột)
INSERT INTO users (name, email, age)
VALUES ('Nguyen Van A', 'a@example.com', 25);

-- 5. Chèn dữ liệu đầy đủ các cột (trừ id)
INSERT INTO users (name, email, age, status, created_date)
VALUES ('Tran Thi B', 'b@example.com', 30, 'inactive', '2025-08-10 14:30:00');

-- 6. Chèn nhiều bản ghi một lúc
INSERT INTO users (name, email, age)
VALUES
('Le Van C', 'c@example.com', 28),
('Pham Thi D', 'd@example.com', 22),
('Hoang Van E', 'e@example.com', 35);

-- 7. Xem dữ liệu đã chèn
SELECT * FROM users;

PHẦN 6. SELECT – Lấy dữ liệu

Mục đích: Dùng để truy vấn và lấy dữ liệu từ bảng. Có thể lấy toàn bộ dữ liệu hoặc chỉ lấy một số cột, kèm điều kiện lọc. Hỗ trợ sắp xếp, giới hạn số bản ghi.

6.1. Lấy toàn bộ dữ liệu

Cú pháp: SELECT * FROM table_name;

Giải thích: SELECT * → Lấy tất cả các cột. FROM table_name → Lấy dữ liệu từ bảng chỉ định.

Ví dụ
SELECT * FROM users;

6.2. Lấy một số cột nhất định

Cú pháp: SELECT column1, column2 FROM table_name;

Giải thích: Chỉ lấy các cột được liệt kê. Giúp giảm dữ liệu trả về và tăng tốc truy vấn.

Ví dụ
SELECT name, email FROM users;

6.3. Lọc dữ liệu với WHERE

Cú pháp: SELECT * FROM table_name WHERE condition;

Giải thích: WHERE → Lọc dữ liệu theo điều kiện. Điều kiện có thể dùng toán tử so sánh (=, >, <, >=, <=, <>) hoặc logic (AND, OR).

Ví dụ
SELECT * FROM users WHERE age >= 30;

6.4. Sắp xếp dữ liệu (ORDER BY)

Cú pháp: SELECT * FROM table_name ORDER BY column ASC|DESC;

Giải thích: ORDER BY → Sắp xếp theo một hoặc nhiều cột. ASC → Tăng dần (mặc định). DESC → Giảm dần.

Ví dụ
SELECT * FROM users ORDER BY age DESC;

6.5. Giới hạn số bản ghi (LIMIT)

Cú pháp: SELECT * FROM table_name LIMIT number;

Giải thích: LIMIT → Giới hạn số bản ghi trả về. Thường dùng khi chỉ cần xem trước dữ liệu.

Ví dụ
SELECT * FROM users LIMIT 3;

6.6. Code ví dụ tổng hợp

SELECT Examples
-- 1. Lấy toàn bộ dữ liệu
SELECT * FROM users;

-- 2. Lấy một số cột
SELECT name, email FROM users;

-- 3. Lọc theo điều kiện
SELECT * FROM users WHERE age >= 30;

-- 4. Sắp xếp giảm dần theo tuổi
SELECT * FROM users ORDER BY age DESC;

-- 5. Lấy 3 bản ghi đầu tiên
SELECT * FROM users LIMIT 3;

PHẦN 7: UPDATE – Cập nhật dữ liệu

Mục đích: Dùng để thay đổi giá trị của các cột trong bảng cho một hoặc nhiều bản ghi. Có thể cập nhật toàn bộ bảng hoặc chỉ những bản ghi thỏa điều kiện (dùng WHERE). Giúp chỉnh sửa dữ liệu bị sai, thêm thông tin mới, hoặc cập nhật trạng thái.

7.1. Cú pháp

UPDATE Syntax
UPDATE table_name
SET column1 = value1, column2 = value2, ...
[WHERE condition];

Giải thích: UPDATE table_name → Chỉ định bảng cần cập nhật. SET column1 = value1 → Gán giá trị mới cho cột. Có thể cập nhật nhiều cột cùng lúc (ngăn cách bằng dấu phẩy). WHERE condition → Điều kiện lọc bản ghi cần cập nhật. Cẩn thận: Nếu không có WHERE, tất cả các bản ghi sẽ bị cập nhật.

7.2. Các tùy chọn thường dùng

Cú pháp Mô tả Ví dụ
Cập nhật 1 cột Thay đổi giá trị một cột UPDATE users SET status='inactive' WHERE id=1;
Cập nhật nhiều cột Thay đổi nhiều cột cùng lúc UPDATE users SET name='Nguyen Van B', age=29 WHERE id=2;
Cập nhật toàn bộ bảng Không dùng WHERE UPDATE users SET status='active';
Sử dụng toán tử Tăng/giảm giá trị số UPDATE users SET age = age + 1 WHERE status='active';

7.3. Ví dụ thực hành

UPDATE Examples
-- 1. Cập nhật trạng thái của người dùng có id = 1
UPDATE users
SET status = 'inactive'
WHERE id = 1;

-- 2. Cập nhật tên và tuổi của người dùng có email 'b@example.com'
UPDATE users
SET name = 'Tran Thi B Updated', age = 31
WHERE email = 'b@example.com';

-- 3. Tăng tuổi của tất cả người dùng có trạng thái active thêm 1
UPDATE users
SET age = age + 1
WHERE status = 'active';

-- 4. Cập nhật trạng thái toàn bộ bảng thành 'pending'
UPDATE users
SET status = 'pending';

Ghi chú quan trọng: Luôn dùng WHERE khi chỉ muốn cập nhật một phần dữ liệu. Có thể kết hợp với ORDER BY và LIMIT để chỉ cập nhật một số bản ghi đầu tiên.

PHẦN 8. DELETE – Xóa dữ liệu

Mục đích: Lệnh DELETE được dùng để xóa một hoặc nhiều bản ghi (hàng) trong bảng MySQL. Khác với DROP: DELETE chỉ xóa dữ liệu, không xóa cấu trúc bảng. DROP sẽ xóa toàn bộ bảng và cấu trúc của nó.

Cú pháp

DELETE Syntax
DELETE FROM table_name
WHERE điều_kiện;

Giải thích: DELETE FROM table_name → Chỉ định bảng muốn xóa dữ liệu. WHERE điều_kiện → Xác định bản ghi cần xóa. Nếu bỏ WHERE → Xóa toàn bộ dữ liệu trong bảng (rất nguy hiểm). Có thể dùng nhiều điều kiện AND, OR để lọc chính xác bản ghi.

Ví dụ

DELETE Examples
-- 1. Xóa một bản ghi cụ thể
DELETE FROM users
WHERE id = 3;

-- 2. Xóa nhiều bản ghi theo điều kiện
DELETE FROM users
WHERE status = 'inactive';

-- 3. Xóa toàn bộ dữ liệu (Cẩn thận!)
DELETE FROM users;

Xử lý lỗi khi gặp Safe Update Mode (Error 1175)

MySQL Workbench có Safe Update Mode để tránh xóa nhầm dữ liệu.

Có 2 cách xử lý:

  • Thêm điều kiện với cột khóa chính: DELETE FROM users WHERE id = 3;
  • Tắt Safe Mode: Vào Edit → Preferences → SQL Editor → bỏ chọn "Safe Updates". Reconnect lại MySQL Workbench.

Ghi chú quan trọng

Lệnh Mô tả
DELETE Xóa dữ liệu theo điều kiện, giữ nguyên cấu trúc bảng
TRUNCATE Xóa toàn bộ dữ liệu, reset AUTO_INCREMENT
DROP TABLE Xóa cả bảng và dữ liệu bên trong

Ví dụ tổng hợp

DELETE Examples
-- 1. Xem dữ liệu hiện tại
SELECT * FROM users;

-- 2. Xóa người có ID = 5
DELETE FROM users
WHERE id = 5;

-- 3. Xóa tất cả user "inactive"
DELETE FROM users
WHERE status = 'inactive';

-- 4. Xóa toàn bộ dữ liệu (không khuyến khích)
DELETE FROM users;

PHẦN 9. LIKE – Tìm kiếm theo mẫu

Mục đích: LIKE dùng để tìm kiếm dữ liệu theo một mẫu ký tự thay vì phải khớp chính xác toàn bộ giá trị. Thường dùng trong tìm kiếm tên, email, địa chỉ hoặc dữ liệu dạng chuỗi. Có thể kết hợp với ký tự đại diện (%, _) để linh hoạt hơn.

Cú pháp

LIKE Syntax
SELECT cột1, cột2, ...
FROM tên_bảng
WHERE cột LIKE 'mẫu_tìm_kiếm';

Giải thích: LIKE → So sánh dữ liệu theo mẫu ký tự.

Ký tự đại diện:

  • % → Đại diện cho 0 hoặc nhiều ký tự bất kỳ.
    • 'A%' → Bắt đầu bằng chữ "A".
    • '%A' → Kết thúc bằng chữ "A".
    • '%A%' → Chứa chữ "A" ở bất kỳ vị trí nào.
  • _ → Đại diện cho một ký tự bất kỳ.
    • 'A_' → "A" + một ký tự bất kỳ (ví dụ: "An", "Ab").
    • 'A__' → "A" + hai ký tự bất kỳ (ví dụ: "Anh", "Abe").

Ví dụ

LIKE Examples
-- 1. Tìm user có tên bắt đầu bằng 'Nguyen'
SELECT * FROM users
WHERE name LIKE 'Nguyen%';

-- 2. Tìm user có email kết thúc bằng '@example.com'
SELECT * FROM users
WHERE email LIKE '%@example.com';

-- 3. Tìm user có tên chứa chữ 'Thi'
SELECT * FROM users
WHERE name LIKE '%Thi%';

-- 4. Tìm user có tên bắt đầu bằng 'T' và có đúng 3 ký tự
SELECT * FROM users
WHERE name LIKE 'T__';

Lưu ý: LIKE không phân biệt hoa thường trong MySQL (trừ khi dùng collation phân biệt). Nếu cần tìm chính xác ký tự % hoặc _, phải dùng ESCAPE. Với dữ liệu lớn, LIKE '%...%' sẽ chậm hơn vì phải quét toàn bộ bảng.

PHẦN 10. IN – Lọc theo danh sách

Mục đích: IN dùng để lọc dữ liệu theo một tập hợp giá trị cụ thể thay vì phải viết nhiều điều kiện OR. Giúp câu lệnh ngắn gọn, dễ đọc. Dùng tốt khi biết trước danh sách giá trị cần tìm.

Cú pháp

IN Syntax
SELECT cột1, cột2, ...
FROM tên_bảng
WHERE cột IN (giá_trị1, giá_trị2, ...);

Giải thích: IN → Chỉ chọn những bản ghi mà giá trị của cột nằm trong danh sách cho sẵn. Có thể dùng số, chuỗi hoặc kết quả của subquery. Tương đương với: cột = giá_trị1 OR cột = giá_trị2 OR ...

Ví dụ

IN Examples
-- 1. Tìm user có tuổi 22, 25 hoặc 30
SELECT * FROM users
WHERE age IN (22, 25, 30);

-- 2. Tìm user có trạng thái 'active' hoặc 'pending'
SELECT * FROM users
WHERE status IN ('active', 'pending');

-- 3. Dùng IN với subquery: Lấy user có ID nằm trong danh sách từ bảng khác
SELECT * FROM users
WHERE id IN (SELECT user_id FROM orders);

Lưu ý: Có thể dùng NOT IN để lọc những giá trị không nằm trong danh sách. Khi danh sách dài, IN gọn hơn nhiều so với dùng OR liên tục. Với bảng lớn, IN có thể chậm hơn JOIN hoặc EXISTS nếu subquery phức tạp.

Ví dụ tổng hợp

IN Combined Example
-- Lấy user có tuổi là 22, 28 hoặc 35 và trạng thái 'active'
SELECT * FROM users
WHERE age IN (22, 28, 35)
AND status = 'active'
ORDER BY age ASC;

PHẦN 11. BETWEEN – Lọc theo khoảng

Mục đích: BETWEEN dùng để lọc dữ liệu nằm trong một khoảng giá trị nhất định (bao gồm cả giá trị đầu và cuối). Dùng được cho số, ngày tháng, ký tự. Giúp câu lệnh ngắn gọn thay vì dùng nhiều điều kiện >= và <=.

Cú pháp

BETWEEN Syntax
SELECT cột1, cột2, ...
FROM tên_bảng
WHERE cột BETWEEN giá_trị_đầu AND giá_trị_cuối;

Giải thích: BETWEEN → Chỉ lấy những bản ghi mà giá trị nằm trong khoảng từ giá_trị_đầu đến giá_trị_cuối. Bao gồm cả giá trị biên (>= giá trị đầu và <= giá trị cuối). NOT BETWEEN → Lấy những giá trị nằm ngoài khoảng.

Ví dụ

BETWEEN Examples
-- 1. Tìm user có tuổi từ 25 đến 30
SELECT * FROM users
WHERE age BETWEEN 25 AND 30;

-- 2. Tìm user có ngày tạo trong tháng 8/2025
SELECT * FROM users
WHERE created_date BETWEEN '2025-08-01' AND '2025-08-31';

-- 3. Tìm user có tên theo bảng chữ cái từ 'H' đến 'T'
SELECT * FROM users
WHERE name BETWEEN 'H' AND 'T';

Lưu ý: Với dữ liệu ngày, nên đảm bảo định dạng YYYY-MM-DD hoặc chuẩn ISO. Khi lọc ngày theo giờ phút giây, cần chú ý thời điểm kết thúc để không bị thiếu dữ liệu. BETWEEN bao gồm cả giá trị đầu và cuối.

Ví dụ tổng hợp

BETWEEN Combined Example
-- Lấy user có tuổi từ 22 đến 30 và trạng thái 'active'
SELECT * FROM users
WHERE age BETWEEN 22 AND 30
AND status = 'active'
ORDER BY age ASC;
