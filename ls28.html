PHẦN 1: LAMBDA BASICS

1.1 LAMBDA EXPRESSION LÀ GÌ?

Định nghĩa:

Lambda Expression là một hàm vô danh (anonymous function) có thể được truyền như một đối tượng.

Nói dễ hiểu:

  • Thay vì viết cả một class dài dòng
  • Chỉ cần viết phần "làm gì" ngắn gọn

Ví dụ so sánh trực tiếp:

Ví dụ
// CÁCH CŨ: Phải viết cả class nặc danh
Runnable runnable1 = new Runnable() {
    @Override
    public void run() {
        System.out.println("Xin chào!");
    }
};

// CÁCH MỚI: Lambda - chỉ viết phần cốt lõi
Runnable runnable2 = () -> System.out.println("Xin chào!");

1.2 CÁC DẠNG CÚ PHÁP LAMBDA

Dạng 1: Lambda không tham số

Cú pháp
() -> { // code }
() -> expression

Giải thích:

  • (): thể hiện không có tham số
  • ->: toán tử lambda
  • {}: chứa các câu lệnh (có thể bỏ nếu chỉ 1 câu lệnh)
LambdaNoParam.java
public class LambdaNoParam {
    public static void main(String[] args) {
        // Cú pháp đầy đủ với {}
        Runnable task1 = () -> {
            System.out.println("Bắt đầu công việc");
            System.out.println("Kết thúc công việc");
        };
        
        // Cú pháp rút gọn với 1 câu lệnh
        Runnable task2 = () -> System.out.println("Công việc đơn giản");
        
        task1.run();
        task2.run();
    }
}

Giải thích ví dụ:

  • task1: có 2 câu lệnh nên phải dùng {}
  • task2: chỉ 1 câu lệnh nên có thể bỏ {}
  • Cả hai đều implement phương thức run() của Runnable
SupplierExample.java
import java.util.function.Supplier;

public class SupplierExample {
    public static void main(String[] args) {
        // Supplier trả về số ngẫu nhiên
        Supplier<Double> randomSupplier = () -> Math.random();
        
        // Supplier trả về chuỗi cố định
        Supplier<String> helloSupplier = () -> "Xin chào!";
        
        System.out.println(randomSupplier.get()); // ví dụ: 0.123456
        System.out.println(helloSupplier.get()); // Xin chào!
    }
}

Giải thích ví dụ:

  • Supplier<Double>: interface có method get() không tham số, trả về Double
  • () -> Math.random(): gọi method Math.random() và trả về kết quả
  • randomSupplier.get(): lấy giá trị từ Supplier

Dạng 2: Lambda một tham số

Cú pháp
(thamSo) -> { // code }
thamSo -> { // code } // có thể bỏ () nếu chỉ 1 tham số
(thamSo) -> expression
thamSo -> expression // bỏ () và {} nếu chỉ 1 câu lệnh

Giải thích:

  • Có thể đặt tên tham số tùy ý
  • Có thể bỏ dấu () nếu chỉ có 1 tham số
  • Java tự suy luận kiểu dữ liệu của tham số
LambdaOneParam.java
import java.util.function.Consumer;

public class LambdaOneParam {
    public static void main(String[] args) {
        // Cách 1: Đầy đủ dấu ngoặc
        Consumer<String> consumer1 = (s) -> {
            System.out.println("Xin chào, " + s);
        };
        
        // Cách 2: Bỏ () vì chỉ 1 tham số
        Consumer<String> consumer2 = s -> {
            System.out.println("Xin chào, " + s);
        };
        
        // Cách 3: Bỏ cả () và {} vì chỉ 1 câu lệnh
        Consumer<String> consumer3 = s -> System.out.println("Xin chào, " + s);
        
        consumer3.accept("Java"); // Xin chào, Java
    }
}

Giải thích ví dụ:

  • Consumer<String>: interface có method accept(T t) nhận 1 tham số
  • s là tham số, kiểu String được tự suy luận
  • Cả 3 cách đều tương đương nhau
FunctionExample.java
import java.util.function.Function;

public class FunctionExample {
    public static void main(String[] args) {
        // Function: nhận String, trả về Integer
        Function<String, Integer> lengthFunc = s -> s.length();
        
        String text = "Java Programming";
        int length = lengthFunc.apply(text);
        
        System.out.println("Độ dài chuỗi: " + length); // 16
    }
}

Giải thích ví dụ:

  • Function<String, Integer>: nhận String, trả về Integer
  • s -> s.length(): với mỗi chuỗi s, trả về độ dài của nó
  • apply(text): áp dụng function lên text
PredicateExample.java
import java.util.function.Predicate;

public class PredicateExample {
    public static void main(String[] args) {
        // Predicate: kiểm tra số chẵn
        Predicate<Integer> isEven = n -> n % 2 == 0;
        
        System.out.println(isEven.test(5)); // false
        System.out.println(isEven.test(6)); // true
        
        // Predicate: kiểm tra chuỗi rỗng
        Predicate<String> isEmpty = s -> s == null || s.isEmpty();
        
        System.out.println(isEmpty.test("")); // true
        System.out.println(isEmpty.test("Java")); // false
    }
}

Giải thích ví dụ:

  • Predicate<Integer>: nhận Integer, trả về boolean
  • n -> n % 2 == 0: kiểm tra n có chia hết cho 2 không
  • test(n): thực thi predicate với giá trị n

Dạng 3: Lambda nhiều tham số

Cú pháp
(thamSo1, thamSo2) -> { // code }
(thamSo1, thamSo2) -> expression

Giải thích:

  • Các tham số đặt trong (), phân cách bằng dấu phẩy
  • Không thể bỏ () khi có nhiều tham số
  • Có thể khai báo kiểu dữ liệu hoặc để Java tự suy luận
LambdaTwoParams.java
import java.util.*;

public class LambdaTwoParams {
    public static void main(String[] args) {
        // Comparator để so sánh hai số
        Comparator<Integer> comparator = (a, b) -> a.compareTo(b);
        
        List<Integer> numbers = new ArrayList<>(Arrays.asList(5, 2, 8, 1, 9));
        Collections.sort(numbers, comparator);
        
        System.out.println(numbers); // [1, 2, 5, 8, 9]
        
        // Có thể viết trực tiếp
        Collections.sort(numbers, (x, y) -> y.compareTo(x)); // giảm dần
        System.out.println(numbers); // [9, 8, 5, 2, 1]
    }
}

Giải thích ví dụ:

  • (a, b) -> a.compareTo(b): so sánh a và b, trả về số âm nếu a < b
  • (x, y) -> y.compareTo(x): so sánh ngược lại để sắp xếp giảm dần
BiFunctionExample.java
import java.util.function.BiFunction;

public class BiFunctionExample {
    public static void main(String[] args) {
        // BiFunction: nhận 2 số, trả về tổng
        BiFunction<Integer, Integer, Integer> sum = (a, b) -> a + b;
        
        // BiFunction: nhận 2 số, trả về tích
        BiFunction<Integer, Integer, Integer> product = (a, b) -> a * b;
        
        System.out.println("5 + 3 = " + sum.apply(5, 3)); // 8
        System.out.println("5 * 3 = " + product.apply(5, 3)); // 15
        
        // BiFunction với kiểu khác
        BiFunction<String, Integer, String> repeat = (s, n) -> {
            String result = "";
            for (int i = 0; i < n; i++) {
                result += s;
            }
            return result;
        };
        
        System.out.println(repeat.apply("Ha", 3)); // HaHaHa
    }
}

Giải thích ví dụ:

  • (a, b) -> a + b: với hai số a và b, trả về a + b
  • (s, n) -> { ... }: với chuỗi s và số n, tạo chuỗi lặp lại
  • apply(a, b): thực thi function với hai tham số
ComplexComparison.java
import java.util.*;

public class ComplexComparison {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>(Arrays.asList(
            "John", "Alice", "Bob", "David", "Eve"
        ));
        
        // Sắp xếp theo độ dài, nếu bằng thì theo alphabet
        names.sort((s1, s2) -> {
            if (s1.length() != s2.length()) {
                return Integer.compare(s1.length(), s2.length());
            }
            return s1.compareTo(s2);
        });
        
        System.out.println(names); // [Bob, Eve, John, Alice, David]
    }
}

Giải thích ví dụ:

  • (s1, s2) -> { ... }: khối code phức tạp phải dùng {}
  • Integer.compare(s1.length(), s2.length()): so sánh độ dài
  • Nếu độ dài bằng nhau, so sánh theo alphabet

Dạng 4: Lambda có kiểu dữ liệu tường minh

Cú pháp
(Kiểu1 thamSo1, Kiểu2 thamSo2) -> expression

Giải thích:

  • Có thể khai báo kiểu dữ liệu cho tham số
  • Giúp code rõ ràng hơn khi cần
  • Thường dùng khi lambda phức tạp
LambdaWithTypes.java
import java.util.function.BiFunction;

public class LambdaWithTypes {
    public static void main(String[] args) {
        // Khai báo kiểu tường minh
        BiFunction<Integer, Integer, Integer> sum1 = (Integer a, Integer b) -> a + b;
        
        // Không khai báo kiểu (Java tự suy luận)
        BiFunction<Integer, Integer, Integer> sum2 = (a, b) -> a + b;
        
        // Cả hai đều hoạt động như nhau
        System.out.println(sum1.apply(5, 3)); // 8
        System.out.println(sum2.apply(5, 3)); // 8
        
        // Khi cần ép kiểu hoặc làm rõ
        BiFunction<String, Integer, String> repeater =
            (String s, Integer n) -> s.repeat(n);
        
        System.out.println(repeater.apply("Hi ", 3)); // Hi Hi Hi
    }
}

Giải thích ví dụ:

  • (Integer a, Integer b): khai báo rõ a và b là Integer
  • Java vẫn suy luận được nếu không khai báo
  • Nên dùng khi lambda phức tạp để dễ đọc

Dạng 5: Lambda với nhiều câu lệnh

Cú pháp
(thamSo) -> {
    câu lệnh 1;
    câu lệnh 2;
    return giá_trị; // nếu có return
}

Giải thích:

  • Phải dùng {} khi có nhiều hơn 1 câu lệnh
  • Phải dùng return nếu cần trả về giá trị
  • Có thể khai báo biến cục bộ bên trong
LambdaMultipleStatements.java
import java.util.function.Function;

public class LambdaMultipleStatements {
    public static void main(String[] args) {
        // Tính giai thừa với nhiều câu lệnh
        Function<Integer, Integer> factorial = (n) -> {
            if (n < 0) {
                throw new IllegalArgumentException("Số âm không hợp lệ");
            }
            
            int result = 1;
            for (int i = 1; i <= n; i++) {
                result *= i;
            }
            return result;
        };
        
        System.out.println("5! = " + factorial.apply(5)); // 120
        System.out.println("3! = " + factorial.apply(3)); // 6
    }
}

Giải thích ví dụ:

  • (n) -> { ... }: lambda nhận n, trả về Integer
  • Bên trong {}: có nhiều câu lệnh
  • return result;: phải có return vì Function cần trả về giá trị
PrimeCheck.java
import java.util.function.Predicate;

public class PrimeCheck {
    public static void main(String[] args) {
        Predicate<Integer> isPrime = (n) -> {
            if (n <= 1) return false;
            if (n == 2) return true;
            if (n % 2 == 0) return false;
            
            for (int i = 3; i <= Math.sqrt(n); i += 2) {
                if (n % i == 0) return false;
            }
            return true;
        };
        
        System.out.println("7 là số nguyên tố? " + isPrime.test(7)); // true
        System.out.println("10 là số nguyên tố? " + isPrime.test(10)); // false
    }
}

Giải thích ví dụ:

  • Predicate<Integer>: trả về boolean
  • Có nhiều return ở các nhánh khác nhau
  • Logic phức tạp phải dùng nhiều câu lệnh

PHẦN 2: FUNCTIONAL INTERFACES

2.1 FUNCTIONAL INTERFACE LÀ GÌ?

Định nghĩa:

Functional Interface là interface chỉ có duy nhất một phương thức trừu tượng (Single Abstract Method - SAM).

Cú pháp khai báo:

Cú pháp
@FunctionalInterface
public interface TênInterface {
    // Chỉ MỘT phương thức trừu tượng
    void methodName(parameters);
    
    // Có thể có nhiều default/static methods
    default void defaultMethod() { }
    static void staticMethod() { }
}

Giải thích:

  • @FunctionalInterface: annotation đánh dấu (không bắt buộc nhưng nên dùng)
  • Nếu có 2 phương thức trừu tượng, compile sẽ báo lỗi

2.2 R U N N A B L E

Định nghĩa:

Runnable là functional interface đại diện cho một tác vụ không nhận tham số và không trả về kết quả.

Cú pháp:

Cú pháp
@FunctionalInterface
public interface Runnable {
    void run();
}
RunnableBasic.java
public class RunnableBasic {
    public static void main(String[] args) {
        // Tạo Runnable với lambda
        Runnable task = () -> System.out.println("Đang chạy tác vụ...");
        
        // Chạy trong thread hiện tại
        task.run();
        
        // Chạy trong thread mới
        Thread thread = new Thread(task);
        thread.start();
    }
}

Giải thích ví dụ:

  • Runnable có method run() không tham số, không return
  • Lambda () -> System.out.println(...) implement method run()
  • Có thể chạy trực tiếp hoặc trong Thread
RunnableComplex.java
public class RunnableComplex {
    public static void main(String[] args) {
        // Runnable với nhiều câu lệnh
        Runnable countTask = () -> {
            System.out.println("Bắt đầu đếm:");
            for (int i = 1; i <= 5; i++) {
                System.out.println("Số: " + i);
                try {
                    Thread.sleep(1000); // Chờ 1 giây
                } catch (InterruptedException e) {
                    e.printStackTrace();
                }
            }
            System.out.println("Kết thúc đếm!");
        };
        
        Thread thread = new Thread(countTask);
        thread.start();
    }
}

Giải thích ví dụ:

  • Lambda có nhiều câu lệnh nên dùng {}
  • Thread.sleep(1000): tạm dừng 1 giây
  • Bắt InterruptedException vì sleep() có thể bị ngắt

2.3 C O M P A R A T O R

Định nghĩa:

Comparator là functional interface dùng để so sánh hai đối tượng, trả về số âm, 0, hoặc dương.

Cú pháp:

Cú pháp
@FunctionalInterface
public interface Comparator<T> {
    int compare(T o1, T o2);
    // Có nhiều default methods khác
}

Giải thích:

compare(o1, o2) trả về:

  • Số âm: nếu o1 < o2
  • 0: nếu o1 = o2
  • Số dương: nếu o1 > o2
ComparatorNumbers.java
import java.util.*;

public class ComparatorNumbers {
    public static void main(String[] args) {
        List<Integer> numbers = new ArrayList<>(Arrays.asList(5, 2, 8, 1, 9, 3));
        
        // Comparator tăng dần
        Comparator<Integer> ascending = (a, b) -> a - b;
        numbers.sort(ascending);
        System.out.println("Tăng dần: " + numbers); // [1, 2, 3, 5, 8, 9]
        
        // Comparator giảm dần
        Comparator<Integer> descending = (a, b) -> b - a;
        numbers.sort(descending);
        System.out.println("Giảm dần: " + numbers); // [9, 8, 5, 3, 2, 1]
    }
}

Giải thích ví dụ:

  • (a, b) -> a - b: nếu a < b thì a - b < 0 → a đứng trước
  • (a, b) -> b - a: ngược lại
  • sort() dùng comparator để sắp xếp
ComparatorStrings.java
import java.util.*;

public class ComparatorStrings {
    public static void main(String[] args) {
        List<String> names = new ArrayList<>(Arrays.asList(
            "John", "Alice", "Bob", "David", "Eve"
        ));
        
        // Sắp xếp theo alphabet
        Comparator<String> alphabetical = (s1, s2) -> s1.compareTo(s2);
        names.sort(alphabetical);
        System.out.println("Alphabet: " + names); // [Alice, Bob, David, Eve, John]
        
        // Sắp xếp theo độ dài
        Comparator<String> byLength = (s1, s2) -> {
            if (s1.length() != s2.length()) {
                return Integer.compare(s1.length(), s2.length());
            }
            return s1.compareTo(s2);
        };
        names.sort(byLength);
        System.out.println("Theo độ dài: " + names); // [Bob, Eve, John, Alice, David]
    }
}

Giải thích ví dụ:

  • s1.compareTo(s2): so sánh theo thứ tự tự nhiên của String
  • byLength: so sánh độ dài trước, nếu bằng thì so alphabet
  • Integer.compare(a, b): method tiện lợi để so sánh số

2.4 C O N S U M E R

Định nghĩa:

Consumer là functional interface nhận một tham số và không trả về kết quả (tiêu thụ dữ liệu).

Cú pháp:

Cú pháp
@FunctionalInterface
public interface Consumer<T> {
    void accept(T t);
    
    default Consumer<T> andThen(Consumer<? super T> after) {
        // kết hợp hai Consumer
    }
}
ConsumerBasic.java
import java.util.*;
import java.util.function.Consumer;

public class ConsumerBasic {
    public static void main(String[] args) {
        // Consumer in hoa chuỗi
        Consumer<String> printUpper = s -> System.out.println(s.toUpperCase());
        
        // Consumer in thường chuỗi
        Consumer<String> printLower = s -> System.out.println(s.toLowerCase());
        
        printUpper.accept("Java"); // JAVA
        printLower.accept("Java"); // java
        
        // Dùng với List
        List<Integer> numbers = Arrays.asList(1, 2, 3, 4, 5);
        Consumer<Integer> printNumber = n -> System.out.print(n + " ");
        
        numbers.forEach(printNumber); // 1 2 3 4 5
    }
}

Giải thích ví dụ:

  • accept(T t): method duy nhất của Consumer
  • s -> System.out.println(...): in tham số s ra console
  • forEach(consumer): áp dụng consumer cho mỗi phần tử
ConsumerAndThen.java
import java.util.function.Consumer;

public class ConsumerAndThen {
    public static void main(String[] args) {
        Consumer<String> print = s -> System.out.print(s);
        Consumer<String> printNewLine = s -> System.out.println();
        
        // Kết hợp: in xong thì xuống dòng
        Consumer<String> printWithNewLine = print.andThen(s -> System.out.println());
        
        printWithNewLine.accept("Hello");
        printWithNewLine.accept("World");
        // Output:
        // Hello
        // World
    }
}

Giải thích ví dụ:

  • andThen(): tạo Consumer mới thực hiện consumer hiện tại rồi đến consumer sau
  • print.andThen(s -> System.out.println()): in xong thì xuống dòng

2.5 S U P P L I E R

Định nghĩa:

Supplier là functional interface không nhận tham số nhưng trả về một giá trị (cung cấp dữ liệu).

Cú pháp:

Cú pháp
@FunctionalInterface
public interface Supplier<T> {
    T get();
}
SupplierBasic.java
import java.util.*;
import java.util.function.Supplier;

public class SupplierBasic {
    public static void main(String[] args) {
        // Supplier cung cấp số ngẫu nhiên
        Supplier<Double> randomSupplier = () -> Math.random();
        
        // Supplier cung cấp ngày giờ hiện tại
        Supplier<String> dateSupplier = () -> new Date().toString();
        
        // Supplier cung cấp chuỗi cố định
        Supplier<String> helloSupplier = () -> "Xin chào!";
        
        System.out.println("Số ngẫu nhiên: " + randomSupplier.get());
        System.out.println("Ngày giờ: " + dateSupplier.get());
        System.out.println(helloSupplier.get());
    }
}

Giải thích ví dụ:

  • get(): method duy nhất của Supplier, trả về giá trị
  • () -> Math.random(): không tham số, trả về double
  • randomSupplier.get(): lấy giá trị từ supplier
LazyInitialization.java
import java.util.function.Supplier;

public class LazyInitialization {
    static class ExpensiveObject {
        public ExpensiveObject() {
            System.out.println("Tạo object tốn kém!");
        }
        
        public void doSomething() {
            System.out.println("Làm việc...");
        }
    }
    
    public static void main(String[] args) {
        // Chưa tạo object
        Supplier<ExpensiveObject> lazySupplier = () -> new ExpensiveObject();
        
        System.out.println("Chưa tạo object");
        
        // Bắt đầu tạo object khi cần
        ExpensiveObject obj = lazySupplier.get(); // In "Tạo object tốn kém!"
        obj.doSomething();
    }
}

Giải thích ví dụ:

  • lazySupplier chỉ là "công thức" tạo object, chưa tạo thật
  • Chỉ khi gọi get() mới thực sự tạo object
  • Gọi là lazy initialization - khởi tạo trễ

2.6 F U N C T I O N

Định nghĩa:

Function là functional interface nhận một tham số và trả về một kết quả (biến đổi dữ liệu).

Cú pháp:

Cú pháp
@FunctionalInterface
public interface Function<T, R> {
    R apply(T t);
    
    default <V> Function<V, R> compose(Function<? super V, ? extends T> before) {
        // thực hiện before trước, rồi mới apply
    }
    
    default <V> Function<T, V> andThen(Function<? super R, ? extends V> after) {
        // thực hiện apply trước, rồi mới after
    }
}
FunctionBasic.java
import java.util.function.Function;

public class FunctionBasic {
    public static void main(String[] args) {
        // Function: String -> Integer (lấy độ dài)
        Function<String, Integer> lengthFunc = s -> s.length();
        
        // Function: Integer -> String (chuyển số thành chuỗi)
        Function<Integer, String> intToString = n -> "Số: " + n;
        
        // Function: String -> String (in hoa)
        Function<String, String> upperFunc = s -> s.toUpperCase();
        
        System.out.println("Độ dài của 'Java': " + lengthFunc.apply("Java")); // 4
        System.out.println(intToString.apply(10)); // Số: 10
        System.out.println(upperFunc.apply("java")); // JAVA
    }
}

Giải thích ví dụ:

  • Function<T, R>: T là kiểu input, R là kiểu output
  • apply(T t): method duy nhất, nhận t, trả về R
  • s -> s.length(): nhận String s, trả về độ dài Integer
FunctionAndThen.java
import java.util.function.Function;

public class FunctionAndThen {
    public static void main(String[] args) {
        Function<Integer, Integer> doubleIt = n -> n * 2;
        Function<Integer, Integer> squareIt = n -> n * n;
        
        // doubleIt trước, squareIt sau
        Function<Integer, Integer> doubleThenSquare = doubleIt.andThen(squareIt);
        
        // squareIt trước, doubleIt sau
        Function<Integer, Integer> squareThenDouble = doubleIt.compose(squareIt);
        
        int num = 5;
        System.out.println("5 * 2 = " + doubleIt.apply(5)); // 10
        System.out.println("5² = " + squareIt.apply(5)); // 25
        System.out.println("(5*2)² = " + doubleThenSquare.apply(5)); // 100
        System.out.println("(5²)*2 = " + squareThenDouble.apply(5)); // 50
    }
}

Giải thích ví dụ:

  • andThen(): f1.andThen(f2) → thực hiện f1 trước, f2 sau
  • compose(): f1.compose(f2) → thực hiện f2 trước, f1 sau
  • doubleThenSquare.apply(5): (5*2)² = 10² = 100

2.7 P R E D I C A T E

Định nghĩa:

Predicate là functional interface nhận một tham số và trả về boolean (kiểm tra điều kiện).

Cú pháp:

Cú pháp
@FunctionalInterface
public interface Predicate<T> {
    boolean test(T t);
    
    default Predicate<T> and(Predicate<? super T> other) {
        // kết hợp với điều kiện AND
    }
    
    default Predicate<T> or(Predicate<? super T> other) {
        // kết hợp với điều kiện OR
    }
    
    default Predicate<T> negate() {
        // phủ định điều kiện
    }
}
PredicateBasic.java
import java.util.function.Predicate;

public class PredicateBasic {
    public static void main(String[] args) {
        // Kiểm tra số chẵn
        Predicate<Integer> isEven = n -> n % 2 == 0;
        
        // Kiểm tra số dương
        Predicate<Integer> isPositive = n -> n > 0;
        
        // Kiểm tra chuỗi rỗng
        Predicate<String> isEmpty = s -> s == null || s.isEmpty();
        
        System.out.println("5 có chẵn? " + isEven.test(5)); // false
        System.out.println("5 có dương? " + isPositive.test(5)); // true
        System.out.println("'' có rỗng? " + isEmpty.test("")); // true
    }
}

Giải thích ví dụ:

  • test(T t): method duy nhất, nhận t, trả về boolean
  • n -> n % 2 == 0: kiểm tra n có chia hết cho 2 không
  • isEmpty.test(""): kiểm tra chuỗi rỗng
PredicateCombine.java
import java.util.function.Predicate;

public class PredicateCombine {
    public static void main(String[] args) {
        Predicate<Integer> isEven = n -> n % 2 == 0;
        Predicate<Integer> isPositive = n -> n > 0;
        Predicate<Integer> isLessThan10 = n -> n < 10;
        
        // AND: số chẵn VÀ dương
        Predicate<Integer> evenAndPositive = isEven.and(isPositive);
        
        // OR: chẵn HOẶC nhỏ hơn 10
        Predicate<Integer> evenOrLessThan10 = isEven.or(isLessThan10);
        
        // NEGATE: không phải số chẵn (số lẻ)
        Predicate<Integer> isOdd = isEven.negate();
        
        System.out.println("6 chẵn và dương? " + evenAndPositive.test(6)); // true
        System.out.println("15 chẵn hoặc <10? " + evenOrLessThan10.test(15)); // false
        System.out.println("5 có lẻ? " + isOdd.test(5)); // true
    }
}

Giải thích ví dụ:

  • and(): kết hợp với điều kiện AND
  • or(): kết hợp với điều kiện OR
  • negate(): phủ định điều kiện

BiFunction - Hai tham số, một kết quả

Cú pháp
@FunctionalInterface

BiFunctionExample.java
import java.util.function.BiFunction;

public class BiFunctionExample {
    public static void main(String[] args) {
        // Cộng hai số
        BiFunction<Integer, Integer, Integer> add = (a, b) -> a + b;

        };

        System.out.println("Max của 5 và 8: " + max.apply(5, 8)); // 8
        System.out.println(repeat.apply("Ha", 3)); // HaHaHa
    }
}

Giải thích ví dụ:

PHẦN 3: METHOD REFERENCES

3.1 METHOD REFERENCE LÀ GÌ?

Định nghĩa:

Method Reference là cú pháp rút gọn của Lambda Expression khi bạn chỉ cần gọi một phương thức có sẵn.

So sánh:

Ví dụ
// Lambda
Consumer<String> c1 = s -> System.out.println(s);

// Method Reference
Consumer<String> c2 = System.out::println;

Cú pháp:

Cú pháp
ĐốiTượng::TênPhươngThức

3.2 LOẠI 1: STATIC METHOD REFERENCE

Cú pháp:

Cú pháp
TênLớp::tênStaticMethod

Giải thích:

  • Gọi phương thức static của một class
  • Các tham số của lambda trở thành tham số của phương thức static
StaticMethodRef1.java
import java.util.function.BinaryOperator;

public class StaticMethodRef1 {
    public static void main(String[] args) {
        // Lambda
        BinaryOperator<Integer> lambdaMax = (a, b) -> Math.max(a, b);
        
        // Method reference
        BinaryOperator<Integer> refMax = Math::max;
        
        System.out.println(lambdaMax.apply(5, 8)); // 8
        System.out.println(refMax.apply(5, 8)); // 8
        
        // Giải thích: Math.max là static method
        // (a, b) -> Math.max(a, b) tương đương Math::max
    }
}

Giải thích:

  • Math.max(a, b) là phương thức static, nhận 2 tham số
  • Lambda (a, b) -> Math.max(a, b) gọi trực tiếp Math.max
  • Method reference Math::max là cách viết ngắn gọn hơn
StaticMethodRef2.java

Giải thích:

  • Integer.parseInt(s) là static method, nhận String, trả về int
  • s -> Integer.parseInt(s) → Integer::parseInt

Cú pháp:

Cú pháp
TênLớp::tênInstanceMethod

Giải thích:

  • Gọi instance method trên một object của class đó
  • Object đầu tiên của lambda sẽ là object gọi method
  • Các tham số còn lại là tham số của method
ArbitraryMethodRef1.java
import java.util.function.Function;

public class ArbitraryMethodRef1 {

Giải thích:

  • s -> s.length(): s là String, gọi length() trên s
  • String::length: nói rằng "gọi method length của String"
  • Tham số đầu tiên của lambda (s) trở thành object gọi method

3.6 TỔNG HỢP CÁC LOẠI METHOD REFERENCE

MethodReferenceSummary.java
import java.util.*;
import java.util.function.*;

public class MethodReferenceSummary {
    public static void main(String[] args) {
        
        // 1. STATIC METHOD REFERENCE
        // Cú pháp: ClassName::staticMethod
        BinaryOperator<Integer> maxRef = Math::max;
        // Tương đương: (a, b) -> Math.max(a, b)
        
        // 2. INSTANCE METHOD OF SPECIFIC OBJECT
        // Cú pháp: object::instanceMethod
        List<String> list = new ArrayList<>();
        Consumer<String> addRef = list::add;
        // Tương đương: s -> list.add(s)
        
        // 3. INSTANCE METHOD OF ARBITRARY OBJECT
        // Cú pháp: ClassName::instanceMethod
        Function<String, Integer> lengthRef = String::length;
        // Tương đương: s -> s.length()
        
        // 4. CONSTRUCTOR REFERENCE
        // Cú pháp: ClassName::new
        Supplier<List<String>> listCreator = ArrayList::new;
        // Tương đương: () -> new ArrayList<>()
        
        // Ví dụ tổng hợp
        List<String> names = Arrays.asList("John", "Alice", "Bob");
        
        // Sử dụng method reference
        names.stream()
            .map(String::toUpperCase) // instance method arbitrary
            .sorted(String::compareTo) // instance method arbitrary
            .forEach(System.out::println); // instance method specific
        
        // Tạo List mới bằng constructor reference
        List<String> copy = names.stream()
                             .filter(s -> s.length() > 3)
                             .collect(Collectors.toCollection(ArrayList::new));
        System.out.println(copy);
    }
}

PHẦN 4: TỔNG KẾT

4.1 BẢNG TÓM TẮT CÚ PHÁP LAMBDA

Loại Lambda Cú pháp Ví dụ
Không tham số () -> expression () -> System.out.println("Hi")
Một tham số p -> expression s -> s.length()
Nhiều tham số (p1, p2) -> expression (a, b) -> a + b
Có kiểu dữ liệu (Type p1, Type p2) -> expression (String s1, String s2) -> s1.length()
Nhiều câu lệnh (p) -> { stmt1; stmt2; } (x) -> { int y = x*2; return y; }

4.2 BẢNG TÓM TẮT FUNCTIONAL INTERFACES

Interface Input Output Method Ví dụ Lambda
Runnable 0 void run() () -> System.out.println("Hi")
Supplier<T> 0 T get() () -> Math.random()
Consumer<T> 1 void accept(T) s -> System.out.println(s)
Function<T,R> 1 R apply(T) s -> s.length()
Predicate<T> 1 boolean test(T) n -> n % 2 == 0
UnaryOperator<T> 1 T apply(T) n -> n * 2
BinaryOperator<T> 2 T apply(T,T) (a, b) -> a + b
BiFunction<T,U,R> 2 R apply(T,U) (s, n) -> s.repeat(n)
BiConsumer<T,U> 2 void accept(T,U) (k, v) -> System.out.println(k+v)

4.3 BẢNG TÓM TẮT METHOD REFERENCES

Loại Cú pháp Lambda tương đương Ví dụ
Static method Math::max (a, b) -> Math.max(a, b) Integer::parseInt
Instance method của object cụ thể System.out::println x -> System.out.println(x) list::add
Instance method của object bất kỳ String::length s -> s.length() String::toUpperCase
Constructor ArrayList::new () -> new ArrayList() String[]::new

Tóm tắt nhanh:

  • Lambda - "Hàm vô danh" - viết ngắn gọn code xử lý
  • Functional Interface - "Khuôn mẫu" - interface có 1 method trừu tượng
  • Method Reference - "Tham chiếu phương thức" - cú pháp rút gọn của lambda

Tổng kết Lambda Expressions

Lambda Functional Interface Method Reference

() -> {} @FunctionalInterface ::

"Lambda làm code ngắn gọn, Functional Interface làm code chuẩn hóa, Method Reference làm code dễ đọc!"
