PHẦN 1: TỔNG QUAN VỀ JAVA

1.1 Java là gì? - Giải thích chi tiết

Định nghĩa đầy đủ: Java là ngôn ngữ lập trình hướng đối tượng (Object-Oriented), đa nền tảng (cross-platform), được phát triển bởi James Gosling và đội ngũ tại Sun Microsystems vào năm 1995.

Tại sao gọi là "Viết một lần, chạy mọi nơi"?

Cùng một file .java này có thể chạy trên: Windows, macOS, Linux, Android, mà không cần sửa đổi code

HelloWorld.java
public class HelloWorld {
    public static void main(String[] args) {
        System.out.println("Chạy trên mọi hệ điều hành!");
    }
}

1.2 Kiến trúc Java Platform

Sơ đồ chi tiết:

Java Source Code (.java) → Java Compiler → Bytecode (.class) → JVM → Native Code

Giải thích từng thành phần:

PHẦN 3: CẤU TRÚC CHƯƠNG TRÌNH JAVA CHI TIẾT

3.1 Phân tích từng thành phần

File: HelloWorld.java
// 1. KHAI BÁO PACKAGE (tùy chọn)
package com.company.buoi1;

// 2. IMPORT THƯ VIỆN (tùy chọn)
import java.util.Date;

// 3. KHAI BÁO LỚP (bắt buộc)
public class HelloWorld {
    
    // 4. PHƯƠNG THỨC MAIN (bắt buộc)
    public static void main(String[] args) {
        
        // 5. CÁC CÂU LỆNH THỰC THI
        System.out.println("Hello Java World!");
        System.out.println("Thời gian hiện tại: " + new Date());
    }
}

3.2 Giải thích CHI TIẾT từng phần

Package (Gói):

package com.company.buoi1;

Giải thích từ khóa:

  • package: Từ khóa khai báo gói
  • com.company.buoi1: Tên gói theo quy tắc domain ngược
  • Mục đích: Tổ chức code, tránh xung đột tên
  • Quy ước: viết thường, phân cách bằng dấu chấm
  • Tương ứng với thư mục: com/company/buoi1/

Import:

import java.util.Date; // Import lớp Date cụ thể
import java.util.*; // Import toàn bộ gói util
import static java.lang.Math.*; // Import phương thức tĩnh

CÁC GÓI QUAN TRỌNG:

  • java.lang - Tự động import (String, System, Math)
  • java.util - Công cụ tiện ích (Date, ArrayList)
  • java.io - Input/Output (File, Scanner)

Class (Lớp):

public class HelloWorld { … }

Giải thích từ khóa:

  • public: Lớp có thể truy cập từ mọi nơi
  • class: Từ khóa khai báo lớp
  • HelloWorld: Tên lớp - VIẾT HOA CHỮ CÁI ĐẦU TỪ
  • QUAN TRỌNG: Tên file PHẢI TRÙNG với tên lớp public

Phương thức main:

public static void main(String[] args) {...}
Từ khóa Giải thích chi tiết
public Ai cũng có thể gọi phương thức này. JVM cần truy cập được để khởi chạy
static Thuộc về LỚP, không thuộc về ĐỐI TƯỢNG. Có thể gọi mà không cần tạo đối tượng. Ví dụ: HelloWorld.main() thay vì new HelloWorld().main()
void Phương thức KHÔNG TRẢ VỀ giá trị. Nếu có return type, phải dùng return statement
main Tên phương thức BẮT BUỘC. JVM tìm phương thức này để bắt đầu thực thi
String[] args Tham số đầu vào (command-line arguments). Mảng các chuỗi ký tự. Ví dụ: java HelloWorld arg1 arg2

PHẦN 4: CÁC LỆNH NHẬP XUẤT TRONG JAVA

3.4.1. Lệnh xuất dữ liệu (Output)

Sử dụng System.out.print() và System.out.println()

Lệnh Mô tả
System.out.print() In ra màn hình không xuống dòng sau khi in
System.out.println() In ra màn hình và xuống dòng sau khi in

Giải thích từ khóa:

  • System: Lớp final trong java.lang. Chứa các thành phần liên quan đến hệ thống. Constructor private → không thể tạo instance
  • out: Biến static trong lớp System. Kiểu: PrintStream. Mặc định trỏ đến stdout (console)
  • println(): Phương thức của PrintStream. In chuỗi và XUỐNG DÒNG. Có thể in bất kỳ kiểu dữ liệu nào
public class OutputExample {
    public static void main(String[] args) {
        System.out.print("Xin chao ");
        System.out.println("Java!");
        System.out.println("Hoc nhap xuat du lieu.");
    }
}

Kết quả hiển thị:

Xin chao Java!
Hoc nhap xuat du lieu.

Giải thích: Dòng đầu tiên dùng print nên "Xin chao " và "Java!" nằm trên cùng một dòng. Dòng thứ hai dùng println nên tự động xuống dòng.

3.4.2. Lệnh nhập dữ liệu (Input)

Trong Java có nhiều cách nhập dữ liệu, nhưng cách phổ biến nhất là dùng lớp Scanner.

Sử dụng lớp Scanner

Cần import lớp này trước khi sử dụng: import java.util.Scanner;

Sau đó tạo đối tượng: Scanner sc = new Scanner(System.in);

Các phương thức nhập phổ biến của Scanner

Phương thức Kiểu dữ liệu nhập Ví dụ
nextLine() Chuỗi (String) String name = sc.nextLine();
next() Chuỗi (chỉ 1 từ) String word = sc.next();
nextInt() Số nguyên (int) int a = sc.nextInt();
nextDouble() Số thực (double) double b = sc.nextDouble();
nextFloat() Số thực (float) float c = sc.nextFloat();
nextBoolean() Kiểu logic (true/false) boolean ok = sc.nextBoolean();

Ví dụ: Nhập chuỗi và số nguyên

import java.util.Scanner;
public class InputExample {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Nhap ten cua ban: ");
        String name = sc.nextLine();

        System.out.print("Nhap tuoi cua ban: ");
        int age = sc.nextInt();

        System.out.println("Chao " + name + "! Ban " + age + " tuoi.");
    }
}

Kết quả khi chạy:

Nhap ten cua ban: Minh
Nhap tuoi cua ban: 21
Chao Minh! Ban 21 tuoi.

3.4.3. Ví dụ tổng hợp nhập xuất

import java.util.Scanner;

public class Example {
    public static void main(String[] args) {
        Scanner sc = new Scanner(System.in);

        System.out.print("Nhap ho ten: ");
        String name = sc.nextLine();

        System.out.print("Nhap diem trung binh: ");
        double score = sc.nextDouble();

        System.out.print("Nhap tuoi: ");
        int age = sc.nextInt();

        System.out.println("\n--- Thong tin sinh vien ---");
        System.out.println("Ho ten: " + name);
        System.out.println("Tuoi: " + age);
        System.out.println("Diem trung binh: " + score);
    }
}

Kết quả khi chạy:

Nhap ho ten: Nguyen Dinh Manh
Nhap diem trung binh: 8.5
Nhap tuoi: 21

--- Thong tin sinh vien ---
Ho ten: Nguyen Dinh Manh
Tuoi: 21
Diem trung binh: 8.5

3.4.4. Lưu ý khi dùng Scanner

Vấn đề thường gặp: Khi dùng nextInt(), nextDouble(), ... rồi sau đó dùng nextLine() → dòng nextLine() có thể bị bỏ qua.

Nguyên nhân: ký tự Enter (\n) sau khi nhập số vẫn còn trong bộ nhớ đệm.

Cách khắc phục: Thêm một dòng sc.nextLine(); sau khi nhập số để xóa bỏ ký tự Enter.

System.out.print("Nhap tuoi: ");
int age = sc.nextInt();
sc.nextLine(); // xoá bỏ ký tự Enter

System.out.print("Nhap dia chi: ");
String address = sc.nextLine();

PHẦN 4: BIẾN VÀ KIỂU DỮ LIỆU

4.1 Khái niệm biến

Định nghĩa kỹ thuật: Biến là vùng nhớ được đặt tên, dùng để lưu trữ dữ liệu trong chương trình.

Phân loại biến trong Java:

public class VariableTypes {
    // 1. BIẾN INSTANCE (Non-static field)
    int instanceVar; // Thuộc về từng đối tượng
    
    // 2. BIẾN STATIC (Class variable)
    static String staticVar = "Hello"; // Thuộc về lớp, dùng chung
    
    // 3. BIẾN LOCAL (Trong phương thức)
    public void method() {
        int localVar = 10; // Phải khởi tạo, chỉ tồn tại trong method
    }
    
    // 4. BIẾN THAM SỐ (Parameter)
    public void calculate(int paramVar) {
        // paramVar nhận giá trị từ tham số truyền vào
    }
}

4.2 Cú pháp khai báo biến

[access-modifier] [static] [final] data-type variable-name [= value];

ACCESS-MODIFIER - Không bắt buộc

  • Là gì: Quy định ai được phép truy cập biến
  • Các loại:
    • public - Mọi nơi đều truy cập được
    • private - Chỉ trong class hiện tại
    • protected - Trong package + class con
    • (không gì) - Chỉ trong package

STATIC - Không bắt buộc

  • Là gì: Biến này thuộc về class, không thuộc về đối tượng
  • Khi nào dùng: Khi muốn dùng chung giá trị cho mọi đối tượng
  • Ví dụ: static int counter = 0;

FINAL - Không bắt buộc

  • Là gì: Biến không thể thay đổi giá trị sau khi gán (hằng số)
  • Đặc điểm: Phải khởi tạo giá trị ngay
  • Ví dụ: final double PI = 3.14;

DATA-TYPE - BẮT BUỘC

  • Là gì: Kiểu dữ liệu của biến
  • Ví dụ: int, String, double, boolean...

VARIABLE-NAME - BẮT BUỘC

  • Là gì: Tên của biến
  • Quy tắc: Viết thường chữ đầu, camelCase
  • Ví dụ: firstName, totalScore

= VALUE - Không bắt buộc

  • Là gì: Gán giá trị ban đầu cho biến
  • Ví dụ: = 10, = "Hello", = true

QUY TẮC QUAN TRỌNG:

  • Phần bắt buộc: data-type và variable-name
  • Phần tùy chọn: access-modifier, static, final, giá trị khởi tạo
  • Thứ tự: Đúng thứ tự như cú pháp
  • Dấu chấm phẩy: Kết thúc bằng ;
// Các cách khai báo:
int x; // Khai báo không khởi tạo
int y = 10; // Khai báo và khởi tạo
final double PI = 3.14; // Hằng số (không thay đổi)
public static int count; // Biến static public

// Khai báo nhiều biến cùng lúc:
int a = 1, b = 2, c = 3;
String name = "John", city = "Hanoi";

// Khai báo với giá trị mặc định:
class DefaultValues {
    int number; // mặc định = 0
    boolean flag; // mặc định = false
    String text; // mặc định = null
    double value; // mặc định = 0.0
}

4.3 Kiểu dữ liệu trong java

4.3.1. Kiểu dữ liệu (Data Type) là gì:

Kiểu dữ liệu (Data Type) là loại của dữ liệu mà biến có thể lưu trữ và cho biết kích thước, phạm vi, và kiểu giá trị mà nó chứa được.

Nói đơn giản: Kiểu dữ liệu giống như "loại hộp" dùng để chứa thông tin trong chương trình.

// Ví dụ minh họa:
int tuoi = 20; // kiểu int: lưu số nguyên
double diem = 8.5; // kiểu double: lưu số thực
char kyTu = 'A'; // kiểu char: lưu ký tự
boolean hopLe = true; // kiểu boolean: lưu true/false
String ten = "Manh"; // kiểu String: lưu chuỗi

4.3.2. Tại sao phải học kiểu dữ liệu?

Có 3 lý do lớn khiến bạn bắt buộc phải hiểu rõ kiểu dữ liệu trong Java

1. Giúp máy tính hiểu và quản lý dữ liệu đúng cách

Máy tính không "hiểu" chữ hay số như con người, nó chỉ hiểu nhị phân (0 và 1). → Java dùng kiểu dữ liệu để nói cho máy biết:

Dữ liệu Cách hiểu Kích thước
int a = 10; Đây là số nguyên, lưu bằng 4 byte 4 byte
double b = 10.5; Đây là số thực, lưu bằng 8 byte 8 byte
char c = 'A'; Đây là ký tự, lưu mã Unicode 2 byte

Nếu không có kiểu dữ liệu, máy không biết nên dùng bao nhiêu byte để lưu giá trị hoặc xử lý phép toán.

2. Đảm bảo chương trình chạy đúng và tránh lỗi

int so = 5;
String chu = "5";

// Không thể cộng trực tiếp int và String:
System.out.println(so + chu); // lỗi nếu không ép kiểu

Vì Java là ngôn ngữ mạnh về kiểu dữ liệu (strongly typed), nên nó bắt bạn phải dùng đúng loại dữ liệu để tránh lỗi logic.

3. Giúp tối ưu bộ nhớ và hiệu suất

byte tuoi = 25; // tiết kiệm hơn int
boolean hopLe = true; // chỉ lưu 1 bit
  • Dùng byte thay vì int nếu chỉ cần lưu giá trị nhỏ (0–127) → tiết kiệm RAM.
  • Dùng float thay vì double nếu độ chính xác không cần cao → chạy nhanh hơn.
  • Dùng boolean khi chỉ cần 2 giá trị true/false → logic rõ ràng, dễ hiểu.

4.3.3. Phân loại kiểu dữ liệu trong Java

Trong Java, kiểu dữ liệu (Data Types) được chia thành 2 nhóm lớn:

  • Kiểu dữ liệu nguyên thủy (Primitive Data Types)
  • Kiểu dữ liệu tham chiếu (Non-Primitive / Reference Data Types)

4.3.3.1. Kiểu dữ liệu nguyên thủy (Primitive Data Types)

Là những kiểu cơ bản nhất, được Java xây sẵn trong ngôn ngữ, lưu trực tiếp giá trị trong bộ nhớ. Java có 8 kiểu dữ liệu nguyên thủy được chia làm 4 nhóm chính

Nhóm Kiểu dữ liệu Kích thước Phạm vi giá trị Ví dụ Mô tả
Số nguyên (Integer) byte 1 byte -128 → 127 byte b = 10; Dùng cho số nhỏ
short 2 byte -32,768 → 32,767 short s = 1000; Dùng khi không cần quá lớn
int 4 byte -2,147,483,648 → 2,147,483,647 int i = 100000; Kiểu số nguyên mặc định
long 8 byte ±9.22×10¹⁸ long l = 10000000000L; Dùng cho số rất lớn
Số thực (Floating-point) float 4 byte ~±3.4×10³⁸ float f = 3.14f; Cần hậu tố f
double 8 byte ~±1.7×10³⁰⁸ double d = 3.14159; Mặc định cho số thực
Ký tự (Character) char 2 byte 0 → 65,535 (Unicode) char c = 'A'; Lưu 1 ký tự
Logic (Boolean) boolean 1 bit (thực tế 1 byte) true hoặc false boolean ok = true; Dùng cho điều kiện logic
public class PrimitiveExample {
    public static void main(String[] args) {
        byte b = 100;
        short s = 2000;
        int i = 50000;
        long l = 9000000000L;
        float f = 3.14f;
        double d = 3.141592653589793;
        char c = 'A';
        boolean isJavaFun = true;

        System.out.println("byte: " + b);
        System.out.println("short: " + s);
        System.out.println("int: " + i);
        System.out.println("long: " + l);
        System.out.println("float: " + f);
        System.out.println("double: " + d);
        System.out.println("char: " + c);
        System.out.println("boolean: " + isJavaFun);
    }
}

4.3.3.2. Kiểu dữ liệu tham chiếu (Reference Data Types)

Đây là những kiểu không lưu trực tiếp giá trị, mà lưu địa chỉ ô nhớ (vị trí) của đối tượng. Bao gồm:

Loại Ví dụ Mô tả
Chuỗi (String) String ten = "Manh"; Chuỗi ký tự, thực chất là một đối tượng của lớp String
Mảng (Array) int[] arr = {1, 2, 3}; Dùng để lưu nhiều giá trị cùng kiểu
Lớp (Class) Student sv = new Student(); Tạo ra đối tượng từ lớp do người dùng định nghĩa
Giao diện (Interface) Runnable r = new MyRunnable(); Dùng để triển khai đa hình
Kiểu đặc biệt khác Enum, Record, Object, ... Các cấu trúc nâng cao trong Java
public class ReferenceExample {
    public static void main(String[] args) {
        String name = "Manh";
        int[] numbers = {1, 2, 3, 4};
        
        System.out.println("Tên: " + name);
        System.out.println("Phần tử đầu tiên trong mảng: " + numbers[0]);
    }
}

Ở đây String và int[] không lưu trực tiếp dữ liệu, mà lưu địa chỉ vùng nhớ nơi dữ liệu thật sự tồn tại trong RAM.

4.4 Ép kiểu (Type Casting)

4.4.1. Khái niệm Ép kiểu (Type Casting) là gì?

Ép kiểu là quá trình chuyển đổi giá trị của một biến từ kiểu dữ liệu này sang kiểu dữ liệu khác.

Mục đích: để có thể thực hiện phép toán hoặc gán giá trị giữa các kiểu khác nhau.

4.4.2. Có mấy loại ép kiểu trong Java?

Loại ép kiểu Tên tiếng Anh Mô tả ngắn gọn
Ép kiểu ngầm định Implicit (Widening) Casting Tự động xảy ra khi gán kiểu nhỏ → kiểu lớn
Ép kiểu tường minh Explicit (Narrowing) Casting Cần chỉ rõ bằng cú pháp khi gán kiểu lớn → kiểu nhỏ

4.4.3. Ép kiểu ngầm định (Widening Casting)

Java tự động chuyển kiểu nhỏ sang kiểu lớn, vì an toàn (không mất dữ liệu).

Thứ tự tăng dần kích thước: byte → short → int → long → float → double

public class Main {
    public static void main(String[] args) {
        int a = 10;
        double b = a; // ép kiểu ngầm định: int → double
        System.out.println("Giá trị b: " + b);
    }
}

Kết quả: Giá trị b: 10.0

Không cần ghi (double) vì Java tự hiểu.

4.4.4. Ép kiểu tường minh (Narrowing Casting)

Khi chuyển kiểu lớn → nhỏ, ta phải chỉ rõ bằng cú pháp: (kiểu_mới) giá_trị

Có thể mất dữ liệu hoặc mất phần thập phân.

public class Main {
    public static void main(String[] args) {
        double x = 9.78;
        int y = (int) x; // ép kiểu tường minh: double → int
        System.out.println("Giá trị y: " + y);
    }
}

Kết quả: Giá trị y: 9

Bị mất phần thập phân (.78) vì int không lưu phần lẻ.

4.4.5. Ép kiểu giữa các kiểu dữ liệu khác

Ép giữa kiểu ký tự và số

char c = 'A';
int code = (int) c; // chuyển ký tự sang mã ASCII
System.out.println(code); // 65

int num = 66;
char letter = (char) num; // chuyển mã sang ký tự
System.out.println(letter); // B

Ép giữa các kiểu số khác nhau

float f = 5.9f;
int i = (int) f; // mất phần thập phân
System.out.println(i); // 5

Trong Java có các lớp bao (wrapper classes) giúp chuyển đổi giữa kiểu nguyên thủy ↔ đối tượng (String, Object, ...)

Float float Float.parseFloat("4.5")
Long long Long.parseLong("9999")
Boolean boolean Boolean.parseBoolean("true")

5.1 Toán tử số học

Toán tử số học (Arithmetic Operators) trong Java được dùng để thực hiện các phép tính số học trên các biến hoặc hằng số số học (như int, float, double, ...).

Toán tử Tên gọi Ví dụ
(nếu a = 10, b = 3)		 Kết quả Giải thích
+ Cộng a + b 13 Cộng hai số
- Trừ a - b 7 Trừ b khỏi a
* Nhân a * b 30 Nhân a với b
/ Chia a / b 3 Chia lấy phần nguyên (vì a và b là int)
% Chia lấy dư a % b 1 Lấy phần dư khi chia a cho b
++ Tăng 1 đơn vị a++ hoặc ++a 11 Tăng giá trị của a lên 1 đơn vị.

        System.out.println("a - b = " + (a - b)); // 11
        System.out.println("a * b = " + (a * b)); // 60
        System.out.println("a / b = " + (a / b)); // 3 (chia nguyên)
        System.out.println("a % b = " + (a % b)); // 3 (số dư)

        double x = 15.0;

        // 🔹 TĂNG/GIẢM GIÁ TRỊ
