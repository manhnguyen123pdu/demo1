PHẦN 1: QUEUE INTERFACE

1.1 QUEUE LÀ GÌ?

Định nghĩa: Queue là interface trong Java, đại diện cho một hàng đợi - cấu trúc dữ liệu hoạt động theo nguyên tắc FIFO (First-In-First-Out) - Vào trước ra trước.

Đặc điểm:

  • ✅ Phần tử được thêm vào cuối hàng đợi
  • ✅ Phần tử được lấy ra từ đầu hàng đợi
  • ✅ Có thể giới hạn dung lượng (bounded) hoặc không giới hạn (unbounded)
  • ✅ Hỗ trợ các thao tác: thêm, lấy, xóa, kiểm tra

Hình dung:

Thêm vào → [ 5 ][ 3 ][ 8 ][ 1 ] → Lấy ra (cuối) (đầu)

1.2 CÚ PHÁP KHAI BÁO QUEUE

1.2.1 Cú pháp tổng quát

Khai báo Queue
// Cách 1: Dùng interface Queue (khuyến nghị)
Queue<KiểuDữLiệu> tênBiến = new PriorityQueue<>();
Queue<KiểuDữLiệu> tênBiến = new LinkedList<>(); // LinkedList implements Queue
Queue<KiểuDữLiệu> tênBiến = new ArrayDeque<>(); // ArrayDeque implements Queue

// Cách 2: Dùng class cụ thể
PriorityQueue<KiểuDữLiệu> tênBiến = new PriorityQueue<>();
LinkedList<KiểuDữLiệu> tênBiến = new LinkedList<>(); // LinkedList cũng là Queue
ArrayDeque<KiểuDữLiệu> tênBiến = new ArrayDeque<>();

1.2.2 Ví dụ khai báo

Ví dụ khai báo
// Queue với kiểu String
Queue<String> stringQueue = new PriorityQueue<>();
Queue<String> linkedListQueue = new LinkedList<>();
Queue<String> arrayDequeQueue = new ArrayDeque<>();

// Queue với kiểu Integer
Queue<Integer> intQueue = new PriorityQueue<>();
Queue<Integer> linkedIntQueue = new LinkedList<>();

// Queue với kiểu đối tượng tự định nghĩa
Queue<Customer> customerQueue = new PriorityQueue<>(); // Customer phải implements Comparable

1.3 CÁC PHƯƠNG THỨC CỦA QUEUE

Thao tác Ném exception Trả về giá trị đặc biệt
Thêm add(e) offer(e)
Lấy và xóa remove() poll()
Lấy (không xóa) element() peek()

1.3.1 Phương thức add(E e)

Cú pháp: boolean add(E e)

Giải thích: Thêm phần tử vào cuối hàng đợi. Nếu thêm thành công: trả về true. Nếu không thể thêm (do giới hạn dung lượng): ném IllegalStateException.

Ví dụ add()
Queue<String> queue = new LinkedList<>();
queue.add("Java");
queue.add("Python");
queue.add("JavaScript");

System.out.println(queue); // [Java, Python, JavaScript]

1.3.2 Phương thức offer(E e)

Cú pháp: boolean offer(E e)

Giải thích: Thêm phần tử vào cuối hàng đợi. Nếu thêm thành công: trả về true. Nếu không thể thêm: trả về false (không ném exception).

Ví dụ offer()
Queue<String> queue = new LinkedList<>();
System.out.println(queue.offer("Java")); // true
System.out.println(queue.offer("Python")); // true
System.out.println(queue); // [Java, Python]

1.3.3 Phương thức remove()

Cú pháp: E remove()

Giải thích: Lấy và xóa phần tử đầu hàng đợi. Nếu hàng đợi không rỗng: trả về phần tử đầu. Nếu hàng đợi rỗng: ném NoSuchElementException.

Ví dụ remove()
Queue<String> queue = new LinkedList<>();
queue.offer("Java");
queue.offer("Python");
queue.offer("JavaScript");

String element = queue.remove();
System.out.println("Phần tử lấy ra: " + element); // Java
System.out.println("Queue còn lại: " + queue); // [Python, JavaScript]

// queue.remove(); // Sẽ ném exception nếu queue rỗng

1.3.4 Phương thức poll()

Cú pháp: E poll()

Giải thích: Lấy và xóa phần tử đầu hàng đợi. Nếu hàng đợi không rỗng: trả về phần tử đầu. Nếu hàng đợi rỗng: trả về null (không ném exception).

Ví dụ poll()
Queue<String> queue = new LinkedList<>();
queue.offer("Java");
queue.offer("Python");

String element = queue.poll();
System.out.println("Phần tử lấy ra: " + element); // Java
System.out.println("Queue còn lại: " + queue); // [Python]

String empty = queue.poll(); // Lấy tiếp
System.out.println(empty); // Python
System.out.println(queue.poll()); // null (queue rỗng)

1.3.5 Phương thức element()

Cú pháp: E element()

Giải thích: Lấy (không xóa) phần tử đầu hàng đợi. Nếu hàng đợi không rỗng: trả về phần tử đầu. Nếu hàng đợi rỗng: ném NoSuchElementException.

Ví dụ element()
Queue<String> queue = new LinkedList<>();
queue.offer("Java");
queue.offer("Python");

String head = queue.element();
System.out.println("Phần tử đầu: " + head); // Java
System.out.println("Queue vẫn giữ nguyên: " + queue); // [Java, Python]

1.3.6 Phương thức peek()

Cú pháp: E peek()

Giải thích: Lấy (không xóa) phần tử đầu hàng đợi. Nếu hàng đợi không rỗng: trả về phần tử đầu. Nếu hàng đợi rỗng: trả về null (không ném exception).

Ví dụ peek()
Queue<String> queue = new LinkedList<>();
queue.offer("Java");
queue.offer("Python");

String head = queue.peek();
System.out.println("Phần tử đầu: " + head); // Java
System.out.println("Queue vẫn giữ nguyên: " + queue); // [Java, Python]

Queue<String> emptyQueue = new LinkedList<>();
System.out.println(emptyQueue.peek()); // null

1.4 SO SÁNH HAI BỘ PHƯƠNG THỨC

Tình huống add/remove/element offer/poll/peek
Thêm thành công add() → true offer() → true
Thêm thất bại Ném exception Trả về false
Lấy từ queue không rỗng remove() → phần tử poll() → phần tử
Lấy từ queue rỗng Ném exception Trả về null
Xem đầu queue không rỗng element() → phần tử peek() → phần tử
Xem đầu queue rỗng Ném exception Trả về null
QueueComparison.java
public class QueueComparison {
    public static void main(String[] args) {
        Queue<String> queue = new LinkedList<>();
        
        // Bộ ném exception
        try {
            queue.add("A");
            String head = queue.element();
            String removed = queue.remove();
        } catch (Exception e) {
            System.out.println("Exception: " + e);
        }
        
        // Bộ trả về giá trị đặc biệt
        queue.offer("A");
        String head = queue.peek();
        String removed = queue.poll();
        String null_value = queue.poll(); // queue rỗng → null
        
        System.out.println("Khi queue rỗng, poll(): " + null_value);
    }
}

PHẦN 2: PRIORITY QUEUE

2.1 PRIORITY QUEUE LÀ GÌ?

Định nghĩa: PriorityQueue là class triển khai Queue interface, nhưng không theo FIFO mà sắp xếp phần tử theo độ ưu tiên.

Đặc điểm:

  • ✅ Phần tử được sắp xếp theo thứ tự ưu tiên
  • ✅ Phần tử có độ ưu tiên cao nhất được lấy ra trước
  • ❌ Không cho phép null
  • ✅ Không thread-safe
  • ⚡ Hiệu suất: O(log n) cho thêm/xóa

Nguyên tắc hoạt động:

  • Mặc định: sắp xếp theo thứ tự tự nhiên (tăng dần với số, alphabet với String)
  • Có thể dùng Comparator để định nghĩa thứ tự ưu tiên
  • Phần tử nhỏ nhất (theo thứ tự) được lấy ra trước

2.2 CÚ PHÁP KHAI BÁO PRIORITY QUEUE

2.2.1 Cú pháp tổng quát

Khai báo PriorityQueue
// Cách 1: Dùng interface Queue (khuyến nghị)
Queue<KiểuDữLiệu> tênBiến = new PriorityQueue<>();

// Cách 2: Dùng class cụ thể
PriorityQueue<KiểuDữLiệu> tênBiến = new PriorityQueue<>();

// Cách 3: Với Comparator tùy chỉnh
Queue<KiểuDữLiệu> tênBiến = new PriorityQueue<>(comparator);
PriorityQueue<KiểuDữLiệu> tênBiến = new PriorityQueue<>(comparator);

2.2.2 Các constructor

Constructors
// 1. Constructor mặc định (thứ tự tự nhiên)
PriorityQueue<Integer> pq1 = new PriorityQueue<>();

// 2. Constructor với initial capacity
PriorityQueue<Integer> pq2 = new PriorityQueue<>(20);

// 3. Constructor với Comparator
PriorityQueue<Integer> pq3 = new PriorityQueue<>(Comparator.reverseOrder());

// 4. Constructor từ Collection khác
List<Integer> list = Arrays.asList(5, 1, 3, 2, 4);
PriorityQueue<Integer> pq4 = new PriorityQueue<>(list);

// 5. Constructor từ PriorityQueue khác
PriorityQueue<Integer> pq5 = new PriorityQueue<>(pq4);

2.2.3 Ví dụ khai báo

Ví dụ khai báo
// PriorityQueue với thứ tự tự nhiên (tăng dần)
Queue<Integer> minHeap = new PriorityQueue<>();
minHeap.offer(5);
minHeap.offer(1);
minHeap.offer(3);
System.out.println(minHeap); // [1, 5, 3] (không đảm bảo thứ tự in, nhưng lấy ra đúng)

// PriorityQueue với Comparator giảm dần (ưu tiên số lớn trước)
Queue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
maxHeap.offer(5);
maxHeap.offer(1);
maxHeap.offer(3);

// PriorityQueue với String (thứ tự alphabet)
Queue<String> stringPQ = new PriorityQueue<>();
stringPQ.offer("Java");
stringPQ.offer("Python");
stringPQ.offer("C++");

2.3 VÍ DỤ VỀ PRIORITY QUEUE

PriorityQueueExample.java
public class PriorityQueueExample {
    public static void main(String[] args) {
        // Tạo PriorityQueue với thứ tự tự nhiên (tăng dần)
        Queue<Integer> pq = new PriorityQueue<>();
        
        // Thêm phần tử
        pq.offer(30);
        pq.offer(10);
        pq.offer(50);
        pq.offer(20);
        pq.offer(40);
        
        System.out.println("PriorityQueue: " + pq); // [10, 20, 50, 30, 40] (cấu trúc heap)
        
        // Lấy phần tử theo thứ tự ưu tiên
        System.out.println("Lấy lần 1: " + pq.poll()); // 10
        System.out.println("Lấy lần 2: " + pq.poll()); // 20
        System.out.println("Lấy lần 3: " + pq.poll()); // 30
        System.out.println("Lấy lần 4: " + pq.poll()); // 40
        System.out.println("Lấy lần 5: " + pq.poll()); // 50
        System.out.println("Lấy lần 6: " + pq.poll()); // null
    }
}

2.4 PRIORITY QUEUE VỚI COMPARATOR TÙY CHỈNH

PriorityQueueWithComparator.java
public class PriorityQueueWithComparator {
    public static void main(String[] args) {
        // 1. Ưu tiên số lớn trước (max heap)
        Queue<Integer> maxHeap = new PriorityQueue<>(Comparator.reverseOrder());
        maxHeap.offer(30);
        maxHeap.offer(10);
        maxHeap.offer(50);
        maxHeap.offer(20);
        
        System.out.println("Max heap (lấy số lớn trước):");
        while (!maxHeap.isEmpty()) {
            System.out.print(maxHeap.poll() + " "); // 50 30 20 10
        }
        System.out.println();
        
        // 2. Ưu tiên chuỗi theo độ dài (ngắn nhất trước)
        Queue<String> byLength = new PriorityQueue<>(Comparator.comparingInt(String::length));
        byLength.offer("Java");
        byLength.offer("Python");
        byLength.offer("C");
        byLength.offer("JavaScript");
        byLength.offer("Go");
        
        System.out.println("Ưu tiên độ dài ngắn nhất:");
        while (!byLength.isEmpty()) {
            System.out.print(byLength.poll() + " "); // C Go Java Python JavaScript
        }
        System.out.println();
        
        // 3. Kết hợp nhiều tiêu chí
        Queue<String> complex = new PriorityQueue<>(
            Comparator.comparingInt(String::length)
                  .thenComparing(Comparator.naturalOrder())
        );
        complex.offer("Banana");
        complex.offer("Apple");
        complex.offer("Kiwi");
        complex.offer("Pear");
        
        System.out.println("Ưu tiên độ dài, sau đó alphabet:");
        while (!complex.isEmpty()) {
            System.out.print(complex.poll() + " "); // Kiwi Pear Apple Banana
        }
    }
}

2.5 PRIORITY QUEUE VỚI ĐỐI TƯỢNG TỰ ĐỊNH NGHĨA

Task.java
class Task implements Comparable<Task> {
    private String name;
    private int priority;
    
    public Task(String name, int priority) {
        this.name = name;
        this.priority = priority;
    }
    
    @Override
    public int compareTo(Task other) {
        // Ưu tiên số priority lớn hơn (cao hơn)
        return Integer.compare(other.priority, this.priority);
    }
    
    @Override
    public String toString() {
        return name + "(" + priority + ")";
    }
}

public class PriorityQueueWithObjects {
    public static void main(String[] args) {
        Queue<Task> taskQueue = new PriorityQueue<>();
        
        taskQueue.offer(new Task("Học Java", 5));
        taskQueue.offer(new Task("Làm bài tập", 3));
        taskQueue.offer(new Task("Nộp báo cáo", 5));
        taskQueue.offer(new Task("Đi họp", 4));
        taskQueue.offer(new Task("Nghỉ trưa", 1));
        
        System.out.println("Công việc theo độ ưu tiên:");
        while (!taskQueue.isEmpty()) {
            System.out.println(" " + taskQueue.poll());
        }
    }
}

2.6 ỨNG DỤNG THỰC TẾ CỦA PRIORITY QUEUE

EmergencyRoom.java
class Patient {
    private String name;
    private int severity; // 1: nhẹ, 5: nguy kịch
    
    public Patient(String name, int severity) {
        this.name = name;
        this.severity = severity;
    }
    
    @Override
    public String toString() {
        return name + " (mức " + severity + ")";
    }
}

public class EmergencyRoom {
    public static void main(String[] args) {
        // Bệnh nhân nguy kịch được ưu tiên trước
        Queue<Patient> emergencyQueue = new PriorityQueue<>(
            (p1, p2) -> Integer.compare(p2.severity, p1.severity)
        );
        
        emergencyQueue.offer(new Patient("Nguyễn Văn A", 3));
        emergencyQueue.offer(new Patient("Trần Thị B", 5)); // Nguy kịch
        emergencyQueue.offer(new Patient("Lê Văn C", 2));
        emergencyQueue.offer(new Patient("Phạm Thị D", 4));
        emergencyQueue.offer(new Patient("Hoàng Văn E", 1));
        
        System.out.println("Thứ tự khám bệnh:");
        while (!emergencyQueue.isEmpty()) {
            System.out.println(" " + emergencyQueue.poll());
        }
    }
}

PHẦN 3: DEQUE INTERFACE

3.1 DEQUE LÀ GÌ?

Định nghĩa: Deque (phát âm là "deck") là interface trong Java, đại diện cho một hàng đợi hai đầu (Double-Ended Queue). Cho phép thêm và lấy phần tử ở cả hai đầu.

Đặc điểm:

  • ✅ Có thể dùng như Queue (FIFO) - thêm ở cuối, lấy ở đầu
  • ✅ Có thể dùng như Stack (LIFO) - thêm và lấy ở cùng một đầu
  • ✅ Hỗ trợ thao tác ở cả đầu và cuối
  • ✅ Có thể giới hạn dung lượng hoặc không

Hình dung:

Thêm/lấy ở đầu ← [ 5 ][ 3 ][ 8 ][ 1 ] → Thêm/lấy ở cuối

3.2 CÚ PHÁP KHAI BÁO DEQUE

3.2.1 Cú pháp tổng quát

Khai báo Deque
// Cách 1: Dùng interface Deque (khuyến nghị)
Deque<KiểuDữLiệu> tênBiến = new ArrayDeque<>();
Deque<KiểuDữLiệu> tênBiến = new LinkedList<>(); // LinkedList implements Deque

// Cách 2: Dùng class cụ thể
ArrayDeque<KiểuDữLiệu> tênBiến = new ArrayDeque<>();
LinkedList<KiểuDữLiệu> tênBiến = new LinkedList<>(); // LinkedList cũng là Deque

3.2.2 Ví dụ khai báo

Ví dụ khai báo
// Deque với ArrayDeque (khuyến nghị)
Deque<String> arrayDeque = new ArrayDeque<>();
Deque<Integer> intDeque = new ArrayDeque<>();

// Deque với LinkedList
Deque<String> linkedDeque = new LinkedList<>();
Deque<Double> doubleDeque = new LinkedList<>();

3.3 CÁC PHƯƠNG THỨC CỦA DEQUE

3.3.1 Thao tác ở ĐẦU (First)

Thao tác Ném exception Trả về giá trị đặc biệt
Thêm addFirst(e) offerFirst(e)
Lấy và xóa removeFirst() pollFirst()
Lấy (không xóa) getFirst() peekFirst()
Ví dụ thao tác đầu
Deque<String> deque = new ArrayDeque<>();

// Thêm ở đầu
deque.addFirst("C");
deque.offerFirst("B");
deque.offerFirst("A");
System.out.println(deque); // [A, B, C]

// Xem đầu
System.out.println(deque.getFirst()); // A
System.out.println(deque.peekFirst()); // A

// Lấy và xóa đầu
System.out.println(deque.removeFirst()); // A
System.out.println(deque.pollFirst()); // B
System.out.println(deque); // [C]

3.3.2 Thao tác ở CUỐI (Last)

Thao tác Ném exception Trả về giá trị đặc biệt
Thêm addLast(e) offerLast(e)
Lấy và xóa removeLast() pollLast()
Lấy (không xóa) getLast() peekLast()
Ví dụ thao tác cuối
Deque<String> deque = new ArrayDeque<>();

// Thêm ở cuối
deque.addLast("A");
deque.offerLast("B");
deque.offerLast("C");
System.out.println(deque); // [A, B, C]

// Xem cuối
System.out.println(deque.getLast()); // C
System.out.println(deque.peekLast()); // C

// Lấy và xóa cuối
System.out.println(deque.removeLast()); // C
System.out.println(deque.pollLast()); // B
System.out.println(deque); // [A]

3.4 DEQUE NHƯ QUEUE (FIFO)

Khi dùng Deque như Queue (hàng đợi):

  • Thêm ở cuối: addLast(e) hoặc offerLast(e)
  • Lấy ở đầu: removeFirst() hoặc pollFirst()
  • Xem đầu: getFirst() hoặc peekFirst()
DequeAsQueue.java
public class DequeAsQueue {
    public static void main(String[] args) {
        Deque<String> queue = new ArrayDeque<>();
        
        // Thêm vào cuối (enqueue)
        queue.offerLast("Java");
        queue.offerLast("Python");
        queue.offerLast("JavaScript");
        
        System.out.println("Queue: " + queue); // [Java, Python, JavaScript]
        
        // Lấy từ đầu (dequeue)
        while (!queue.isEmpty()) {
            System.out.println("Lấy: " + queue.pollFirst());
        }
    }
}

3.5 DEQUE NHƯ STACK (LIFO)

Khi dùng Deque như Stack (ngăn xếp):

  • Thêm ở đầu: addFirst(e) hoặc offerFirst(e)
  • Lấy ở đầu: removeFirst() hoặc pollFirst()
  • Xem đầu: getFirst() hoặc peekFirst()
DequeAsStack.java
public class DequeAsStack {
    public static void main(String[] args) {
        Deque<String> stack = new ArrayDeque<>();
        
        // Push - thêm vào đầu
        stack.offerFirst("Java");
        stack.offerFirst("Python");
        stack.offerFirst("JavaScript");
        
        System.out.println("Stack: " + stack); // [JavaScript, Python, Java]
        
        // Pop - lấy từ đầu
        while (!stack.isEmpty()) {
            System.out.println("Pop: " + stack.pollFirst());
        }
    }
}

PHẦN 4: ARRAY DEQUE

4.1 ARRAY DEQUE LÀ GÌ?

Định nghĩa: ArrayDeque là class triển khai Deque interface, sử dụng mảng động để lưu trữ. Đây là lựa chọn tốt nhất cho Queue và Stack trong hầu hết trường hợp.

Đặc điểm:

  • ✅ Nhanh hơn LinkedList cho hầu hết thao tác
  • ✅ Không cho phép null
  • ✅ Tự động mở rộng kích thước
  • ✅ Có thể dùng làm Queue (FIFO) hoặc Stack (LIFO)
  • ❌ Không thread-safe

4.2 CÚ PHÁP KHAI BÁO ARRAY DEQUE

4.2.1 Cú pháp tổng quát

Khai báo ArrayDeque
// Cách 1: Dùng interface Deque (khuyến nghị)
Deque<KiểuDữLiệu> tênBiến = new ArrayDeque<>();

// Cách 2: Dùng class cụ thể
ArrayDeque<KiểuDữLiệu> tênBiến = new ArrayDeque<>();

4.4 SO SÁNH ARRAY DEQUE VỚI STACK VÀ QUEUE

ArrayDequeComparison.java
public class ArrayDequeComparison {
    public static void main(String[] args) {
        // 1. Dùng như Queue (FIFO) - nhanh hơn LinkedList
        Deque<String> queue = new ArrayDeque<>();
        queue.offer("A");
        queue.offer("B");
        queue.offer("C");
        
        System.out.println("Queue FIFO:");
        while (!queue.isEmpty()) {
            System.out.println(" " + queue.poll()); // A, B, C
        }
        
        // 2. Dùng như Stack (LIFO) - nhanh hơn Stack class
        Deque<String> stack = new ArrayDeque<>();
        stack.push("A"); // push = addFirst
        stack.push("B");
        stack.push("C");
        
        System.out.println("Stack LIFO:");
        while (!stack.isEmpty()) {
            System.out.println(" " + stack.pop()); // C, B, A
        }
        
        // 3. So sánh tốc độ
        long start, end;
        
        // ArrayDeque
        Deque<Integer> arrayDeque = new ArrayDeque<>();
        start = System.nanoTime();
        for (int i = 0; i < 1000000; i++) {
            arrayDeque.addFirst(i);
        }
        end = System.nanoTime();
        System.out.println("ArrayDeque thêm 1 triệu phần tử: " + (end - start) + " ns");
        
        // LinkedList
        Deque<Integer> linkedList = new LinkedList<>();
        start = System.nanoTime();
        for (int i = 0; i < 1000000; i++) {
            linkedList.addFirst(i);
        }
        end = System.nanoTime();
        System.out.println("LinkedList thêm 1 triệu phần tử: " + (end - start) + " ns");
    }
}

        System.out.println("List: " + list);
        
        // 8. Stream operations
        System.out.println("Số chẵn:");
        Arrays.stream(numbers)
              .filter(n -> n % 2 == 0)
              .sorted()
              .forEach(n -> System.out.print(n + " "));
        System.out.println();
        
        // 9. Thống kê
        System.out.println("Min: " + Arrays.stream(numbers).min().getAsInt());
        System.out.println("Max: " + Arrays.stream(numbers).max().getAsInt());
        System.out.println("Sum: " + Arrays.stream(numbers).sum());
    }
}
