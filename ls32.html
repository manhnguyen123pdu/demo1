PHẦN 1: NIO (NEW I/O) - JAVA.NIO

1.1 NIO LÀ GÌ?

Định nghĩa:

NIO (New I/O) là API được giới thiệu từ Java 1.4, cung cấp các tính năng I/O hiệu suất cao hơn so với IO truyền thống (java.io).

Mục đích:

  • Tăng tốc độ xử lý I/O với buffer và channel
  • Hỗ trợ non-blocking I/O (I/O không chặn)
  • Xử lý file hiệu quả hơn với Path, Files
  • Hỗ trợ asynchronous I/O (Java 7+)

So sánh IO truyền thống vs NIO:

Đặc điểm IO (java.io) NIO (java.nio)
Hướng tiếp cận Stream-oriented Buffer-oriented
Đọc/ghi Theo từng byte/ký tự Đọc/ghi vào buffer
Blocking Blocking Non-blocking có thể
Xử lý file File class Path, Files, FileSystem
Hiệu suất Trung bình Cao hơn

1.2 PATH VÀ FILES - API CƠ BẢN CỦA NIO

1.2.1 Path Interface

Định nghĩa:

Path là interface đại diện cho đường dẫn file/thư mục, thay thế cho File class trong IO truyền thống.

Cú pháp khởi tạo Path:
Cú pháp
// Dùng Paths.get()
Path path1 = Paths.get("C:/test/data.txt");
Path path2 = Paths.get("C:/test", "data.txt");
Path path3 = Paths.get("C:", "test", "data.txt");

// Dùng Path.of() (Java 11+)
Path path4 = Path.of("C:/test/data.txt");
Path path5 = Path.of("C:/test", "data.txt");

Giải thích cú pháp:

  • Paths.get(String first, String... more): tạo Path từ một hoặc nhiều chuỗi
  • Path.of(): tương tự, tiện lợi hơn (Java 11+)
PathExample.java
import java.nio.file.Path;
import java.nio.file.Paths;

public class PathExample {
    public static void main(String[] args) {
        // Tạo Path
        Path path1 = Paths.get("C:/test/data.txt");
        Path path2 = Paths.get("C:/test", "data.txt");
        Path path3 = Path.of("C:/test/data.txt");
        
        System.out.println("path1: " + path1);
        System.out.println("path2: " + path2);
        System.out.println("path3: " + path3);
        
        // Các thông tin của Path
        System.out.println("Tên file: " + path1.getFileName()); // data.txt
        System.out.println("Thư mục cha: " + path1.getParent()); // C:/test
        System.out.println("Gốc: " + path1.getRoot()); // C:/
        System.out.println("Số phần tử: " + path1.getNameCount()); // 2
        
        // Duyệt các phần tử
        for (int i = 0; i < path1.getNameCount(); i++) {
            System.out.println("Phần tử " + i + ": " + path1.getName(i));
        }
    }
}

1.2.2 Các phương thức quan trọng của Path

Phương thức Mô tả
getFileName() Lấy tên file/thư mục cuối cùng
getParent() Lấy thư mục cha
getRoot() Lấy gốc đường dẫn
getNameCount() Số lượng phần tử trong đường dẫn
getName(int index) Lấy phần tử tại vị trí index
toAbsolutePath() Chuyển thành đường dẫn tuyệt đối
resolve(String other) Nối đường dẫn
resolveSibling(String other) Nối với cùng thư mục cha
relativize(Path other) Lấy đường dẫn tương đối
normalize() Chuẩn hóa đường dẫn (bỏ . và ..)
toFile() Chuyển thành File (tương thích ngược)
PathMethodsExample.java
import java.nio.file.Path;
import java.nio.file.Paths;

public class PathMethodsExample {
    public static void main(String[] args) {
        Path path = Paths.get("C:/test/folder/../data.txt");
        
        System.out.println("Original: " + path);
        System.out.println("Absolute: " + path.toAbsolutePath());
        System.out.println("Normalize: " + path.normalize()); // bỏ ".."
        
        // Resolve - nối đường dẫn
        Path base = Paths.get("C:/test");
        Path file = base.resolve("data.txt");
        System.out.println("Resolve: " + file); // C:/test/data.txt
        
        // Relativize - lấy đường dẫn tương đối
        Path p1 = Paths.get("C:/test/folder/file.txt");
        Path p2 = Paths.get("C:/test/data.txt");
        Path relative = p1.relativize(p2);
        System.out.println("Relativize: " + relative); // ../data.txt
    }
}

1.3 FILES CLASS - TIỆN ÍCH XỬ LÝ FILE

1.3.1 Files class là gì?

Định nghĩa:

Files là class tiện ích (utility class) cung cấp các phương thức static để thao tác với file và thư mục, thay thế cho File class cũ.

Mục đích:

  • Kiểm tra, tạo, xóa, copy, move file
  • Đọc/ghi nội dung file một cách đơn giản
  • Làm việc với thuộc tính file

1.3.2 Kiểm tra và lấy thông tin

Cú pháp:
Cú pháp
boolean exists(Path path) // kiểm tra tồn tại
boolean notExists(Path path) // kiểm tra không tồn tại
boolean isRegularFile(Path path) // kiểm tra file thường
boolean isDirectory(Path path) // kiểm tra thư mục
boolean isReadable(Path path) // có thể đọc?
boolean isWritable(Path path) // có thể ghi?
boolean isExecutable(Path path) // có thể thực thi?
long size(Path path) throws IOException // kích thước
FileTime getLastModifiedTime(Path path) throws IOException // thời gian sửa
FilesInfoExample.java
import java.nio.file.*;
import java.io.IOException;

public class FilesInfoExample {
    public static void main(String[] args) {
        Path path = Paths.get("C:/test/data.txt");
        
        try {
            System.out.println("Tồn tại: " + Files.exists(path));
            System.out.println("Là file: " + Files.isRegularFile(path));
            System.out.println("Là thư mục: " + Files.isDirectory(path));
            System.out.println("Có thể đọc: " + Files.isReadable(path));
            System.out.println("Có thể ghi: " + Files.isWritable(path));
            
            if (Files.exists(path)) {
                System.out.println("Kích thước: " + Files.size(path) + " bytes");
                System.out.println("Sửa lần cuối: " + Files.getLastModifiedTime(path));
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

            System.out.println("File tạm: " + tempFile);
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Cú pháp:
Cú pháp

FilesCopyMoveDeleteExample.java
import java.nio.file.*;
import java.io.IOException;

public class FilesCopyMoveDeleteExample {
    public static void main(String[] args) {
        try {
            Path source = Paths.get("C:/test/data.txt");
            Path dest = Paths.get("C:/test/backup/data.txt");
            
            // Copy file
            Files.copy(source, dest, StandardCopyOption.REPLACE_EXISTING);
            System.out.println("Đã copy file");
            
            // Move (rename) file
            Path renamed = Paths.get("C:/test/backup/newdata.txt");
            Files.move(dest, renamed, StandardCopyOption.REPLACE_EXISTING);
            System.out.println("Đã move file");
            
            // Delete file
            boolean deleted = Files.deleteIfExists(renamed);
            System.out.println("Đã xóa? " + deleted);
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

import java.nio.file.*;

            String text = Files.readString(path);

1.4 BUFFER VÀ CHANNEL

1.4.1 Buffer là gì?

Định nghĩa:

Buffer là một vùng nhớ (memory block) dùng để lưu trữ dữ liệu tạm thời trong quá trình I/O.

Các loại Buffer:

  • ByteBuffer: lưu byte
  • CharBuffer: lưu ký tự
  • IntBuffer: lưu số nguyên
  • LongBuffer: lưu số long
  • FloatBuffer: lưu số thực
  • DoubleBuffer: lưu số thực double

Các thuộc tính quan trọng của Buffer:

Thuộc tính Mô tả
capacity Dung lượng tối đa (không đổi)
position Vị trí hiện tại để đọc/ghi
limit Giới hạn đọc/ghi
mark Đánh dấu vị trí (tùy chọn)

Các phương thức chính:

Phương thức
// Tạo buffer
ByteBuffer.allocate(int capacity) // Tạo buffer trên heap
ByteBuffer.allocateDirect(int capacity) // Tạo buffer trực tiếp (off-heap)
ByteBuffer.wrap(byte[] array) // Buffer từ mảng có sẵn

// Đọc/ghi
buffer.put(byte b) // Ghi 1 byte
buffer.put(byte[] src) // Ghi mảng byte
buffer.get() // Đọc 1 byte
buffer.get(byte[] dst) // Đọc vào mảng

// Chuyển đổi chế độ
buffer.flip() // Chuyển từ ghi → đọc
buffer.clear() // Chuẩn bị ghi lại (xóa)
buffer.rewind() // Đọc lại từ đầu
buffer.compact() // Nén dữ liệu
BufferExample.java
import java.nio.ByteBuffer;

public class BufferExample {
    public static void main(String[] args) {
        // Tạo buffer 10 byte
        ByteBuffer buffer = ByteBuffer.allocate(10);
        
        System.out.println("Sau khi tạo:");
        System.out.println("Capacity: " + buffer.capacity());
        System.out.println("Position: " + buffer.position());
        System.out.println("Limit: " + buffer.limit());
        
        // Ghi dữ liệu
        buffer.put((byte) 10);
        buffer.put((byte) 20);
        buffer.put((byte) 30);
        
        System.out.println("\nSau khi ghi 3 byte:");
        System.out.println("Position: " + buffer.position());
        System.out.println("Limit: " + buffer.limit());
        
        // Chuyển sang chế độ đọc
        buffer.flip();
        
        System.out.println("\nSau flip (sẵn sàng đọc):");
        System.out.println("Position: " + buffer.position());
        System.out.println("Limit: " + buffer.limit());
        
        // Đọc dữ liệu
        System.out.println("\nĐọc dữ liệu:");
        while (buffer.hasRemaining()) {
            System.out.println("Byte: " + buffer.get());
        }
        
        // Chuẩn bị ghi lại
        buffer.clear();
        System.out.println("\nSau clear:");
        System.out.println("Position: " + buffer.position());
        System.out.println("Limit: " + buffer.limit());
    }
}

1.4.2 Channel là gì?

Định nghĩa:

Channel là kênh kết nối giữa chương trình và nguồn dữ liệu (file, socket). Channel đọc/ghi dữ liệu thông qua Buffer.

Các loại Channel:

  • FileChannel: đọc/ghi file
  • SocketChannel: đọc/ghi qua socket (TCP)
  • ServerSocketChannel: lắng nghe kết nối TCP
  • DatagramChannel: đọc/ghi qua UDP

Các phương thức chính của FileChannel:

Phương thức Mô tả
int read(ByteBuffer dst) Đọc vào buffer
int write(ByteBuffer src) Ghi từ buffer
long position() Lấy vị trí hiện tại
FileChannel position(long newPosition) Di chuyển vị trí
long size() Lấy kích thước file
FileChannel truncate(long size) Cắt file
void force(boolean metaData) Đẩy dữ liệu xuống ổ đĩa
long transferTo(long position, long count, WritableByteChannel target) Chuyển dữ liệu sang channel khác
FileChannelReadExample.java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.io.IOException;

public class FileChannelReadExample {
    public static void main(String[] args) {
        Path path = Paths.get("C:/test/data.txt");
        
        try (FileChannel channel = FileChannel.open(path, StandardOpenOption.READ)) {
            // Tạo buffer
            ByteBuffer buffer = ByteBuffer.allocate(1024);
            
            // Đọc từ channel vào buffer
            int bytesRead = channel.read(buffer);
            
            while (bytesRead != -1) {
                buffer.flip(); // sẵn sàng đọc từ buffer
                
                while (buffer.hasRemaining()) {
                    System.out.print((char) buffer.get());
                }
                
                buffer.clear(); // chuẩn bị đọc tiếp
                bytesRead = channel.read(buffer);
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
FileChannelWriteExample.java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.io.IOException;

public class FileChannelWriteExample {
    public static void main(String[] args) {
        Path path = Paths.get("C:/test/output.txt");
        String data = "Hello, NIO Channel!";
        
        try (FileChannel channel = FileChannel.open(path,
                StandardOpenOption.CREATE,
                StandardOpenOption.WRITE)) {
            
            // Tạo buffer và ghi dữ liệu
            ByteBuffer buffer = ByteBuffer.allocate(1024);
            buffer.put(data.getBytes());
            buffer.flip(); // sẵn sàng đọc để ghi
            
            // Ghi từ buffer vào channel
            while (buffer.hasRemaining()) {
                channel.write(buffer);
            }
            
            System.out.println("Đã ghi file thành công!");
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}
FileChannelCopyExample.java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.io.IOException;

public class FileChannelCopyExample {
    public static void main(String[] args) {
        Path source = Paths.get("C:/test/source.dat");
        Path dest = Paths.get("C:/test/dest.dat");
        
        try (FileChannel sourceChannel = FileChannel.open(source, StandardOpenOption.READ);
             FileChannel destChannel = FileChannel.open(dest,
                     StandardOpenOption.CREATE,
                     StandardOpenOption.WRITE)) {
            
            // Copy trực tiếp từ sourceChannel sang destChannel
            long position = 0;
            long size = sourceChannel.size();
            long transferred = sourceChannel.transferTo(position, size, destChannel);
            
            System.out.println("Đã copy " + transferred + " bytes");
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

1.4.3 Memory-Mapped File

Định nghĩa:

Memory-Mapped File cho phép ánh xạ một vùng file vào bộ nhớ, cho phép đọc/ghi file như thao tác với mảng.

Cú pháp:
Cú pháp
MappedByteBuffer map(FileChannel.MapMode mode, long position, long size)
MapMode:
  • READ_ONLY: chỉ đọc
  • READ_WRITE: đọc và ghi
  • PRIVATE: thay đổi không ảnh hưởng file gốc
MemoryMappedFileExample.java
import java.nio.*;
import java.nio.channels.*;
import java.nio.file.*;
import java.io.IOException;

public class MemoryMappedFileExample {
    public static void main(String[] args) {
        Path path = Paths.get("C:/test/mapped.dat");
        
        try (FileChannel channel = FileChannel.open(path,
                StandardOpenOption.READ,
                StandardOpenOption.WRITE,
                StandardOpenOption.CREATE)) {
            
            // Ánh xạ file vào bộ nhớ
            MappedByteBuffer buffer = channel.map(
                FileChannel.MapMode.READ_WRITE, 0, 1024);
            
            // Ghi dữ liệu vào buffer (tự động ghi xuống file)
            String message = "Hello, Memory-Mapped File!";
            buffer.put(message.getBytes());
            
            // Đọc dữ liệu từ buffer
            buffer.flip();
            byte[] data = new byte[buffer.remaining()];
            buffer.get(data);
            
            System.out.println("Đọc từ mapped buffer: " + new String(data));
            
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

PHẦN 2: SERIALIZATION

2.1 SERIALIZATION LÀ GÌ?

Định nghĩa:

Serialization là quá trình chuyển đổi đối tượng thành dòng byte để có thể:

  • Lưu xuống file
  • Truyền qua mạng
  • Lưu vào database
  • Cache

Deserialization là quá trình ngược lại: chuyển dòng byte thành đối tượng.

Mục đích:

  • Persist (lưu trữ) đối tượng
  • Truyền đối tượng giữa các JVM khác nhau
  • Deep copy đối tượng

2.2 ĐIỀU KIỆN ĐỂ SERIALIZATION

Điều kiện:

  • Class phải implement interface Serializable (marker interface - không có phương thức)
  • Tất cả các field cũng phải Serializable (hoặc được đánh dấu transient)
  • Cần serialVersionUID để đảm bảo tương thích
Cú pháp:
Cú pháp
import java.io.Serializable;

public class Student implements Serializable {
    private static final long serialVersionUID = 1L;
    
    private int id;
    private String name;
    private transient String password; // không được serialize
    private double gpa;
    
    // constructors, getters, setters
}

Giải thích:

  • implements Serializable: đánh dấu class có thể serialize
  • serialVersionUID: định danh phiên bản class
  • transient: đánh dấu field không được serialize

               ", password='" + password + "'}";

2.5 SERIALVERSIONUID

2.5.1 serialVersionUID là gì?

Định nghĩa:

serialVersionUID là một số định danh duy nhất cho mỗi class, dùng để đảm bảo rằng class khi deserialize khớp với class lúc serialize.

Mục đích:

  • Xác định phiên bản của class
  • Nếu không khớp, sẽ ném InvalidClassException
Cú pháp:
Cú pháp
private static final long serialVersionUID = 1L;
Product.java
import java.io.Serializable;

public class Product implements Serializable {
    private static final long serialVersionUID = 123456789L;
    
    private int id;
    private String name;
    private double price;
    
    // Nếu sau này thêm field mới
    // private String category; // Vẫn deserialize được object cũ
}

2.6 TRANSIENT KEYWORD

2.6.1 transient là gì?

Định nghĩa:

transient là từ khóa dùng để đánh dấu các field không được serialize.

Mục đích:

  • Bỏ qua các thông tin nhạy cảm (password, credit card)
  • Bỏ qua các field có thể tính toán lại
  • Bỏ qua các field không Serializable
TransientExample.java
import java.io.*;

class User implements Serializable {
    private static final long serialVersionUID = 1L;
    
    String username;
    transient String password; // không lưu
    transient int loginAttempts; // không lưu, có thể tính lại
    
    public User(String username, String password) {
        this.username = username;
        this.password = password;
        this.loginAttempts = 0;
    }
    
    @Override
    public String toString() {
        return "User{username='" + username + "', password='" +
               password + "', attempts=" + loginAttempts + "}";
    }
}

public class TransientExample {
    public static void main(String[] args) {
        User user = new User("john123", "secret");
        
        // Serialize
        try (ObjectOutputStream oos = new ObjectOutputStream(
                new FileOutputStream("user.ser"))) {
            oos.writeObject(user);
        } catch (IOException e) {
            e.printStackTrace();
        }
        
        // Deserialize
        try (ObjectInputStream ois = new ObjectInputStream(
                new FileInputStream("user.ser"))) {
            User deserialized = (User) ois.readObject();
            System.out.println(deserialized);
            // password = null, attempts = 0
        } catch (IOException | ClassNotFoundException e) {
            e.printStackTrace();
        }
    }
}

2.8 CUSTOM SERIALIZATION

2.8.1 writeObject() và readObject() riêng

Mục đích:

Tự định nghĩa cách serialize/deserialize cho class.

Cú pháp:
Cú pháp
private void writeObject(ObjectOutputStream out) throws IOException
private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException
Employee.java
import java.io.*;

class Employee implements Serializable {
    private static final long serialVersionUID = 1L;
    
    private String name;
    private transient String encryptedPassword;
    private double salary;
    
    public Employee(String name, String password, double salary) {
        this.name = name;
        this.encryptedPassword = encrypt(password);
        this.salary = salary;
    }
    
    private static String encrypt(String s) {
        return "ENC{" + s + "}";
    }
    
    private static String decrypt(String s) {
        return s.substring(4, s.length() - 1);
    }
    
    // Custom serialization
    private void writeObject(ObjectOutputStream out) throws IOException {
        out.defaultWriteObject(); // ghi các field không transient
        out.writeUTF(encryptedPassword); // ghi transient field
    }
    
    private void readObject(ObjectInputStream in) throws IOException, ClassNotFoundException {
        in.defaultReadObject(); // đọc các field không transient
        encryptedPassword = in.readUTF(); // đọc transient field
    }
    
    @Override
    public String toString() {
        return "Employee{name='" + name + "', password='" +
               decrypt(encryptedPassword) + "', salary=" + salary + "}";
    }
}

2.9 EXTERNALIZABLE INTERFACE

2.9.1 Externalizable là gì?

Định nghĩa:

Externalizable là interface cho phép kiểm soát hoàn toàn quá trình serialization.

Khác với Serializable:

  • Serializable: tự động serialize tất cả field (trừ transient)
  • Externalizable: phải tự viết logic serialize/deserialize
Cú pháp:
Cú pháp
public interface Externalizable extends Serializable {
    void writeExternal(ObjectOutput out) throws IOException;
    void readExternal(ObjectInput in) throws IOException, ClassNotFoundException;
}
Book.java
import java.io.*;

class Book implements Externalizable {
    private String title;
    private String author;
    private double price;
    private transient String isbn;
    
    // Bắt buộc có constructor không tham số
    public Book() {}
    
    public Book(String title, String author, double price, String isbn) {
        this.title = title;
        this.author = author;
        this.price = price;
        this.isbn = isbn;
    }
    
    @Override
    public void writeExternal(ObjectOutput out) throws IOException {
        out.writeUTF(title);
        out.writeUTF(author);
        out.writeDouble(price);
        // isbn không được ghi
    }
    
    @Override
    public void readExternal(ObjectInput in) throws IOException, ClassNotFoundException {
        title = in.readUTF();
        author = in.readUTF();
        price = in.readDouble();
        isbn = "unknown"; // giá trị mặc định
    }
    
    @Override
    public String toString() {
        return "Book{title='" + title + "', author='" + author +
               "', price=" + price + ", isbn='" + isbn + "'}";
    }
}
