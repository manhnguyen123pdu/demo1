PHẦN 1: FILE CLASS

1.1 FILE LÀ GÌ?

Định nghĩa:

File là lớp trong Java (thuộc gói java.io) đại diện cho một đường dẫn đến file hoặc thư mục trên hệ thống tệp tin.

Mục đích:

  • Kiểm tra sự tồn tại của file/thư mục
  • Tạo, xóa, đổi tên file/thư mục
  • Lấy thông tin về file (kích thước, quyền truy cập, thời gian sửa đổi)
  • Liệt kê nội dung thư mục
  • Không dùng để đọc/ghi nội dung file (Stream mới làm việc đó)

Cú pháp khởi tạo:

Cú pháp
File file = new File(String pathname);
File file = new File(String parent, String child);
File file = new File(File parent, String child);

Giải thích cú pháp:

  • String pathname: đường dẫn tuyệt đối hoặc tương đối đến file/thư mục
  • String parent, String child: thư mục cha và tên file con
  • File parent, String child: đối tượng File cha và tên file con

1.2 KIỂM TRA VÀ LẤY THÔNG TIN FILE

1.2.1 exists() - Kiểm tra tồn tại

Cú pháp
boolean exists()

Mục đích:

Kiểm tra file hoặc thư mục có tồn tại trên hệ thống hay không.

FileExistsExample.java
import java.io.File;

public class FileExistsExample {
    public static void main(String[] args) {
        File file = new File("C:/test/data.txt");
        
        if (file.exists()) {
            System.out.println("File tồn tại!");
        } else {
            System.out.println("File không tồn tại!");
        }
    }
}

1.2.2 isFile() và isDirectory()

Cú pháp
boolean isFile()
boolean isDirectory()

Mục đích:

  • isFile(): kiểm tra đối tượng có phải là file (không phải thư mục)
  • isDirectory(): kiểm tra đối tượng có phải là thư mục
FileTypeExample.java
import java.io.File;

public class FileTypeExample {
    public static void main(String[] args) {
        File file1 = new File("C:/test/data.txt");
        File file2 = new File("C:/test/folder");
        
        if (file1.exists()) {
            System.out.println("data.txt là file? " + file1.isFile()); // true
            System.out.println("data.txt là thư mục? " + file1.isDirectory()); // false
        }
        
        if (file2.exists()) {
            System.out.println("folder là file? " + file2.isFile()); // false
            System.out.println("folder là thư mục? " + file2.isDirectory()); // true
        }
    }
}

String getAbsolutePath() // lấy đường dẫn tuyệt đối

Mục đích:

Lấy các thông tin về tên và đường dẫn của file.

FilePathExample.java
import java.io.File;

public class FilePathExample {
    public static void main(String[] args) {
        File file = new File("C:/test/data.txt");
        
        System.out.println("Tên file: " + file.getName()); // data.txt
        System.out.println("Đường dẫn: " + file.getPath()); // C:/test/data.txt
        System.out.println("Đường dẫn tuyệt đối: " + file.getAbsolutePath()); // C:/test/data.txt
        
        File relative = new File("data.txt");
        System.out.println("Tên: " + relative.getName()); // data.txt
        System.out.println("Đường dẫn: " + relative.getPath()); // data.txt
        System.out.println("Đường dẫn tuyệt đối: " + relative.getAbsolutePath()); // /current/path/data.txt
    }
}

1.2.4 length() - Lấy kích thước

Cú pháp
long length()

Mục đích:

Lấy kích thước của file (số byte). Trả về 0 nếu file không tồn tại hoặc là thư mục.

FileLengthExample.java
import java.io.File;

public class FileLengthExample {
    public static void main(String[] args) {
        File file = new File("C:/test/data.txt");
        
        if (file.exists() && file.isFile()) {
            long bytes = file.length();
            double kilobytes = bytes / 1024.0;
            double megabytes = kilobytes / 1024.0;
            
            System.out.println("Kích thước: " + bytes + " bytes");
            System.out.println("Kích thước: " + kilobytes + " KB");
            System.out.println("Kích thước: " + megabytes + " MB");
        }
    }
}

1.3.5 list() và listFiles() - Liệt kê nội dung thư mục

Cú pháp
String[] list() // trả về mảng tên

Mục đích:

Liệt kê nội dung bên trong thư mục.

ListDirectoryExample.java
import java.io.File;

public class ListDirectoryExample {
    public static void main(String[] args) {
        File dir = new File("C:/test");
        
        if (dir.exists() && dir.isDirectory()) {
            // Cách 1: lấy tên
            System.out.println("=== Danh sách tên ===");
            String[] names = dir.list();
            for (String name : names) {
                System.out.println(" " + name);
            }
            
            // Cách 2: lấy đối tượng File
            System.out.println("\n=== Danh sách File objects ===");
            File[] files = dir.listFiles();
            for (File f : files) {
                if (f.isFile()) {
                    System.out.println("File: " + f.getName() + " (" + f.length() + " bytes)");
                } else if (f.isDirectory()) {
                    System.out.println("Thư mục: " + f.getName());
                }
            }
        }
    }
}

import java.io.File;
import java.io.FilenameFilter;

public class ListFilteredExample {
    public static void main(String[] args) {
        File dir = new File("C:/test");
        
        // Lọc chỉ lấy file .txt

        // Dùng lambda

PHẦN 2: BYTE STREAMS

2.1 BYTE STREAMS LÀ GÌ?

Định nghĩa:

Byte Streams là các luồng xử lý dữ liệu dưới dạng byte (8-bit), phù hợp cho mọi loại file (văn bản, hình ảnh, âm thanh, video).

Đặc điểm:

  • Đọc/ghi dữ liệu theo từng byte
  • Không quan tâm đến encoding
  • Phù hợp cho dữ liệu nhị phân

Hai lớp cơ bản:

  • InputStream: lớp trừu tượng cho luồng đọc byte
  • OutputStream: lớp trừu tượng cho luồng ghi byte

Các lớp con thường dùng:

  • FileInputStream: đọc byte từ file
  • FileOutputStream: ghi byte vào file
  • BufferedInputStream: đọc có bộ đệm
  • BufferedOutputStream: ghi có bộ đệm

2.2 FILE INPUT STREAM

2.2.1 FileInputStream - Đọc byte từ file

Cú pháp khởi tạo:
Cú pháp
FileInputStream(String name) throws FileNotFoundException
FileInputStream(File file) throws FileNotFoundException
Các phương thức chính:
Phương thức Mô tả
int read() Đọc một byte, trả về giá trị 0-255, -1 nếu hết file
int read(byte[] b) Đọc vào mảng byte, trả về số byte đọc được
int read(byte[] b, int off, int len) Đọc len byte vào mảng b từ vị trí off
long skip(long n) Bỏ qua n byte
int available() Số byte có thể đọc được
void close() Đóng luồng
ByteStreamReadExample.java
import java.io.FileInputStream;
import java.io.IOException;

public class ByteStreamReadExample {
    public static void main(String[] args) {
        FileInputStream fis = null;
        
        try {
            // Mở file để đọc
            fis = new FileInputStream("C:/test/data.txt");
            
            int byteData;
            // Đọc từng byte cho đến khi hết file (-1)
            while ((byteData = fis.read()) != -1) {
                System.out.print((char) byteData); // Chuyển byte thành ký tự
            }
            
        } catch (IOException e) {
            System.out.println("Lỗi: " + e.getMessage());
        } finally {
            // Đóng luồng
            try {
                if (fis != null) {
                    fis.close();
                }
            } catch (IOException e) {
                System.out.println("Lỗi đóng luồng: " + e.getMessage());
            }
        }
    }
}
ByteStreamReadArrayExample.java
import java.io.FileInputStream;
import java.io.IOException;

public class ByteStreamReadArrayExample {
    public static void main(String[] args) {
        FileInputStream fis = null;
        
        try {
            fis = new FileInputStream("C:/test/data.txt");
            
            // Tạo bộ đệm 1024 byte
            byte[] buffer = new byte[1024];
            int bytesRead;
            
            // Đọc vào mảng, trả về số byte thực tế đọc được

                System.out.print("Đọc " + bytesRead + " byte: ");
                System.out.println(new String(buffer, 0, bytesRead));
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            try {
                if (fis != null) fis.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

2.3 FILE OUTPUT STREAM

2.3.1 FileOutputStream - Ghi byte vào file

Cú pháp khởi tạo:
Cú pháp
FileOutputStream(String name) throws FileNotFoundException

FileOutputStream(File file) throws FileNotFoundException

Giải thích:

  • append = true: ghi thêm vào cuối file
  • append = false: ghi đè lên file cũ (mặc định)
Các phương thức chính:
Phương thức Mô tả
void write(int b) Ghi một byte
void write(byte[] b) Ghi mảng byte
void write(byte[] b, int off, int len) Ghi len byte từ mảng b từ vị trí off
void flush() Đẩy dữ liệu từ bộ đệm xuống file
void close() Đóng luồng
ByteStreamWriteExample.java
import java.io.FileOutputStream;
import java.io.IOException;

public class ByteStreamWriteExample {
    public static void main(String[] args) {
        FileOutputStream fos = null;
        
        try {
            // Ghi đè file
            fos = new FileOutputStream("C:/test/output.txt");
            
            String text = "Hello, World!";
            
            // Ghi từng ký tự
            for (int i = 0; i < text.length(); i++) {
                fos.write(text.charAt(i)); // char thành int (byte)
            }
            
            System.out.println("Đã ghi file thành công!");
            
        } catch (IOException e) {
            System.out.println("Lỗi: " + e.getMessage());
        } finally {
            try {
                if (fos != null) fos.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

PHẦN 3: CHARACTER STREAMS

3.1 CHARACTER STREAMS LÀ GÌ?

Định nghĩa:

Character Streams là các luồng xử lý dữ liệu dưới dạng ký tự (16-bit Unicode), phù hợp cho file văn bản.

Đặc điểm:

  • Đọc/ghi dữ liệu theo từng ký tự
  • Xử lý tự động chuyển đổi encoding
  • Phù hợp cho file văn bản (text files)

Hai lớp cơ bản:

  • Reader: lớp trừu tượng cho luồng đọc ký tự
  • Writer: lớp trừu tượng cho luồng ghi ký tự

Các lớp con thường dùng:

  • FileReader: đọc ký tự từ file
  • FileWriter: ghi ký tự vào file
  • BufferedReader: đọc có bộ đệm, có phương thức readLine()
  • BufferedWriter: ghi có bộ đệm, có phương thức newLine()
  • InputStreamReader: chuyển byte stream → character stream
  • OutputStreamWriter: chuyển character stream → byte stream

3.2 FILE READER

PHẦN 4: TRY-WITH-RESOURCES

4.1 TRY-WITH-RESOURCES LÀ GÌ?

Định nghĩa:

Try-with-resources là cú pháp giới thiệu từ Java 7, tự động đóng các tài nguyên (resources) sau khi sử dụng, mà không cần finally block.

Mục đích:

  • Tự động đóng resources (file streams, database connections, ...)
  • Giảm boilerplate code (finally block dài dòng)
  • Tránh lỗi quên đóng resource
  • Xử lý exception tốt hơn

Yêu cầu:

Resource phải implement interface AutoCloseable hoặc Closeable.

4.2 CÚ PHÁP

Cú pháp cơ bản:

Cú pháp
try (ResourceType resource = new ResourceType()) {
    // Sử dụng resource
} catch (Exception e) {
    // Xử lý exception
}

Cú pháp với nhiều resources:

Cú pháp
try (ResourceType1 r1 = new ResourceType1();
     ResourceType2 r2 = new ResourceType2()) {
    // Sử dụng resources
} catch (Exception e) {
    // Xử lý exception
}

Giải thích:

  • Resources được khai báo trong dấu () sau try
  • Các resources tự động đóng khi thoát khỏi try block
  • Thứ tự đóng ngược với thứ tự khai báo

4.3 VÍ DỤ SO SÁNH

4.3.1 Cách cũ (trước Java 7)

OldWayExample.java
import java.io.*;

public class OldWayExample {
    public static void main(String[] args) {
        FileReader fr = null;
        BufferedReader br = null;
        
        try {
            fr = new FileReader("C:/test/data.txt");
            br = new BufferedReader(fr);
            
            String line;
            while ((line = br.readLine()) != null) {
                System.out.println(line);
            }
            
        } catch (IOException e) {
            e.printStackTrace();
        } finally {
            // Phải tự đóng và xử lý exception
            try {
                if (br != null) br.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
            try {
                if (fr != null) fr.close();
            } catch (IOException e) {
                e.printStackTrace();
            }
        }
    }
}

Vấn đề:

  • Code dài dòng, khó đọc
  • Dễ quên đóng resource
  • Phải xử lý nested try-catch

4.3.2 Cách mới (try-with-resources)

        } catch (IOException e) {
            e.printStackTrace();
        }

    }
