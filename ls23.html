PHẦN 1: THROWING EXCEPTIONS - CÁCH NÉM EXCEPTION

1.1 THROW LÀ GÌ? - GIẢI THÍCH CỰC KỲ CHI TIẾT

Hãy tưởng tượng:

Bạn là nhân viên ngân hàng. Một khách hàng đến rút tiền:

  • Nếu khách đưa sổ đỏ, đúng chữ ký → Bạn đưa tiền ✅
  • Nếu khách không có sổ, chữ ký sai → Bạn KHÔNG đưa tiền và phải BÁO CÁO lỗi

Trong lập trình cũng vậy:

  • Khi dữ liệu hợp lệ → Xử lý bình thường
  • Khi dữ liệu không hợp lệ → Bạn phải NÉM EXCEPTION để báo cáo

Trong Java, throw là từ khóa dùng để ném một exception ra khỏi chương trình hiện tại.

Cú pháp:

Cú pháp
throw new TenException("Thông báo lỗi");

Giải thích từng thành phần:

Thành phần Ý nghĩa Ví dụ
throw Từ khóa báo hiệu "tôi đang ném exception" throw
new Tạo một object mới new
TenException Tên của lớp exception IllegalArgumentException
"Thông báo lỗi" Thông điệp mô tả lỗi "Tuổi không hợp lệ"

1.2 TẠI SAO PHẢI DÙNG THROW?

Hãy nhìn vào đoạn code KHÔNG dùng throw:

Calculator.java - KHÔNG dùng throw
public class Calculator {
    
    public int divide(int a, int b) {
        // Nếu b = 0 thì sao?
        return a / b; // Sẽ bị lỗi ArithmeticException
    }
    
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        int result = calc.divide(10, 0); // Chương trình crash
        System.out.println("Kết quả: " + result); // Không bao giờ chạy đến đây
    }
}

Kết quả:

Exception in thread "main" java.lang.ArithmeticException: / by zero
    at Calculator.divide(Calculator.java:4)
    at Calculator.main(Calculator.java:8)

Vấn đề:

  • Java tự động ném ArithmeticException
  • Chương trình dừng đột ngột
  • User không biết chuyện gì xảy ra
  • Không có cơ hội xử lý lỗi

GIẢI PHÁP: Tự kiểm tra và throw exception có ý nghĩa

Calculator.java - CÓ dùng throw
public class Calculator {
    
    public int divide(int a, int b) {
        // Tự kiểm tra và throw exception
        if (b == 0) {
            throw new IllegalArgumentException("Không thể chia cho 0. Số chia phải khác 0");
        }
        return a / b;
    }
    
    public static void main(String[] args) {
        Calculator calc = new Calculator();
        
        try {
            int result = calc.divide(10, 0);
            System.out.println("Kết quả: " + result);
        } catch (IllegalArgumentException e) {
            System.out.println("Lỗi: " + e.getMessage());
            // Có thể xử lý tiếp: yêu cầu nhập lại, dùng giá trị mặc định, v.v.
        }
    }
}

1.3 CÁC LOẠI EXCEPTION CÓ SẴN TRONG JAVA

Java cung cấp rất nhiều exception có sẵn. Dưới đây là các loại thường dùng nhất:

1.3.1 IllegalArgumentException - LỖI THAM SỐ KHÔNG HỢP LỆ

Định nghĩa: Dùng khi tham số truyền vào method không hợp lệ.

ViduIllegalArgumentException.java
public class ViduIllegalArgumentException {
    
    public void setTuoi(int tuoi) {
        // IllegalArgumentException: Dùng khi tham số không hợp lệ
        if (tuoi < 0 || tuoi > 150) {
            throw new IllegalArgumentException("Tuổi phải từ 0-150. Nhận được: " + tuoi);
        }
        System.out.println("Tuổi hợp lệ: " + tuoi);
    }
    
    public static void main(String[] args) {
        ViduIllegalArgumentException obj = new ViduIllegalArgumentException();
        
        try {
            obj.setTuoi(-5); // Sẽ throw exception
        } catch (IllegalArgumentException e) {
            System.out.println("Bắt được lỗi: " + e.getMessage());
        }
    }
}

Định nghĩa: Dùng khi đối tượng là null nhưng cố gọi phương thức.

    }
}

1.3.4 NumberFormatException - LỖI ĐỊNH DẠNG SỐ

1.3.5 IndexOutOfBoundsException - LỖI CHỈ SỐ

Định nghĩa: Dùng khi truy cập chỉ số ngoài phạm vi mảng/list.

ViduIndexOutOfBoundsException.java
public class ViduIndexOutOfBoundsException {
    
    public int layPhanTu(int[] arr, int index) {
        // IndexOutOfBoundsException: Dùng khi index không hợp lệ
        if (index < 0 || index >= arr.length) {
            throw new IndexOutOfBoundsException(
                String.format("Index %d không hợp lệ. Độ dài mảng: %d", index, arr.length)
            );
        }
        return arr[index];
    }
    
    public static void main(String[] args) {
        ViduIndexOutOfBoundsException obj = new ViduIndexOutOfBoundsException();
        int[] arr = {1, 2, 3};
        
        try {
            obj.layPhanTu(arr, 5); // Sẽ throw exception
        } catch (IndexOutOfBoundsException e) {
            System.out.println("Bắt được lỗi: " + e.getMessage());
        }
    }
}

1.5 THROW VỚI CHECKED VS UNCHECKED - GIẢI THÍCH CHI TIẾT

1.5.1 Unchecked Exception (RuntimeException)

Đặc điểm:

  • KHÔNG bắt buộc khai báo throws
  • KHÔNG bắt buộc xử lý try-catch
  • Thường dùng cho lỗi lập trình
UncheckedExample.java
public class UncheckedExample {
    
    // KHÔNG cần throws trong khai báo
    public void validateAge(int age) {
        if (age < 0) {
            // Unchecked exception - không cần khai báo
            throw new IllegalArgumentException("Tuổi không thể âm: " + age);
        }
    }
    
    public void process(String data) {
        // Unchecked exception - không cần khai báo
        if (data == null) {
            throw new NullPointerException("Dữ liệu không được null");
        }
    }
    
    public static void main(String[] args) {
        UncheckedExample ex = new UncheckedExample();
        
        // Có thể xử lý hoặc không
        try {
            ex.validateAge(-5);
        } catch (IllegalArgumentException e) {
            System.out.println("Bắt lỗi: " + e.getMessage());
        }
        
        // Hoặc không xử lý - chương trình vẫn compile
        ex.process(null); // Sẽ crash khi chạy
    }
}

1.5.2 Checked Exception

Đặc điểm:

  • BẮT BUỘC khai báo throws
  • BẮT BUỘC xử lý try-catch hoặc throws tiếp
  • Thường dùng cho lỗi từ bên ngoài (file, network, database)
CheckedExample.java
import java.io.*;

public class CheckedExample {
    
    // PHẢI khai báo throws
    public void readFile(String filename) throws FileNotFoundException, IOException {
        FileReader fr = new FileReader(filename); // Có thể throw FileNotFoundException
        BufferedReader br = new BufferedReader(fr);
        String line = br.readLine(); // Có thể throw IOException
        System.out.println(line);
        br.close();
    }
    
    // Cách 1: Xử lý bằng try-catch
    public void processFile1(String filename) {
        try {
            readFile(filename);
        } catch (FileNotFoundException e) {
            System.out.println("Không tìm thấy file: " + filename);
        } catch (IOException e) {
            System.out.println("Lỗi đọc file: " + e.getMessage());
        }
    }
    
    // Cách 2: throws tiếp cho caller xử lý
    public void processFile2(String filename) throws FileNotFoundException, IOException {
        readFile(filename);
    }
    
    public static void main(String[] args) {
        CheckedExample ex = new CheckedExample();
        
        // Cách 1: try-catch
        ex.processFile1("test.txt");
        
        // Cách 2: phải try-catch ở đây
        try {
            ex.processFile2("test.txt");
        } catch (FileNotFoundException e) {
            System.out.println("File không tồn tại");
        } catch (IOException e) {
            System.out.println("Lỗi IO");
        }
    }
}

PHẦN 2: THROW TRONG THỰC TẾ - VÍ DỤ ỨNG DỤNG

2.1 HỆ THỐNG QUẢN LÝ NHÂN VIÊN

I. Mục tiêu

Xây dựng chương trình quản lý nhân viên bằng Java, sử dụng:

  • Mảng đối tượng (Employee[])
  • Xử lý ngoại lệ (Exception Handling)
  • Kiểm tra dữ liệu đầu vào (Validation)
  • Menu console cho phép thao tác

II. Yêu cầu chức năng

1. Thêm nhân viên

Cho phép nhập:

  • Mã nhân viên (String)
  • Tên nhân viên (String)
  • Tuổi (int)
  • Lương (double)

Điều kiện kiểm tra:

  • Mã không được để trống
  • Mã không được trùng trong mảng
  • Tên không được để trống
  • Tên từ 2–50 ký tự
  • Tuổi từ 18–100
  • Lương >= 0

Nếu sai, ném IllegalArgumentException.

2. Tìm nhân viên theo mã

  • Nhập mã nhân viên
  • Duyệt mảng để tìm
  • Nếu không tìm thấy, ném NullPointerException
  • Nếu tìm thấy, in thông tin nhân viên

3. Tăng lương nhân viên

  • Nhập mã nhân viên
  • Nhập phần trăm tăng (1–100%)
  • Cập nhật lương theo công thức:
Lương mới = Lương cũ * (1 + percent / 100)

Nếu phần trăm không hợp lệ, ném IllegalArgumentException.

4. Xóa nhân viên

  • Nhập mã nhân viên cần xóa
  • Nếu không tồn tại, ném NullPointerException
  • Nếu tồn tại:
    • Dịch các phần tử phía sau lên trước
    • Giảm số lượng nhân viên

III. Yêu cầu kỹ thuật

  • Khai báo mảng: Employee[] employees;
  • Biến quản lý số lượng: int count;
  • Số lượng tối đa: ví dụ 100 nhân viên
  • Không sử dụng ArrayList
  • Không sử dụng HashMap
  • Phải duyệt mảng bằng vòng lặp for

IV. Menu chương trình

=== QUẢN LÝ NHÂN VIÊN ===
1. Thêm nhân viên
2. Tìm nhân viên
3. Tăng lương
4. Xóa nhân viên
0. Thoát

Chương trình chạy lặp cho đến khi người dùng chọn 0.

Employee.java
public class Employee {

    private String id;
    private String name;
    private int age;
    private double salary;

    public Employee(String id, String name, int age, double salary) {
        this.id = id;
        this.name = name;
        this.age = age;
        this.salary = salary;
    }

    public double getSalary() { return salary; }
    public void setSalary(double salary) { this.salary = salary; }
}
EmployeeManager.java
import java.util.Scanner;

public class EmployeeManager {

    private Employee[] employees = new Employee[100];
    private int count = 0;

    public void addEmployee(String id, String name, int age, double salary) {
        if (findIndex(id) != -1)
            throw new IllegalArgumentException("Mã đã tồn tại");

        employees[count++] = new Employee(id, name, age, salary);
    }

    public Employee findEmployee(String id) {
        int index = findIndex(id);
        if (index == -1)
            throw new NullPointerException("Không tìm thấy");
        return employees[index];
    }

    public void increaseSalary(String id, double percent) {
        Employee emp = findEmployee(id);
        double newSalary = emp.getSalary() * (1 + percent / 100);
        emp.setSalary(newSalary);
    }

    public void deleteEmployee(String id) {
        int index = findIndex(id);
        if (index == -1)
            throw new NullPointerException("Không tìm thấy");

        for (int i = index; i < count - 1; i++) {
            employees[i] = employees[i + 1];
        }
        count--;
    }

    private int findIndex(String id) {
        for (int i = 0; i < count; i++) {
            if (employees[i].getId().equals(id))
                return i;
        }
        return -1;
    }
}
Main.java
import java.util.Scanner;

public class Main {
    public static void main(String[] args) {
        EmployeeManager manager = new EmployeeManager();
        Scanner sc = new Scanner(System.in);

        manager.addEmployee("E01", "Nam", 25, 5000);
        manager.increaseSalary("E01", 10);
        System.out.println( manager.findEmployee("E01").getSalary());
    }
}

PHẦN 3: BEST PRACTICES

3.1 NGUYÊN TẮC 1: THROW EARLY (NÉM SỚM)

Nguyên tắc: Kiểm tra và ném exception NGAY KHI PHÁT HIỆN LỖI, không để chương trình chạy tiếp.

❌ TỆ - Phát hiện muộn
public double tinhTrungBinh(int[] numbers) {
    int sum = 0;
    
    // Tính tổng trước - Nếu numbers null, sẽ lỗi ở đây
    for (int num : numbers) {
        sum += num;
    }
    
    // Mãi sau mới kiểm tra
    if (numbers == null) { // ❌ Quá muộn! Chương trình đã lỗi
        throw new IllegalArgumentException("Mảng không được null");
    }
    
    return (double) sum / numbers.length;
}
✅ TỐT - Kiểm tra ngay đầu
public double tinhTrungBinhTot(int[] numbers) {
    // Kiểm tra NGAY LẬP TỨC
    if (numbers == null) {
        throw new IllegalArgumentException("Mảng không được null");
    }
    
    if (numbers.length == 0) {
        throw new IllegalArgumentException("Mảng không được rỗng");
    }
    
    int sum = 0;
    for (int num : numbers) {
        sum += num;
    }
    
    return (double) sum / numbers.length;
}

3.2 NGUYÊN TẮC 2: THROW MEANINGFUL EXCEPTIONS (NÉM EXCEPTION CÓ Ý NGHĨA)

Nguyên tắc: Chọn đúng loại exception và message phải rõ ràng, chi tiết.

❌ TỆ - Dùng sai loại exception
public void setTuoi(int tuoi) {
    if (tuoi < 0) {
        throw new RuntimeException("Lỗi"); // Quá chung chung
    }
}
❌ CŨNG TỆ - Message không rõ ràng
public void setTuoi2(int tuoi) {
    if (tuoi < 0) {
        throw new IllegalArgumentException("Tuổi không hợp lệ");
    }
}
✅ TỐT - Đúng loại, message chi tiết
public void setTuoi3(int tuoi) {
    if (tuoi < 0) {
        throw new IllegalArgumentException(
            String.format("Tuổi không thể âm. Nhận được: %d", tuoi)
        );
    }
    
    if (tuoi > 150) {
        throw new IllegalArgumentException(
            String.format("Tuổi quá lớn (tối đa 150). Nhận được: %d", tuoi)
        );
    }
}

3.3 NGUYÊN TẮC 3: NEVER SWALLOW EXCEPTIONS (KHÔNG NUỐT EXCEPTION)

Nuốt exception là gì? Bắt exception nhưng không làm gì hoặc làm không đúng.

❌ TỆ NHẤT - Nuốt hoàn toàn
try {
    int result = 10 / 0;
} catch (Exception e) {
    // Không làm gì - RẤT NGUY HIỂM!
}
❌ TỆ - Chỉ in ra nhưng vẫn nuốt
try {
    processData();
} catch (Exception e) {
    e.printStackTrace(); // Chương trình vẫn chạy tiếp với state sai
}
❌ TỆ - Trả về giá trị mặc định sai
public int divide(int a, int b) {
    try {
        return a / b;
    } catch (ArithmeticException e) {
        return 0; // ❌ Trả về kết quả sai, che giấu lỗi
    }
}
✅ TỐT - Xử lý có ý nghĩa
public void processData() {
    try {
        // Code có thể gây lỗi
        readFromDatabase();
    } catch (SQLException e) {
        // 1. Log lỗi
        logger.error("Lỗi database", e);
        
        // 2. Thông báo cho user
        showError("Không thể đọc dữ liệu từ database");
        
        // 3. Rollback transaction
        rollback();
        
        // 4. Ném lại exception nếu cần
        throw new ApplicationException("Xử lý dữ liệu thất bại", e);
    }
}

3.4 NGUYÊN TẮC 4: PRESERVE CAUSE (GIỮ NGUYÊN NGUYÊN NHÂN)

Nguyên tắc: Khi chuyển exception này thành exception khác, phải giữ nguyên cause.

❌ TỆ - Mất cause
public void badMethod() {
    try {
        readFromDatabase();
    } catch (SQLException e) {
        throw new DataAccessException("Lỗi database"); // Mất thông tin gốc
    }
}
❌ CŨNG TỆ - Chỉ lấy message
public void worseMethod() {
    try {
        readFromDatabase();
    } catch (SQLException e) {
        throw new DataAccessException("Lỗi database: " + e.getMessage()); // Mất stack trace
    }
}
✅ TỐT - Giữ cause
public void goodMethod() throws DataAccessException {
    try {
        readFromDatabase();
    } catch (SQLException e) {
        throw new DataAccessException("Không thể đọc dữ liệu từ database", e);
    }
}

class DataAccessException extends Exception {
    public DataAccessException(String message, Throwable cause) {
        super(message, cause); // ✅ Giữ nguyên cause
    }
}

3.5 NGUYÊN TẮC 5: DON'T USE EXCEPTIONS FOR FLOW CONTROL

Nguyên tắc: Không dùng exception để điều khiển luồng chương trình.

❌ TỆ - Dùng exception để điều khiển
public class BadExample {
    
    public User findUser(String id) {
        try {
            // Giả sử database.find throws exception nếu không tìm thấy
            return database.find(id);
        } catch (UserNotFoundException e) {
            return null; // Dùng exception để trả về null
        }
    }
    
    public int getLength(String str) {
        try {
            return str.length();
        } catch (NullPointerException e) {
            return 0; // Dùng exception để xử lý null
        }
    }
}
✅ TỐT - Dùng cách thông thường
public class GoodExample {
    
    public Optional findUser(String id) {
        return database.findOptional(id); // Trả về Optional
    }
    
    public int getLength(String str) {
        return str != null ? str.length() : 0; // Kiểm tra null
    }
}

3.6 CHECKLIST KIỂM TRA

ChecklistExample.java
public class ChecklistExample {
    
    public void process(String data, int value) {
        // [ ] 1. Đã kiểm tra null chưa?
        if (data == null) {
            throw new IllegalArgumentException("Data cannot be null");
        }
        
        // [ ] 2. Đã kiểm tra giá trị hợp lệ chưa?
        if (value < 0 || value > 100) {
            throw new IllegalArgumentException(
                String.format("Value must be 0-100, but was: %d", value)
            );
        }
        
        // [ ] 3. Có dùng đúng loại exception không?
        // Dùng IllegalArgumentException cho lỗi tham số
        // Dùng NullPointerException cho lỗi null
        // Dùng ArithmeticException cho lỗi toán học
        
        // [ ] 4. Message có rõ ràng không?
        // throw new IllegalArgumentException("Error"); // ❌
        // throw new IllegalArgumentException("Value invalid: " + value); // ✅
        
        // [ ] 5. Có nuốt exception không?
        try {
            // code
        } catch (Exception e) {
            logger.error("Error processing data: " + data, e); // Log đầy đủ
            throw e; // Hoặc throw lại
        }
    }
}
