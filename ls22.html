PHẦN 1: TỔNG QUAN VỀ EXCEPTION TRONG JAVA

1.1 EXCEPTION LÀ GÌ?

Định nghĩa:

Exception (ngoại lệ) là một sự kiện xảy ra trong quá trình thực thi chương trình, phá vỡ luồng bình thường của các instructions.

Ví dụ về exception trong thực tế:

ExceptionExample.java
public class ExceptionExample {
    public static void main(String[] args) {
        // Ví dụ 1: Chia cho 0
        int result = 10 / 0; // ArithmeticException
        
        // Ví dụ 2: Truy cập mảng ngoài phạm vi
        int[] numbers = {1, 2, 3};
        int num = numbers[5]; // ArrayIndexOutOfBoundsException
        
        // Ví dụ 3: Null pointer
        String text = null;
        int length = text.length(); // NullPointerException
        
        // Ví dụ 4: Parse số sai định dạng
        int value = Integer.parseInt("abc"); // NumberFormatException
        
        System.out.println("Chương trình kết thúc");
    }
}

2.4 CHECKED VS UNCHECKED EXCEPTIONS

2.4.1 Checked Exceptions (Compile-time)

Định nghĩa:

Checked exceptions là những exception được kiểm tra tại thời điểm compile. Bắt buộc phải xử lý (try-catch) hoặc khai báo (throws).

Đặc điểm:
  • Phải xử lý khi compile
  • Thường liên quan đến external factors (file, network, database)
  • Kế thừa từ Exception nhưng không phải RuntimeException
CheckedExceptionExample.java
import java.io.*;

public class CheckedExceptionExample {
    public static void main(String[] args) {
        // ❌ Lỗi compile - phải xử lý
        // FileReader file = new FileReader("test.txt");
        
        // ✅ Cách 1: try-catch
        try {
            FileReader file = new FileReader("test.txt");
            BufferedReader reader = new BufferedReader(file);
            System.out.println(reader.readLine());
            reader.close();
        } catch (FileNotFoundException e) {
            System.out.println("File không tồn tại!");
        } catch (IOException e) {
            System.out.println("Lỗi đọc file!");
        }
        
        // ✅ Cách 2: throws
        readFile();
    }
    
    static void readFile() throws IOException {
        FileReader file = new FileReader("test.txt");
        BufferedReader reader = new BufferedReader(file);
        System.out.println(reader.readLine());
        reader.close();
    }
}
Các Checked Exception thường gặp:
  • IOException - Lỗi input/output
  • SQLException - Lỗi database
  • ClassNotFoundException - Không tìm thấy class
  • InterruptedException - Thread bị interrupt

2.4.2 Unchecked Exceptions (Runtime)

Định nghĩa:

Unchecked exceptions là những exception xảy ra tại runtime, không bắt buộc phải xử lý.

Đặc điểm:
  • Không bắt buộc xử lý
  • Thường do lỗi logic của lập trình viên
  • Kế thừa từ RuntimeException
UncheckedExceptionExample.java
public class UncheckedExceptionExample {
    public static void main(String[] args) {
        // Có thể xử lý hoặc không
        
        // Không xử lý - chương trình vẫn compile
        String str = null;
        // str.length(); // NullPointerException khi chạy
        
        // Có xử lý
        try {
            int[] numbers = {1, 2, 3};
            System.out.println(numbers[5]); // ArrayIndexOutOfBoundsException
        } catch (ArrayIndexOutOfBoundsException e) {
            System.out.println("Chỉ số mảng không hợp lệ!");
        }
        
        // ArithmeticException
        try {
            int result = 10 / 0;
        } catch (ArithmeticException e) {
            System.out.println("Không thể chia cho 0!");
        }
    }
}
Các Unchecked Exception thường gặp:
  • NullPointerException - Truy cập đối tượng null
  • ArrayIndexOutOfBoundsException - Chỉ số mảng không hợp lệ
  • ArithmeticException - Lỗi toán học (chia cho 0)
  • IllegalArgumentException - Tham số không hợp lệ
  • NumberFormatException - Định dạng số sai

3.1 TRY BLOCK

Định nghĩa:

try block chứa code có thể throw exception.

Cú pháp:

Cú pháp
try {
    // Code có thể throw exception
}

Ví dụ:

TryBlockExample.java
public class TryBlockExample {
    public static void main(String[] args) {

3.2 CATCH BLOCK

Định nghĩa:

catch block xử lý exception được throw từ try block.

Cú pháp:

Cú pháp
try {
    // Code có thể throw exception
} catch (ExceptionType1 e1) {
    // Xử lý exception type 1
} catch (ExceptionType2 e2) {
    // Xử lý exception type 2
}

Ví dụ 1: Multiple catch blocks

MultipleCatchExample.java
public class MultipleCatchExample {
    public static void main(String[] args) {
        try {
            Scanner scanner = new Scanner(System.in);
            
            System.out.print("Nhập số thứ nhất: ");
            int a = Integer.parseInt(scanner.nextLine());
            
            System.out.print("Nhập số thứ hai: ");
            int b = Integer.parseInt(scanner.nextLine());
            
            int result = a / b;
            System.out.println("Kết quả: " + result);
            
        } catch (NumberFormatException e) {
            System.out.println("Lỗi: Vui lòng nhập số hợp lệ!");
        } catch (ArithmeticException e) {
            System.out.println("Lỗi: Không thể chia cho 0!");
        } catch (Exception e) {
            System.out.println("Lỗi không xác định: " + e.getMessage());
        }
    }
}

Ví dụ 2: Lưu ý về thứ tự catch blocks

CatchOrderExample.java
public class CatchOrderExample {
    public static void main(String[] args) {
        // ❌ CATCH QUÁ TỔNG QUÁT - không nên
        try {
            // Code có thể throw exception
        } catch (Exception e) {
            System.out.println("Bắt tất cả lỗi");
        } catch (ArithmeticException e) {
            // ❌ LỖI COMPILE - không thể reachable

            // Code
        } catch (ArithmeticException e) {
            System.out.println("Lỗi toán học");
        } catch (NullPointerException e) {
            System.out.println("Lỗi null");
        } catch (Exception e) {
            System.out.println("Các lỗi khác");
        }
    }
}

3.3 FINALLY BLOCK

Định nghĩa:

finally block luôn được thực thi, bất kể có exception hay không. Thường dùng để dọn dẹp tài nguyên (đóng file, connection).

Cú pháp:

Cú pháp
try {
    // Code có thể throw exception
} catch (Exception e) {
    // Xử lý exception
} finally {
    // Luôn được thực thi
}

Ví dụ 1: Finally luôn chạy

FinallyAlwaysRun.java
public class FinallyAlwaysRun {
    public static void main(String[] args) {
        try {
            System.out.println("Trong try block");
            int result = 10 / 0; // Throw exception
            System.out.println("Dòng này không chạy");
        } catch (ArithmeticException e) {
            System.out.println("Trong catch block: " + e);
        } finally {
            System.out.println("Trong finally block - LUÔN CHẠY!");
        }
        
        System.out.println("Chương trình tiếp tục...");
    }
}

Ví dụ 2: Finally để đóng tài nguyên

FinallyResourceExample.java
import java.io.*;

public class FinallyResourceExample {
    public static void main(String[] args) {
        FileReader reader = null;
        try {
            reader = new FileReader("test.txt");
            int data = reader.read();
            while (data != -1) {
                System.out.print((char) data);
                data = reader.read();
            }
        } catch (FileNotFoundException e) {
            System.out.println("File không tồn tại!");
        } catch (IOException e) {
            System.out.println("Lỗi đọc file!");
        } finally {
            // Đóng file trong mọi trường hợp
            try {
                if (reader != null) {
                    reader.close();
                    System.out.println("\nĐã đóng file!");
                }
            } catch (IOException e) {
                System.out.println("Lỗi đóng file!");
            }
        }
    }
}

PHẦN 4: TRY-WITH-RESOURCES (JAVA 7+)

4.1 GIỚI THIỆU

Định nghĩa:

Try-with-resources là cú pháp tự động đóng các tài nguyên (resources) sau khi sử dụng, được giới thiệu từ Java 7.

Vấn đề trước Java 7:

❌ Trước Java 7 - phải đóng thủ công
public void readFileOld() {
    FileReader reader = null;
    BufferedReader br = null;
    try {
        reader = new FileReader("test.txt");
        br = new BufferedReader(reader);
        System.out.println(br.readLine());
    } catch (IOException e) {
        e.printStackTrace();
    } finally {
        // Phải đóng thủ công và xử lý exception khi đóng
        try {
            if (br != null) br.close();
            if (reader != null) reader.close();
        } catch (IOException e) {
            e.printStackTrace();
        }
    }
}

Vấn đề:

  • Code dài dòng, khó đọc
  • Dễ quên đóng resource
  • Phải xử lý nested try-catch

4.2 CÚ PHÁP

Cú pháp
try (ResourceType resource = new ResourceType()) {
    // Sử dụng resource
} catch (Exception e) {
    // Xử lý exception
}
// Resource tự động đóng sau khi thoát khỏi try block

4.3 AUTO CLOSEABLE INTERFACE

Điều kiện để dùng try-with-resources:

Class phải implement AutoCloseable hoặc Closeable.

// Ví dụ về Closeable

        System.out.println("Đang làm việc...");

public class TryWithResourcesExample {

    }
}

public class TryWithResourcesDB {

4.6 SO SÁNH TRUYỀN THỐNG VS TRY-WITH-RESOURCES

So sánh
// Cách truyền thống - dài dòng
public void traditionalWay() {
    Connection conn = null;
    Statement stmt = null;
    ResultSet rs = null;
    
    try {
        conn = DriverManager.getConnection(url, user, pass);
        stmt = conn.createStatement();
        rs = stmt.executeQuery("SELECT * FROM users");
        
        while (rs.next()) {
            System.out.println(rs.getString("name"));
        }
    } catch (SQLException e) {
        e.printStackTrace();
    } finally {
        // Phải đóng từng resource và xử lý exception
        try { if (rs != null) rs.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (stmt != null) stmt.close(); } catch (SQLException e) { e.printStackTrace(); }
        try { if (conn != null) conn.close(); } catch (SQLException e) { e.printStackTrace(); }
    }
}

// Cách try-with-resources - ngắn gọn
public void newWay() {
    try (Connection conn = DriverManager.getConnection(url, user, pass);
         Statement stmt = conn.createStatement();
         ResultSet rs = stmt.executeQuery("SELECT * FROM users")) {
        
        while (rs.next()) {
            System.out.println(rs.getString("name"));
        }
    } catch (SQLException e) {
        e.printStackTrace();
    }
}

5.1 THROW KEYWORD

Định nghĩa:

throw dùng để ném một exception một cách chủ động.

Cú pháp:

Cú pháp
throw new ExceptionType("Error message");

Ví dụ : Throw checked exception

ThrowCheckedExample.java
public class ThrowCheckedExample {
    public static void main(String[] args) {
        try {
            validateAge(-5);
        } catch (Exception e) {
            System.out.println("Lỗi: " + e.getMessage());
        }
    }
    
    public static void validateAge(int age) throws Exception {

    }
}

Ví dụ: Throw unchecked exception

ThrowUncheckedExample.java
public class ThrowUncheckedExample {

        // Không cần try-catch vì là unchecked
        setAge(150); // Sẽ throw exception
    }

            throw new IllegalArgumentException("Tuổi không hợp lệ: " + age);

    }

5.2 THROWS KEYWORD

Định nghĩa:

throws dùng trong khai báo method để chỉ ra method đó có thể throw exception, và caller phải xử lý.

Cú pháp:

Cú pháp
public void methodName() throws ExceptionType1, ExceptionType2 {
    // method body

Ví dụ throws Exception

5.3 PHÂN BIỆT THROW VS THROWS

Tiêu chí throw throws
Vị trí Trong method body Trong khai báo method
Mục đích Ném exception Khai báo exception có thể ném
Số lượng Một exception tại một thời điểm Nhiều exception có thể khai báo
Kiểu Chỉ ném object Throwable Khai báo các lớp Exception
Bắt buộc Không Có cho checked exception
ThrowVsThrows.java
public class ThrowVsThrows {
    // throws - khai báo
    public void method1() throws IOException, SQLException {
        // method body
    }
    
    public void method2() {
        try {
            // throw - ném exception
            throw new IOException("Lỗi IO");
        } catch (IOException e) {
            System.out.println("Xử lý: " + e);
        }
    }
}

PHẦN 6: CUSTOM EXCEPTIONS

6.1 TẠI SAO CẦN CUSTOM EXCEPTION?

Lý do:

  • Tạo exception đặc thù cho business logic
  • Cung cấp thông tin lỗi chi tiết hơn
  • Dễ dàng xử lý và debug

Vấn đề với exception có sẵn:

❌ Không rõ ràng - dùng exception có sẵn
public void withdraw(double amount) throws Exception {
    if (amount <= 0) {
        throw new Exception("Số tiền không hợp lệ");
    }
    if (balance < amount) {
        throw new Exception("Số dư không đủ");
    }
}
// Exception quá chung chung, khó xử lý riêng biệt

6.2 TẠO CUSTOM EXCEPTION

Các bước tạo custom exception:

  1. Kế thừa từ Exception (checked) hoặc RuntimeException (unchecked)
  2. Tạo constructors
  3. Thêm các fields nếu cần
Tạo Custom Exception

PHẦN 7: BEST PRACTICES

7.1 NGUYÊN TẮC VÀNG KHI XỬ LÝ EXCEPTION

1. KHÔNG BAO GIỜ BỎ QUA EXCEPTION

❌ RẤT TỆ - bỏ qua exception
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    // Không làm gì - RẤT NGUY HIỂM
}
✅ TỐT - xử lý hoặc log
try {
    int result = 10 / 0;
} catch (ArithmeticException e) {
    logger.error("Lỗi chia cho 0: ", e);
    throw new BusinessException("Không thể thực hiện phép tính", e);
}

2. CATCH CỤ THỂ, KHÔNG CATCH TỔNG QUÁT

❌ TỆ - catch Exception quá tổng quát
try {
    // code
} catch (Exception e) {
    // Không biết lỗi gì
}
✅ TỐT - catch cụ thể
try {
    // code
} catch (FileNotFoundException e) {
    System.out.println("File không tồn tại");
} catch (IOException e) {
    System.out.println("Lỗi đọc file");
} catch (SQLException e) {
    System.out.println("Lỗi database");
}

3. KHÔNG DÙNG EXCEPTION CHO ĐIỀU KHIỂN LUỒNG

❌ TỆ - dùng exception để điều khiển
try {
    int i = 0;
    while (true) {
        System.out.println(array[i]);
        i++;
    }
} catch (ArrayIndexOutOfBoundsException e) {
    // Kết thúc vòng lặp - SAI
}
✅ TỐT - dùng điều kiện bình thường
for (int i = 0; i < array.length; i++) {
    System.out.println(array[i]);
}

4. LOG EXCEPTION ĐẦY ĐỦ

❌ TỆ - chỉ log message
catch (Exception e) {
    System.out.println(e.getMessage());
}
✅ TỐT - log full stack trace
catch (Exception e) {
    logger.error("Chi tiết lỗi: ", e); // Log stack trace
    e.printStackTrace(); // In stack trace
}

5. NÉM EXCEPTION SỚM, BẮT EXCEPTION MUỘN

✅ Ném sớm - validate ngay
public void setAge(int age) {
    if (age < 0 || age > 150) {
        throw new IllegalArgumentException("Age invalid: " + age);
    }
    this.age = age;
}
✅ Bắt muộn - xử lý ở nơi có thể xử lý
public void processUser() {
    try {
        userService.updateUser(user);
    } catch (ValidationException e) {
        // Xử lý lỗi validation ở đây
        showErrorMessage(e.getErrors());
    }
}

7.2 CÁC PATTERN THÔNG DỤNG

Pattern 1: Try-Catch với Resource Cleanup

Resource Cleanup Pattern
public void processFile(String filename) {
    BufferedReader reader = null;
    try {
        reader = new BufferedReader(new FileReader(filename));
        String line;
        while ((line = reader.readLine()) != null) {
            processLine(line);
        }
    } catch (FileNotFoundException e) {
        throw new BusinessException("Không tìm thấy file: " + filename, e);
    } catch (IOException e) {
        throw new BusinessException("Lỗi đọc file: " + filename, e);
    } finally {
        if (reader != null) {
            try {
                reader.close();
            } catch (IOException e) {
                logger.warn("Lỗi đóng file", e);
            }
        }
    }
}

Pattern 2: Exception Translation

Exception Translation Pattern
public class UserService {
    private UserDAO userDAO;
    
    public User findUser(String id) {
        try {
            return userDAO.findById(id);
        } catch (SQLException e) {
            // Chuyển exception cụ thể thành business exception
            throw new UserServiceException("Không thể tìm user: " + id, e);
        }
    }
}

Pattern 3: Exception Chain

Exception Chain Pattern
try {
    // code
} catch (IOException e) {
    // Giữ nguyên cause để debug
    throw new BusinessException("Lỗi xử lý file", e);
}

7.3 CHECKLIST KHI XỬ LÝ EXCEPTION

  • ✅ Đã chọn đúng loại exception (checked vs unchecked)?
  • ✅ Exception message có rõ ràng, có ý nghĩa?
  • ✅ Đã xử lý tất cả các exception có thể xảy ra?
  • ✅ Có finally block để dọn dẹp tài nguyên?
  • ✅ Đã dùng try-with-resources cho AutoCloseable?
  • ✅ Có log exception đầy đủ (stack trace)?
  • ✅ Có throw exception phù hợp với business logic?
  • ✅ Có tránh dùng exception cho flow control?
  • ✅ Có catch đúng thứ tự (cụ thể trước, tổng quát sau)?
  • ✅ Có documentation cho các exception có thể throw?
